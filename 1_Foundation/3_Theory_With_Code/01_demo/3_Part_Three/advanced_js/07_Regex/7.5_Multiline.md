# Multiline Mode: Flag "m" Summary

The `m` flag enables multiline mode, making anchors `^` and `$` match at the start/end of **each line** (not just the whole string).

## Without `m` Flag (Default)

```javascript
let str = `1st place: Winnie
2nd place: Piglet
3rd place: Eeyore`;

console.log(str.match(/^\d/g)); // ['1'] (only first line)
console.log(str.match(/\d$/g)); // ['3'] (only last line)
```

## With `m` Flag

```javascript
let str = `1st place: Winnie
2nd place: Piglet
3rd place: Eeyore`;

// ^ matches start of each line
console.log(str.match(/^\d/gm)); // ['1', '2', '3']

// $ matches end of each line
console.log(str.match(/\d$/gm)); // ['1', '2', '3']
```

## How Anchors Work in Multiline Mode

| Anchor | Without `m`            | With `m`                                |
| ------ | ---------------------- | --------------------------------------- |
| `^`    | Start of entire string | Start of string **and** after each `\n` |
| `$`    | End of entire string   | End of string **and** before each `\n`  |

## Practical Examples

### Extract All Line Numbers

```javascript
let log = `[ERROR] Disk full
[WARN] Memory low
[INFO] Service started`;

let errorLevels = log.match(/^\[(\w+)\]/gm);
console.log(errorLevels); // ['[ERROR]', '[WARN]', '[INFO]']
```

### Validate Each Line

```javascript
let csv = `name,age
John,25
Jane,30
Bob,invalid`;

// Check each line matches "name,age" format
let validLines = csv.match(/^\w+,\d+$/gm);
console.log(validLines); // ['John,25', 'Jane,30']
```

### Find Lines Starting with Spaces

```javascript
let code = `function test() {
    return true;
  }`;

let indented = code.match(/^ {2,}/gm);
console.log(indented); // ['    ', '  '] (indentation spaces)
```

### Remove Empty Lines

```javascript
let text = `Line 1

Line 2

Line 3`;
let cleaned = text.replace(/^$\n/gm, "");
console.log(cleaned);
// Line 1
// Line 2
// Line 3
```

## Anchors vs `\n`

| Pattern | Matches                   | Includes `\n`? |
| ------- | ------------------------- | -------------- |
| `\d$`   | Digit at line end         | No             |
| `\d\n`  | Digit followed by newline | Yes            |
| `^\d`   | Digit at line start       | No             |
| `\n\d`  | Newline followed by digit | Yes            |

### Difference Example

```javascript
let str = `Winnie: 1
Piglet: 2
Eeyore: 3`;

// $ matches at line end (before \n or end)
console.log(str.match(/\d$/gm)); // ['1', '2', '3'] (3 matches)

// \n matches actual newline characters
console.log(str.match(/\d\n/g)); // ['1\n', '2\n'] (2 matches, no match for last line)
```

## Line Break Characters

`^` matches after any line break character:

- `\n` (newline)
- `\r\n` (Windows CRLF)
- `\r` (carriage return)

`$` matches before any line break character.

## Common Use Cases

### Parse Log Files

```javascript
let logs = `2024-01-01 ERROR: Connection failed
2024-01-01 INFO: Retrying
2024-01-01 ERROR: Timeout`;

let errors = logs.match(/^.*ERROR.*$/gm);
console.log(errors);
// ['2024-01-01 ERROR: Connection failed', '2024-01-01 ERROR: Timeout']
```

### Process CSV Data

```javascript
let csv = `id,name,email
1,John,john@test.com
2,Jane,jane@test.com
3,Bob,bob@test.com`;

let rows = csv.match(/^\d+,.*$/gm);
console.log(rows); // ['1,John,john@test.com', '2,Jane,jane@test.com', '3,Bob,bob@test.com']
```

### Find All Headers in Markdown

```javascript
let markdown = `# Title 1
Content here
## Title 2
More content
# Title 3`;

let headers = markdown.match(/^#{1,6} .+$/gm);
console.log(headers); // ['# Title 1', '## Title 2', '# Title 3']
```

## Summary

| Flag     | `^` Matches                         | `$` Matches                        |
| -------- | ----------------------------------- | ---------------------------------- |
| No flag  | Only start of string                | Only end of string                 |
| `m` flag | Start of string and after each `\n` | End of string and before each `\n` |

**Key Points:**

- `m` flag makes `^` and `$` work per line
- Useful for processing multiline text (logs, CSV, code)
- `^` matches after `\n` (line breaks)
- `$` matches before `\n` (line breaks)
- Use `\n` in pattern if you need newline characters in results
- Anchors are zero-width; `\n` is an actual character
