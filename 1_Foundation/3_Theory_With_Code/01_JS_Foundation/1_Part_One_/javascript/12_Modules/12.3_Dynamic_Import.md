# Dynamic Imports

## Overview

Static `import` statements are fixed and must be at the top level. Dynamic `import()` allows loading modules conditionally or on-demand.

## Why Static Imports Are Limited

Static imports cannot:

- Use dynamic paths (must be string literals)
- Be placed conditionally (if blocks, loops, etc.)

```javascript
// Not allowed:
import ... from getModuleName();     // Error: path must be string literal

if (condition) {
  import ... from './module.js';    // Error: import must be top-level
}
```

## Dynamic `import()` Syntax

`import(module)` returns a promise that resolves to the module object.

```javascript
import(modulePath)
  .then((module) => {
    // Use module exports
  })
  .catch((err) => {
    // Handle loading error
  });
```

### With async/await

```javascript
let module = await import("./say.js");
```

## Basic Examples

### Module with Named Exports

```javascript
// 📁 say.js
export function hi() {
  alert("Hello");
}
export function bye() {
  alert("Bye");
}

// 📁 main.js
let { hi, bye } = await import("./say.js");
hi(); // Hello
bye(); // Bye
```

### Module with Default Export

```javascript
// 📁 say.js
export default function () {
  alert("Module loaded!");
}

// 📁 main.js
let obj = await import("./say.js");
let say = obj.default;
say(); // Module loaded!

// Or with destructuring:
let { default: say } = await import("./say.js");
say();
```

## Practical Usage

### Conditional Loading

```javascript
let modulePath = prompt("Which module to load?");
import(modulePath)
  .then((module) => {
    // Use dynamically loaded module
  })
  .catch((err) => console.error(err));
```

### On-Demand Loading (Button Click)

```html
<button onclick="load()">Load Module</button>

<script>
  async function load() {
    let module = await import("./heavy-module.js");
    module.init();
  }
</script>
```

## Important Notes

| Feature                  | Static `import`       | Dynamic `import()` |
| ------------------------ | --------------------- | ------------------ |
| Location                 | Top-level only        | Anywhere           |
| Path                     | String literal        | Any expression     |
| Returns                  | Nothing (synchronous) | Promise            |
| Requires `type="module"` | Yes                   | No                 |

**Key Points:**

- `import()` is not a function (it's a special syntax) – cannot be copied or used with `call/apply`
- Can be used in regular scripts (no need for `type="module"`)
- Great for code splitting, lazy loading, and conditional imports

## Summary

| Use Case                        | Solution                              |
| ------------------------------- | ------------------------------------- |
| Static, known imports           | `import ... from 'module'`            |
| Dynamic, conditional, on-demand | `await import(modulePath)`            |
| Code splitting (performance)    | Dynamic imports for non-critical code |

**Best Practice:** Use dynamic imports to load modules only when needed (e.g., on user interaction), reducing initial page load time.
