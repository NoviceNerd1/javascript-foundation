# Property Getters and Setters

## Overview

There are two types of object properties:

- **Data properties** – regular key-value pairs (what we've used so far)
- **Accessor properties** – functions that execute on getting/setting, but appear as regular properties

## Getters and Setters

```javascript
let obj = {
  get propName() {
    // runs when obj.propName is read
  },
  set propName(value) {
    // runs when obj.propName = value is assigned
  },
};
```

### Basic Example

```javascript
let user = {
  name: "John",
  surname: "Smith",

  get fullName() {
    return `${this.name} ${this.surname}`;
  },

  set fullName(value) {
    [this.name, this.surname] = value.split(" ");
  },
};

alert(user.fullName); // John Smith (getter runs)
user.fullName = "Alice Cooper"; // setter runs
alert(user.name); // Alice
alert(user.surname); // Cooper
```

## Accessor Descriptors

Accessor descriptors have different properties than data descriptors:

| Data Property  | Accessor Property |
| -------------- | ----------------- |
| `value`        | `get`             |
| `writable`     | `set`             |
| `enumerable`   | `enumerable`      |
| `configurable` | `configurable`    |

### Using `defineProperty` with Accessors

```javascript
let user = {
  name: "John",
  surname: "Smith",
};

Object.defineProperty(user, "fullName", {
  get() {
    return `${this.name} ${this.surname}`;
  },
  set(value) {
    [this.name, this.surname] = value.split(" ");
  },
  enumerable: true,
  configurable: true,
});

alert(user.fullName); // John Smith
```

**Important:** A property cannot be both a data property and an accessor property.

```javascript
// Error: Invalid property descriptor
Object.defineProperty({}, "prop", {
  get() {
    return 1;
  },
  value: 2,
});
```

## Smarter Getters/Setters – Validation

```javascript
let user = {
  get name() {
    return this._name;
  },

  set name(value) {
    if (value.length < 4) {
      alert("Name too short, need at least 4 characters");
      return;
    }
    this._name = value;
  },
};

user.name = "Pete"; // Works
user.name = "Tom"; // Alert: Name too short
```

> **Convention:** Properties starting with `_` (underscore) are considered internal and should not be accessed directly from outside.

## Using for Compatibility

Add accessors to existing code without breaking it:

```javascript
function User(name, birthday) {
  this.name = name;
  this.birthday = birthday;

  // Add age property without changing existing code
  Object.defineProperty(this, "age", {
    get() {
      let todayYear = new Date().getFullYear();
      return todayYear - this.birthday.getFullYear();
    },
  });
}

let john = new User("John", new Date(1992, 6, 1));
alert(john.birthday); // Works (original)
alert(john.age); // Calculated on demand
```

## Summary

| Feature         | Data Property       | Accessor Property       |
| --------------- | ------------------- | ----------------------- |
| Store value     | `value`             | `get`/`set` functions   |
| Descriptor keys | `value`, `writable` | `get`, `set`            |
| Can't have both | ❌                  | ❌ (mutually exclusive) |

**Key Points:**

- Getters/setters make properties look normal but add logic
- Use `_` convention for internal properties
- Accessors are great for validation, computed properties, and backward compatibility
- Define accessors in object literals with `get`/`set` or with `Object.defineProperty`
