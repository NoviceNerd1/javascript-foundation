# Data Types

JavaScript is **dynamically typed** - variables can hold any type.

```javascript
let message = "hello";
message = 123456; // no error, type changes
```

## 8 Basic Data Types

### 1. Number

Integer and floating point numbers. Also special values: `Infinity`, `-Infinity`, `NaN`.

```javascript
let n = 123;
n = 12.345;

alert(1 / 0); // Infinity
alert("not a number" / 2); // NaN

alert(NaN + 1); // NaN (sticky)
alert(NaN ** 0); // 1 (only exception)
```

**Safe integer range:** ±(2⁵³-1) (about 16 quadrillion). Math operations are safe - never fatal errors.

### 2. BigInt

Integers of arbitrary length. Created by appending `n` to an integer.

```javascript
const bigInt = 1234567890123456789012345678901234567890n;
```

### 3. String

Three types of quotes:

```javascript
let str = "Hello"; // double quotes
let str2 = "Single quotes"; // single quotes
let phrase = `can embed ${str}`; // backticks

let name = "John";
alert(`Hello, ${name}!`); // Hello, John!
alert(`result: ${1 + 2}`); // result: 3
alert("result: ${1 + 2}"); // result: ${1 + 2} (double quotes don't work)
```

**No character type** - only strings (empty, one character, or many).

### 4. Boolean

Only two values: `true` and `false`.

```javascript
let nameFieldChecked = true;
let isGreater = 4 > 1;
alert(isGreater); // true
```

### 5. `null`

Represents "nothing", "empty", or "value unknown".

```javascript
let age = null; // age is unknown
```

### 6. `undefined`

Means "value not assigned".

```javascript
let age;
alert(age); // undefined

// Can assign explicitly (not recommended)
age = undefined;
```

**Best practice:** Use `null` for "empty/unknown" values, `undefined` is the default for unassigned variables.

### 7. Symbol

Creates unique identifiers for objects.

### 8. Object

Stores collections of data and complex entities. All other types are "primitive" (single value only).

## The `typeof` Operator

Returns the type name as a string.

```javascript
typeof undefined; // "undefined"
typeof 0; // "number"
typeof 10n; // "bigint"
typeof true; // "boolean"
typeof "foo"; // "string"
typeof Symbol("id"); // "symbol"
typeof Math; // "object"
typeof null; // "object"  (official error!)
typeof alert; // "function"
```

**Note about `typeof null`:** Returns `"object"` - this is an officially recognized error kept for compatibility. `null` is NOT an object.

**Syntax:** `typeof x` or `typeof(x)` (both work, `typeof` is an operator, not a function).

## Summary

| Type        | Description                              |
| ----------- | ---------------------------------------- |
| `number`    | Integers and floats (safe ±2⁵³-1)        |
| `bigint`    | Arbitrary-length integers                |
| `string`    | Text (no separate char type)             |
| `boolean`   | `true` / `false`                         |
| `null`      | Unknown/empty value                      |
| `undefined` | Unassigned value                         |
| `symbol`    | Unique identifiers                       |
| `object`    | Collections/complex data (non-primitive) |
