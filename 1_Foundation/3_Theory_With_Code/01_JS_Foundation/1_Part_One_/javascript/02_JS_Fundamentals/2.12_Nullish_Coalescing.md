# Nullish Coalescing Operator `??`

## Overview

The nullish coalescing operator `??` returns the first argument if it's **defined** (not `null` or `undefined`), otherwise returns the second argument.

```javascript
result = a ?? b;
// Equivalent to:
result = a !== null && a !== undefined ? a : b;
```

## Basic Usage

### Provide Default Values

```javascript
let user;
alert(user ?? "Anonymous"); // Anonymous (user is undefined)

let user = "John";
alert(user ?? "Anonymous"); // John (user is defined)
```

### Chain Multiple Values

Returns the first defined value from a list:

```javascript
let firstName = null;
let lastName = null;
let nickName = "Supercoder";

alert(firstName ?? lastName ?? nickName ?? "Anonymous"); // Supercoder
```

## `??` vs `||`

Key difference: `||` returns first **truthy** value; `??` returns first **defined** value.

```javascript
let height = 0;

alert(height || 100); // 100 (0 is falsy)
alert(height ?? 100); // 0   (0 is defined, not null/undefined)
```

| Value              | `\|\|`               | `??`                 |
| ------------------ | -------------------- | -------------------- |
| `0`                | returns second value | returns `0`          |
| `""`               | returns second value | returns `""`         |
| `false`            | returns second value | returns `false`      |
| `null`/`undefined` | returns second value | returns second value |

## Precedence

- `??` has same precedence as `||` (MDN table level 3)
- Evaluated before `=` and `?`, but after most operations like `+`, `*`

```javascript
let height = null;
let width = null;

// Use parentheses for correct evaluation
let area = (height ?? 100) * (width ?? 50);
alert(area); // 5000

// Without parentheses - incorrect:
let area = height ?? 100 * width ?? 50;
// Evaluates as: height ?? (100 * width) ?? 50
```

## Restrictions

Cannot use `??` with `&&` or `||` without explicit parentheses:

```javascript
let x = 1 && 2 ?? 3;  // Syntax error

// Fix with parentheses:
let x = (1 && 2) ?? 3;  // Works
alert(x);  // 2
```

## Summary

- Returns first **defined** (non-null, non-undefined) value
- Perfect for providing defaults while preserving valid falsy values (`0`, `""`, `false`)
- Low precedence - use parentheses in complex expressions
- Cannot be mixed with `&&`/`||` without parentheses
