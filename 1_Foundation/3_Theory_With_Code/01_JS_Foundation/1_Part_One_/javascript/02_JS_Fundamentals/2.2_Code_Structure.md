# Code Structure

## Statements

Statements are syntax constructs and commands that perform actions.

```javascript
alert("Hello");
alert("World"); // Multiple statements on one line

alert("Hello"); // Usually on separate lines for readability
alert("World");
```

## Semicolons

A semicolon can be omitted in most cases when a line break exists (automatic semicolon insertion).

```javascript
// This works (line break implies semicolon)
alert("Hello");
alert("World");
```

**But "most cases" does not mean "always"!**

### Example where JavaScript fails:

```javascript
alert("Hello") // No semicolon
  [(1, 2)].forEach(alert); // Error! Only "Hello" shows
```

JavaScript does not assume a semicolon before square brackets `[...]`. The engine sees it as:

```javascript
alert("Hello")[(1, 2)].forEach(alert); // Wrong merging
```

### Example where newline does NOT mean semicolon:

```javascript
alert(3 + 1 + 2); // Outputs 6 (incomplete expression, semicolon would be incorrect)
```

**Recommendation:** Put semicolons between statements even if separated by newlines. Safer for beginners.

## Comments

Comments are ignored by the engine. They describe what code does and why.

### One-line comments (`//`)

```javascript
// This comment occupies a full line
alert("Hello");

alert("World"); // This comment follows a statement
```

### Multiline comments (`/* */`)

```javascript
/* An example with two messages.
This is a multiline comment.
*/
alert("Hello");
alert("World");
```

### Disabling code temporarily:

```javascript
/* Commenting out the code
alert('Hello');
*/
alert("World");
```

### Hotkeys (most editors):

- Single-line comment: `Ctrl+/` (`Cmd+/` on Mac)
- Multiline comment: `Ctrl+Shift+/` (`Cmd+Option+/` on Mac)

## Important Restriction

**Nested multiline comments are NOT supported:**

```javascript
/*
  /* nested comment ?!? */  // Error!
*/
alert('World');
```

## Summary

- **Statements** = commands that perform actions
- **Semicolons** = recommended between statements (even with line breaks)
- **Comments** = ignored by engine, use `//` for single line or `/* */` for multiline
- Comments don't affect production (minifiers remove them)
