# Modules: Introduction

## Overview

A module is simply a file. JavaScript modules use `export` and `import` keywords to share functionality between files. Modern modules are supported in all major browsers and Node.js.

## Basic Example

### Exporting from a Module

```javascript
// 📁 sayHi.js
export function sayHi(user) {
  alert(`Hello, ${user}!`);
}
```

### Importing into Another Module

```javascript
// 📁 main.js
import { sayHi } from "./sayHi.js";

sayHi("John"); // Hello, John!
```

### Using in HTML

```html
<script type="module">
  import { sayHi } from "./sayHi.js";
  sayHi("John");
</script>
```

> **Important:** Modules only work over HTTP(s), not from `file://` protocol. Use a local web server.

## Core Module Features

### 1. Always "use strict"

```html
<script type="module">
  a = 5; // Error: variable not defined
</script>
```

### 2. Module-Level Scope

Each module has its own top-level scope. Variables are not shared globally.

```html
<!-- Won't work - separate modules -->
<script type="module">
  let user = "John";
</script>
<script type="module">
  alert(user); // Error: user is not defined
</script>
```

### 3. Code Evaluated Once

A module's code executes only the first time it's imported. Exports are shared between all importers.

```javascript
// 📁 admin.js
export let admin = { name: "John" };

// 📁 1.js
import { admin } from "./admin.js";
admin.name = "Pete";

// 📁 2.js
import { admin } from "./admin.js";
alert(admin.name); // Pete (shared object)
```

### 4. Configuration Pattern

```javascript
// 📁 admin.js
export let config = {};

export function sayHi() {
  alert(`Ready to serve, ${config.user}!`);
}

// 📁 init.js (first script)
import { config } from "./admin.js";
config.user = "Pete";

// 📁 another.js
import { sayHi } from "./admin.js";
sayHi(); // Ready to serve, Pete!
```

### 5. `import.meta` – Module Info

```html
<script type="module">
  alert(import.meta.url); // Script URL
</script>
```

### 6. `this` is `undefined` at Top Level

```html
<script type="module">
  alert(this); // undefined
</script>
```

## Browser-Specific Features

### Deferred Execution

Module scripts are **deferred** – they wait for HTML to load before running.

```html
<script type="module">
  alert(typeof button); // object (can see elements below)
</script>

<button id="button">Button</button>
```

### `async` Works on Inline Modules

```html
<script async type="module">
  import { counter } from "./analytics.js";
  counter.count(); // Runs as soon as ready
</script>
```

### External Module Scripts

- Same `src` runs only once
- Cross-origin scripts require **CORS headers**

```html
<!-- Runs once -->
<script type="module" src="my.js"></script>
<script type="module" src="my.js"></script>

<!-- Requires CORS -->
<script type="module" src="http://another-site.com/their.js"></script>
```

### No Bare Modules

Paths must be relative or absolute URLs:

```javascript
import { sayHi } from "sayHi"; // Error!
import { sayHi } from "./sayHi.js"; // Correct
```

## Fallback for Old Browsers

```html
<script type="module">
  // Modern browsers
</script>

<script nomodule>
  // Old browsers only
</script>
```

## Build Tools (Production)

In production, tools like **Webpack** bundle modules into single files, allowing:

- Bare module resolution
- Tree-shaking (removing unused code)
- Minification
- Modern syntax transpilation (Babel)

## Summary

| Feature           | Description                               |
| ----------------- | ----------------------------------------- |
| `export`          | Exposes functionality from a module       |
| `import`          | Imports functionality from another module |
| `type="module"`   | Required in browser `<script>` tags       |
| Module scope      | Each module has its own top-level scope   |
| Single evaluation | Module code runs once; exports are shared |
| Deferred          | Modules wait for HTML to load             |
| CORS              | Cross-origin modules require CORS headers |

**Key Points:**

- Modules provide encapsulation and dependency management
- Use `export` and `import` to share code between files
- Module code runs in strict mode with its own scope
- Modules are executed only once, exports are shared
- Use bundlers (Webpack) for production deployment
