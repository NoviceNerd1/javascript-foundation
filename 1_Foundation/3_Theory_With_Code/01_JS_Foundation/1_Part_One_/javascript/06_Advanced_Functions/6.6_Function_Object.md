# Function Object, NFE

## Overview

In JavaScript, functions are **objects**. They can be called and also treated as regular objects (add properties, pass by reference, etc.).

## Function Properties

### `name` Property

Returns the function's name. JavaScript intelligently assigns names even from context.

```javascript
function sayHi() {}
alert(sayHi.name); // sayHi

let sayHi = function () {};
alert(sayHi.name); // sayHi (contextual name)

let user = {
  sayHi() {},
  sayBye: function () {},
};
alert(user.sayHi.name); // sayHi
alert(user.sayBye.name); // sayBye

// No name case
let arr = [function () {}];
alert(arr[0].name); // "" (empty string)
```

### `length` Property

Returns the number of function parameters. Rest parameters are **not** counted.

```javascript
function f1(a) {}
function f2(a, b) {}
function many(a, b, ...more) {}

alert(f1.length); // 1
alert(f2.length); // 2
alert(many.length); // 2 (rest not counted)
```

### Custom Properties

Add arbitrary properties to functions.

```javascript
function sayHi() {
  alert("Hi");
  sayHi.counter++;
}
sayHi.counter = 0;

sayHi(); // Hi
sayHi(); // Hi
alert(sayHi.counter); // 2
```

**Important:** Function properties are separate from local variables.

### Function Properties vs Closures

```javascript
// Using closure
function makeCounter() {
  let count = 0;
  return function () {
    return count++;
  };
}

// Using function property
function makeCounter() {
  function counter() {
    return counter.count++;
  }
  counter.count = 0;
  return counter;
}

let counter = makeCounter();
counter.count = 10; // External code can modify (unlike closure)
alert(counter()); // 10
```

## Named Function Expression (NFE)

A Function Expression with a **name** allows the function to reference itself internally.

```javascript
let sayHi = function func(who) {
  if (who) {
    alert(`Hello, ${who}`);
  } else {
    func("Guest"); // Internal reference (reliable)
  }
};

sayHi(); // Hello, Guest
func(); // Error: func is not defined (not visible outside)
```

### Why Not Just Use the Variable Name?

```javascript
let sayHi = function (who) {
  if (who) {
    alert(`Hello, ${who}`);
  } else {
    sayHi("Guest"); // Problem: sayHi might change
  }
};

let welcome = sayHi;
sayHi = null;
welcome(); // Error: sayHi is not a function
```

**With NFE, it works because `func` always references the function itself.**

```javascript
let sayHi = function func(who) {
  if (who) {
    alert(`Hello, ${who}`);
  } else {
    func("Guest"); // Always works
  }
};

let welcome = sayHi;
sayHi = null;
welcome(); // Hello, Guest (works!)
```

**Note:** NFE is only for Function Expressions, not Function Declarations.

## Summary

| Feature           | Description                                                           |
| ----------------- | --------------------------------------------------------------------- |
| `name` property   | Function name (contextually assigned)                                 |
| `length` property | Number of parameters (rest not counted)                               |
| Custom properties | Add arbitrary data to functions                                       |
| NFE               | Named Function Expression – internal name for reliable self-reference |

## Tasks

### 1. Set and Decrease for Counter

```javascript
function makeCounter() {
  let count = 0;

  function counter() {
    return count++;
  }

  counter.set = function (value) {
    count = value;
  };

  counter.decrease = function () {
    count--;
  };

  return counter;
}

let counter = makeCounter();
alert(counter()); // 0
alert(counter()); // 1
counter.set(10);
alert(counter()); // 10
counter.decrease();
alert(counter()); // 10
```

### 2. Sum with Arbitrary Brackets

```javascript
function sum(a) {
  let currentSum = a;

  function f(b) {
    currentSum += b;
    return f;
  }

  f.toString = function () {
    return currentSum;
  };

  return f;
}

alert(sum(1)(2)); // 3
alert(sum(1)(2)(3)); // 6
alert(sum(5)(-1)(2)); // 6
```
