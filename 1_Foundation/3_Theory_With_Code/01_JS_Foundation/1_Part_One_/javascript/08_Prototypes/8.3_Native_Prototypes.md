# Native Prototypes

## Overview

All built-in constructor functions (Object, Array, Function, etc.) use prototypes to store methods. This makes inheritance memory-efficient.

## Object.prototype

```javascript
let obj = {};
alert(obj); // "[object Object]" – where does this come from?

// obj = {} is same as new Object()
// Object.prototype provides toString and other methods

alert(obj.__proto__ === Object.prototype); // true
alert(obj.toString === Object.prototype.toString); // true

// Top of chain
alert(Object.prototype.__proto__); // null
```

## Other Built-in Prototypes

```javascript
let arr = [1, 2, 3];

alert(arr.__proto__ === Array.prototype); // true
alert(arr.__proto__.__proto__ === Object.prototype); // true
alert(arr.__proto__.__proto__.__proto__); // null

// Array has its own toString
alert(arr); // "1,2,3" (Array.prototype.toString)
```

### Prototype Chain Structure

```
null
 ↑
Object.prototype
 ↑
Array.prototype / Function.prototype / etc.
 ↑
Array / Function instances
```

## Primitives and Wrappers

Strings, numbers, booleans have temporary wrapper objects with methods in `String.prototype`, `Number.prototype`, `Boolean.prototype`.

```javascript
let str = "Hello";
alert(str.toUpperCase()); // Temporary String object created

// null and undefined have no wrappers
alert(null.toString()); // Error!
```

## Modifying Native Prototypes

**Generally discouraged** – can cause conflicts.

### Allowed Use: Polyfills

```javascript
if (!String.prototype.repeat) {
  String.prototype.repeat = function (n) {
    return new Array(n + 1).join(this);
  };
}

alert("La".repeat(3)); // "LaLaLa"
```

## Borrowing from Prototypes

Take methods from one object and use them on another:

```javascript
let obj = {
  0: "Hello",
  1: "world!",
  length: 2,
};

obj.join = Array.prototype.join;
alert(obj.join(",")); // "Hello,world!"
```

## Summary

| Built-in  | Prototype                                 | Top of Chain |
| --------- | ----------------------------------------- | ------------ |
| Objects   | `Object.prototype`                        | `null`       |
| Arrays    | `Array.prototype` → `Object.prototype`    | `null`       |
| Functions | `Function.prototype` → `Object.prototype` | `null`       |
| Strings   | `String.prototype` → `Object.prototype`   | `null`       |

**Key Points:**

- All built-ins inherit from `Object.prototype` (except `null`)
- Primitives use temporary wrapper objects
- Modifying native prototypes is risky (except polyfills)
- Method borrowing allows using array methods on array-like objects

## Tasks

### 1. Add `f.defer(ms)` – Run Function After Delay

```javascript
Function.prototype.defer = function (ms) {
  setTimeout(this, ms);
};

function f() {
  alert("Hello!");
}

f.defer(1000); // Shows "Hello!" after 1 second
```

### 2. Add `f.defer(ms)` – Return Wrapper with Arguments

```javascript
Function.prototype.defer = function (ms) {
  let func = this;
  return function (...args) {
    setTimeout(() => func.apply(this, args), ms);
  };
};

function f(a, b) {
  alert(a + b);
}

f.defer(1000)(1, 2); // Shows 3 after 1 second
```
