# Async/Await

## Overview

`async/await` is a special syntax that makes working with promises more comfortable and readable.

## Async Functions

The `async` keyword before a function ensures it **always returns a promise**. Non-promise values are automatically wrapped.

```javascript
async function f() {
  return 1;
}

f().then(alert); // 1

// Equivalent to:
async function f() {
  return Promise.resolve(1);
}
```

## Await

The `await` keyword makes JavaScript wait until a promise settles and returns its result. It only works inside `async` functions.

```javascript
async function f() {
  let promise = new Promise((resolve, reject) => {
    setTimeout(() => resolve("done!"), 1000);
  });

  let result = await promise; // Waits 1 second
  alert(result); // "done!"
}

f();
```

**Important:** `await` suspends function execution, but doesn't block CPU. The engine can do other tasks meanwhile.

## Error Handling with try..catch

If a promise rejects, `await` throws the error – can be caught with `try..catch`:

```javascript
async function f() {
  try {
    let response = await fetch("http://no-such-url");
    let user = await response.json();
  } catch (err) {
    alert(err); // Handles fetch and json errors
  }
}

f();
```

Without `try..catch`, the async function returns a rejected promise:

```javascript
async function f() {
  let response = await fetch("http://no-such-url");
}

f().catch(alert); // Handle error with .catch
```

## Practical Example: showAvatar

```javascript
async function showAvatar() {
  // Read JSON
  let response = await fetch("/article/promise-chaining/user.json");
  let user = await response.json();

  // Read GitHub user
  let githubResponse = await fetch(`https://api.github.com/users/${user.name}`);
  let githubUser = await githubResponse.json();

  // Show avatar
  let img = document.createElement("img");
  img.src = githubUser.avatar_url;
  document.body.append(img);

  // Wait 3 seconds
  await new Promise((resolve) => setTimeout(resolve, 3000));

  img.remove();
  return githubUser;
}

showAvatar();
```

## Top-Level Await

In modules, `await` can be used at top level:

```javascript
// In a module
let response = await fetch("/article/promise-chaining/user.json");
let user = await response.json();
console.log(user);
```

For non-module environments, wrap in an async function:

```javascript
(async () => {
  let response = await fetch("/article/promise-chaining/user.json");
  let user = await response.json();
  // ...
})();
```

## await with Thenables

`await` accepts any "thenable" object (objects with a `.then` method):

```javascript
class Thenable {
  constructor(num) {
    this.num = num;
  }
  then(resolve, reject) {
    setTimeout(() => resolve(this.num * 2), 1000);
  }
}

async function f() {
  let result = await new Thenable(1);
  alert(result); // 2 after 1 second
}

f();
```

## Async Class Methods

```javascript
class Waiter {
  async wait() {
    return await Promise.resolve(1);
  }
}

new Waiter().wait().then(alert); // 1
```

## async/await with Promise.all

```javascript
let results = await Promise.all([fetch(url1), fetch(url2), fetch(url3)]);
```

## Summary

| Keyword | Purpose                                                                |
| ------- | ---------------------------------------------------------------------- |
| `async` | Makes function return a promise; enables `await` inside                |
| `await` | Pauses execution until promise settles; returns result or throws error |

**Key Points:**

- `async` functions always return promises
- `await` only works inside `async` functions (except modules)
- Use `try..catch` for error handling instead of `.catch`
- Combine with `Promise.all` for parallel operations

## Tasks

### 1. Rewrite Using async/await

```javascript
async function loadJson(url) {
  let response = await fetch(url);

  if (response.status == 200) {
    return response.json();
  } else {
    throw new Error(response.status);
  }
}

loadJson("https://javascript.info/no-such-user.json").catch(alert); // Error: 404
```

### 2. Rewrite "rethrow" with async/await

```javascript
class HttpError extends Error {
  constructor(response) {
    super(`${response.status} for ${response.url}`);
    this.name = "HttpError";
    this.response = response;
  }
}

async function loadJson(url) {
  let response = await fetch(url);

  if (response.status == 200) {
    return response.json();
  } else {
    throw new HttpError(response);
  }
}

async function demoGithubUser() {
  let user;
  while (true) {
    let name = prompt("Enter a name?", "iliakan");

    try {
      user = await loadJson(`https://api.github.com/users/${name}`);
      break;
    } catch (err) {
      if (err instanceof HttpError && err.response.status == 404) {
        alert("No such user, please reenter.");
      } else {
        throw err;
      }
    }
  }

  alert(`Full name: ${user.name}.`);
  return user;
}

demoGithubUser();
```

### 3. Call Async from Non-Async

```javascript
async function wait() {
  await new Promise((resolve) => setTimeout(resolve, 1000));
  return 10;
}

function f() {
  wait().then((result) => alert(result));
  // or: wait().then(alert);
}

f(); // Shows 10 after 1 second
```

### 4. Dangerous Promise.all

```javascript
// Problem: If Promise.all rejects early, disconnect() never runs!
// Other queries continue but results ignored.

// Fix: Use Promise.allSettled to ensure all settle before disconnecting
async function run() {
  connect();

  try {
    let results = await Promise.allSettled([
      delay(() => database.query(true), 100),
      delay(() => database.query(false), 200),
      delay(() => database.query(false), 300),
    ]);

    // Check results
    results.forEach((result) => {
      if (result.status === "rejected") {
        console.log("Query failed:", result.reason);
      }
    });
  } finally {
    disconnect(); // Always runs
  }
}
```
