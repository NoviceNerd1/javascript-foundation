# Code Editors

Programmers spend most of their time in code editors. There are two main types: **IDEs** and **lightweight editors**.

## IDE (Integrated Development Environment)

A powerful editor that operates on a "whole project" - not just an editor, but a full-scale development environment.

**Features:**

- Loads entire projects (multiple files)
- Navigation between files
- Autocompletion based on whole project
- Version management integration (git)
- Testing environment integration

**Options:**

| Editor                      | Platform       | Price                                   |
| --------------------------- | -------------- | --------------------------------------- |
| Visual Studio Code          | Cross-platform | Free                                    |
| WebStorm                    | Cross-platform | Paid                                    |
| Visual Studio (not VS Code) | Windows only   | Paid (Free Community version available) |

> **Note:** Visual Studio (Windows-only, paid) is different from Visual Studio Code. Visual Studio is well-suited for .NET platform and also good at JavaScript.

Many IDEs have trial periods. Their cost is negligible compared to a qualified developer's salary.

## Lightweight Editors

Faster, elegant, and simpler than IDEs. Mainly used to open and edit a file instantly.

**Key difference:** IDEs work on project-level (loads more data); lightweight editors are much faster for single files.

**Options:**

| Editor       | Platform       | Price     |
| ------------ | -------------- | --------- |
| Sublime Text | Cross-platform | Shareware |
| Notepad++    | Windows        | Free      |
| Vim          | Cross-platform | Free      |
| Emacs        | Cross-platform | Free      |

> **Note:** Lightweight editors can have plugins (directory-level syntax analyzers, autocompleters), so the border between lightweight editors and IDEs is not strict.

## Author's Personal Opinion

- **Mostly frontend development** → Use Visual Studio Code
- **Mostly another language/platform + partially frontend** → Consider:
  - XCode (Mac)
  - Visual Studio (Windows)
  - Jetbrains family (WebStorm, PHPStorm, RubyMine, etc., depending on language)

## Summary

The choice of an editor is **individual** - depends on your projects, habits, and personal preferences. Choose the one you like most.
