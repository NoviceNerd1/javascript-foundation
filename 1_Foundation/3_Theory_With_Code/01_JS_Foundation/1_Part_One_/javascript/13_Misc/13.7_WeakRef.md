# WeakRef and FinalizationRegistry

## Overview

**WeakRef** and **FinalizationRegistry** are advanced JavaScript features for managing memory and cleanup. They are rarely needed in everyday development.

## Strong vs Weak References

- **Strong reference** – Prevents garbage collection (normal object references)
- **Weak reference** – Does NOT prevent garbage collection

```javascript
// Strong reference
let user = { name: "John" }; // Object stays in memory

// Weak reference
let admin = new WeakRef(user); // Doesn't prevent garbage collection
```

## WeakRef

Creates a weak reference to an object. Does not keep the object alive.

```javascript
let user = { name: "John" };
let weakRef = new WeakRef(user);

// Later, after strong references are gone
user = null;

// Try to get the object (may be undefined if collected)
let ref = weakRef.deref();
if (ref) {
  // Object still exists
} else {
  // Object was garbage collected
}
```

### WeakRef Use Cases

- Caching large objects without preventing GC
- Associating data with DOM elements

### Example: Weak Cache

```javascript
function weakRefCache(fetchImg) {
  const imgCache = new Map();

  return (imgName) => {
    const cachedImg = imgCache.get(imgName);

    if (cachedImg?.deref()) {
      return cachedImg.deref(); // Return cached image
    }

    const newImg = fetchImg(imgName);
    imgCache.set(imgName, new WeakRef(newImg));
    return newImg;
  };
}
```

## FinalizationRegistry

Registers cleanup callbacks that run when objects are garbage collected.

```javascript
const registry = new FinalizationRegistry((heldValue) => {
  console.log(`${heldValue} was collected`);
});

let user = { name: "John" };
registry.register(user, user.name);

// Later, when user is garbage collected, callback runs
user = null;
```

### Registry Methods

| Method                                           | Description                      |
| ------------------------------------------------ | -------------------------------- |
| `register(target, heldValue, [unregisterToken])` | Register object for cleanup      |
| `unregister(unregisterToken)`                    | Unregister object before cleanup |

### Example: Cache with Cleanup

```javascript
function weakRefCache(fetchImg) {
  const imgCache = new Map();

  const registry = new FinalizationRegistry((imgName) => {
    const cachedImg = imgCache.get(imgName);
    // Remove only if value was actually collected
    if (cachedImg && !cachedImg.deref()) {
      imgCache.delete(imgName);
    }
  });

  return (imgName) => {
    const cachedImg = imgCache.get(imgName);

    if (cachedImg?.deref()) {
      return cachedImg.deref();
    }

    const newImg = fetchImg(imgName);
    imgCache.set(imgName, new WeakRef(newImg));
    registry.register(newImg, imgName);
    return newImg;
  };
}
```

## Important Notes

1. **Not guaranteed** – Cleanup callbacks may never run (e.g., tab closed, registry unreachable)
2. **Timing unpredictable** – No guarantee when GC runs or callbacks execute
3. **Complexity** – Adds significant complexity; avoid unless absolutely necessary
4. **Alternatives** – Most use cases can be solved with `Map`/`Set`/`WeakMap`

## When to Use

- Caching resource-intensive objects that should not prevent GC
- Cleaning up external resources when objects are collected
- **Rarely needed** – Consider carefully before using

## Summary

| Feature                | Purpose                                        |
| ---------------------- | ---------------------------------------------- |
| `WeakRef`              | Create weak reference (doesn't prevent GC)     |
| `FinalizationRegistry` | Run cleanup code when objects are collected    |
| `deref()`              | Get referenced object (may return `undefined`) |

**Key Points:**

- Use only when necessary – adds significant complexity
- Cannot rely on cleanup callbacks running predictably
- Consider `WeakMap`/`WeakSet` as simpler alternatives
- Primarily useful for large caches and resource management
