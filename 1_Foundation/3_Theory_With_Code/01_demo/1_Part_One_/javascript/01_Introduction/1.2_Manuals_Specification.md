# Manuals and Specifications

## Specification (ECMA-262)

The most in-depth, formalized definition of JavaScript. A new version is released yearly.

**Use when:** You need the most trustworthy source of language details.

**Resources:**

- Latest draft: https://tc39.es/ecma262/
- New features ("stage 3" proposals): https://github.com/tc39/proposals

> **Note:** Being highly formalized, it's difficult to understand at first. Not for everyday use.

## Manuals

### MDN JavaScript Reference

The main manual with examples and in-depth information about language functions and methods.

**URL:** https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference

**Best way to use:** Search `MDN [term]` on Google, e.g., `MDN parseInt`

## Compatibility Tables

Check which JavaScript features are supported across different browsers and engines:

| Resource                              | Purpose                                                   |
| ------------------------------------- | --------------------------------------------------------- |
| https://caniuse.com                   | Per-feature support tables (e.g., cryptography functions) |
| https://kangax.github.io/compat-table | Language feature support across engines                   |

## Summary

- **Specification** = Most formal/trustworthy, but difficult for beginners
- **MDN** = Best everyday manual with examples
- **Compatibility tables** = Check which features work in which browsers

> Remember these resources for real-life development when you need in-depth information about specific features.
