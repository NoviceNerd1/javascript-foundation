# Variable Scope, Closure

## Code Blocks and Variable Scope

Variables declared with `let` or `const` inside a block `{...}` are only visible inside that block.

```javascript
{
  let message = "Hello";
  alert(message); // Hello
}
alert(message); // Error: message is not defined
```

### Block Scope in Loops and Conditionals

```javascript
if (true) {
  let phrase = "Hello!";
  alert(phrase); // Hello!
}
alert(phrase); // Error

for (let i = 0; i < 3; i++) {
  alert(i); // 0, 1, 2
}
alert(i); // Error
```

## Nested Functions

A function created inside another function can access outer variables.

```javascript
function sayHiBye(firstName, lastName) {
  function getFullName() {
    return firstName + " " + lastName; // Accesses outer variables
  }
  alert("Hello, " + getFullName());
  alert("Bye, " + getFullName());
}
```

## Lexical Environment (Internal Mechanism)

Every running function, block `{...}`, and the script has a hidden **Lexical Environment**:

- **Environment Record**: Stores local variables
- **Reference to outer** Lexical Environment

### Variable Lookup

When code accesses a variable, the inner Lexical Environment is searched first, then outer, then more outer, up to global.

## Closures

A **closure** is a function that remembers its outer variables and can access them.

```javascript
function makeCounter() {
  let count = 0;

  return function () {
    return count++; // Remembers count from outer scope
  };
}

let counter = makeCounter();
alert(counter()); // 0
alert(counter()); // 1
alert(counter()); // 2
```

### How It Works

- All functions have a hidden `[[Environment]]` property referencing the Lexical Environment where they were created
- When called, a new Lexical Environment is created with outer reference from `[[Environment]]`

## Garbage Collection

Lexical Environments are kept in memory while reachable (e.g., a nested function referencing them).

```javascript
function f() {
  let value = 123;
  return function () {
    alert(value);
  };
}

let g = f(); // value stays in memory while g exists
g = null; // Now memory is cleaned
```

## Tasks

### 1. Does Function Pick Up Latest Changes?

```javascript
let name = "John";
function sayHi() {
  alert("Hi, " + name);
}
name = "Pete";
sayHi(); // "Hi, Pete" (uses current value, not creation-time value)
```

### 2. Which Variables Are Available?

```javascript
function makeWorker() {
  let name = "Pete";
  return function () {
    alert(name);
  };
}
let name = "John";
let work = makeWorker();
work(); // "Pete" (uses variable from creation place)
```

### 3. Are Counters Independent?

```javascript
function makeCounter() {
  let count = 0;
  return function () {
    return count++;
  };
}

let counter = makeCounter();
let counter2 = makeCounter();
alert(counter()); // 0
alert(counter()); // 1
alert(counter2()); // 0 (independent)
alert(counter2()); // 1
```

### 4. Counter Object

```javascript
function Counter() {
  let count = 0;
  this.up = function () {
    return ++count;
  };
  this.down = function () {
    return --count;
  };
}
let counter = new Counter();
alert(counter.up()); // 1
alert(counter.up()); // 2
alert(counter.down()); // 1 (both methods share same count)
```

### 5. Function in if

```javascript
let phrase = "Hello";
if (true) {
  let user = "John";
  function sayHi() {
    alert(`${phrase}, ${user}`);
  }
}
sayHi(); // Error (function declared inside block is not visible outside)
```

### 6. Sum with Closures

```javascript
function sum(a) {
  return function (b) {
    return a + b;
  };
}
alert(sum(1)(2)); // 3
alert(sum(5)(-1)); // 4
```

### 7. Is Variable Visible?

```javascript
let x = 1;
function func() {
  console.log(x); // ReferenceError: Cannot access 'x' before initialization
  let x = 2;
}
func();
// Temporal Dead Zone: variable exists but not initialized until declaration
```

### 8. Filter Through Function

```javascript
function inBetween(a, b) {
  return function (x) {
    return x >= a && x <= b;
  };
}

function inArray(arr) {
  return function (x) {
    return arr.includes(x);
  };
}

let arr = [1, 2, 3, 4, 5, 6, 7];
alert(arr.filter(inBetween(3, 6))); // 3,4,5,6
alert(arr.filter(inArray([1, 2, 10]))); // 1,2
```

### 9. Sort by Field

```javascript
function byField(field) {
  return (a, b) => (a[field] > b[field] ? 1 : -1);
}

let users = [
  { name: "John", age: 20, surname: "Johnson" },
  { name: "Pete", age: 18, surname: "Peterson" },
  { name: "Ann", age: 19, surname: "Hathaway" },
];

users.sort(byField("name"));
users.sort(byField("age"));
```

### 10. Army of Functions

```javascript
// Problem: all shooters reference the same i variable
function makeArmy() {
  let shooters = [];
  for (let i = 0; i < 10; i++) {
    // Use let for block scope
    let shooter = function () {
      alert(i);
    };
    shooters.push(shooter);
  }
  return shooters;
}
// Or using IIFE:
function makeArmy() {
  let shooters = [];
  for (var i = 0; i < 10; i++) {
    (function (j) {
      shooters.push(function () {
        alert(j);
      });
    })(i);
  }
  return shooters;
}

let army = makeArmy();
army[0](); // 0
army[5](); // 5
```

## Summary

| Concept                 | Description                                                       |
| ----------------------- | ----------------------------------------------------------------- |
| **Block scope**         | `let`/`const` variables visible only inside `{...}`               |
| **Nested function**     | Function created inside another function                          |
| **Lexical Environment** | Internal object storing variables + outer reference               |
| **Closure**             | Function that remembers outer variables                           |
| **`[[Environment]]`**   | Hidden property storing reference to creation Lexical Environment |

**Key Points:**

- Functions remember where they were created, not where called
- Each function call creates its own Lexical Environment
- Closures allow functions to access outer variables even after outer function completes
- Use `let` in loops to create separate variables for each iteration
