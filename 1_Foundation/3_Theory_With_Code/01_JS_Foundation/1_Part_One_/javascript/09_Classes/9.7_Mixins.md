# Mixins

## Overview

Mixins provide a way to add functionality to classes without using inheritance. Since JavaScript classes can only inherit from one class, mixins allow adding multiple behaviors by copying methods.

## Basic Mixin

```javascript
let sayHiMixin = {
  sayHi() {
    alert(`Hello ${this.name}`);
  },
  sayBye() {
    alert(`Bye ${this.name}`);
  },
};

class User {
  constructor(name) {
    this.name = name;
  }
}

// Copy methods to the class prototype
Object.assign(User.prototype, sayHiMixin);

new User("Dude").sayHi(); // Hello Dude
```

### Mixin with Inheritance Inside

Mixins can have their own inheritance chain using `__proto__`:

```javascript
let sayMixin = {
  say(phrase) {
    alert(phrase);
  },
};

let sayHiMixin = {
  __proto__: sayMixin,

  sayHi() {
    super.say(`Hello ${this.name}`);
  },
  sayBye() {
    super.say(`Bye ${this.name}`);
  },
};

class User {
  constructor(name) {
    this.name = name;
  }
}

Object.assign(User.prototype, sayHiMixin);

new User("Dude").sayHi(); // Hello Dude
```

**Important:** `super` in mixin methods looks up the mixin's prototype chain (`sayMixin`), not the class's prototype.

## EventMixin Example

A practical mixin that adds event handling to any class:

```javascript
let eventMixin = {
  on(eventName, handler) {
    if (!this._eventHandlers) this._eventHandlers = {};
    if (!this._eventHandlers[eventName]) {
      this._eventHandlers[eventName] = [];
    }
    this._eventHandlers[eventName].push(handler);
  },

  off(eventName, handler) {
    let handlers = this._eventHandlers?.[eventName];
    if (!handlers) return;
    for (let i = 0; i < handlers.length; i++) {
      if (handlers[i] === handler) {
        handlers.splice(i--, 1);
      }
    }
  },

  trigger(eventName, ...args) {
    if (!this._eventHandlers?.[eventName]) return;
    this._eventHandlers[eventName].forEach((handler) =>
      handler.apply(this, args),
    );
  },
};
```

### Using EventMixin

```javascript
class Menu {
  choose(value) {
    this.trigger("select", value);
  }
}

// Add event methods
Object.assign(Menu.prototype, eventMixin);

let menu = new Menu();

// Subscribe to event
menu.on("select", (value) => alert(`Selected: ${value}`));

// Trigger event
menu.choose("123"); // Alert: Selected: 123
```

## Summary

| Concept            | Description                                                |
| ------------------ | ---------------------------------------------------------- |
| **Mixin**          | Object with methods that can be added to other classes     |
| **Implementation** | Copy methods using `Object.assign(Class.prototype, mixin)` |
| **Benefits**       | Add multiple behaviors without multiple inheritance        |
| **Caveat**         | Can overwrite existing methods (naming conflicts)          |

**Key Points:**

- Mixins allow adding functionality to classes without inheritance
- Implemented by copying methods to the class prototype
- Mixins can have their own internal inheritance using `__proto__`
- Useful for cross-cutting concerns like event handling
- Be careful with method names to avoid conflicts
