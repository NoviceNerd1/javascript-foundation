# Basic Operators, Maths

## Terminology

- **Operand** - what operators are applied to (e.g., in `5 * 2`, both `5` and `2` are operands)
- **Unary operator** - has single operand (e.g., unary negation: `-x`)
- **Binary operator** - has two operands (e.g., binary subtraction: `y - x`)

```javascript
let x = 1;
x = -x; // unary negation, x = -1

let y = 3;
alert(y - x); // binary subtraction, 3 - (-1) = 4
```

## Basic Math Operations

| Operator | Name           | Example       |
| -------- | -------------- | ------------- |
| `+`      | Addition       | `5 + 2 = 7`   |
| `-`      | Subtraction    | `5 - 2 = 3`   |
| `*`      | Multiplication | `5 * 2 = 10`  |
| `/`      | Division       | `5 / 2 = 2.5` |
| `%`      | Remainder      | `5 % 2 = 1`   |
| `**`     | Exponentiation | `2 ** 3 = 8`  |

### Remainder (`%`)

```javascript
alert(5 % 2); // 1 (remainder of 5 divided by 2)
alert(8 % 3); // 2
alert(8 % 4); // 0
```

### Exponentiation (`**`)

```javascript
alert(2 ** 2); // 4
alert(2 ** 3); // 8
alert(4 ** (1 / 2)); // 2 (square root)
alert(8 ** (1 / 3)); // 2 (cubic root)
```

## String Concatenation with Binary `+`

If any operand is a string, the other is converted to string and they are concatenated.

```javascript
alert("1" + 2); // "12"
alert(2 + "1"); // "21"
alert(2 + 2 + "1"); // "41" (2+2=4, then 4+'1'="41")
alert("1" + 2 + 2); // "122" ('1'+2="12", "12"+2="122")
```

**Other arithmetic operators** (`-`, `*`, `/`) only work with numbers and convert operands to numbers:

```javascript
alert(6 - "2"); // 4 (converts '2' to number)
alert("6" / "2"); // 3 (converts both)
```

## Unary Plus (`+`)

Converts non-numbers to numbers (same as `Number()` but shorter).

```javascript
// No effect on numbers
let x = 1;
alert(+x); // 1

// Converts non-numbers
alert(+true); // 1
alert(+""); // 0

// Converting strings to numbers before summing
let apples = "2";
let oranges = "3";
alert(+apples + +oranges); // 5
```

## Operator Precedence

Operators have precedence numbers (higher = executes first). Unary operators have higher precedence than binary ones.

| Precedence | Name           | Sign |
| ---------- | -------------- | ---- |
| 14         | unary plus     | `+`  |
| 14         | unary negation | `-`  |
| 13         | exponentiation | `**` |
| 12         | multiplication | `*`  |
| 12         | division       | `/`  |
| 11         | addition       | `+`  |
| 11         | subtraction    | `-`  |
| 2          | assignment     | `=`  |

Parentheses override precedence: `(1 + 2) * 2`

## Assignment (`=`)

Assignment is an operator with very low precedence (2). It returns the assigned value.

```javascript
let a = 1;
let b = 2;
let c = 3 - (a = b + 1); // (a = 3) returns 3, then c = 3 - 3 = 0
alert(a); // 3
alert(c); // 0
```

### Chaining Assignments (right-to-left)

```javascript
let a, b, c;
a = b = c = 2 + 2; // all become 4

// Better for readability:
c = 2 + 2;
b = c;
a = c;
```

## Modify-in-Place

```javascript
let n = 2;
n += 5; // n = 7 (same as n = n + 5)
n *= 2; // n = 14
n *= 3 + 5; // n = n * 8 (right side evaluated first)
alert(n); // 16
```

Operators: `+=`, `-=`, `*=`, `/=`, etc.

## Increment/Decrement (`++` / `--`)

Increase or decrease a variable by 1. Can only be applied to variables.

```javascript
let counter = 2;
counter++; // postfix: counter becomes 3
++counter; // prefix: counter becomes 4
```

### Prefix vs Postfix (difference in return value)

| Form                  | Returns                      |
| --------------------- | ---------------------------- |
| `++counter` (prefix)  | New value (after increment)  |
| `counter++` (postfix) | Old value (before increment) |

```javascript
let counter = 1;
let a = ++counter; // a = 2, counter = 2

let counter = 1;
let b = counter++; // b = 1, counter = 2

// When result not used, no difference
let counter = 0;
counter++;
++counter;
alert(counter); // 2
```

### In expressions (higher precedence than most operators)

```javascript
let counter = 1;
alert(2 * ++counter); // 4 (counter becomes 2, then 2*2)
alert(2 * counter++); // 2 (2*1, then counter becomes 2)
```

**Best practice:** One line - one action. Don't combine increment with other operations in complex expressions.

## Bitwise Operators (rarely used)

Treat arguments as 32-bit integers: `&` (AND), `|` (OR), `^` (XOR), `~` (NOT), `<<` (LEFT SHIFT), `>>` (RIGHT SHIFT), `>>>` (ZERO-FILL RIGHT SHIFT)

## Comma Operator (`,`)

Evaluates several expressions, returns only the last result. Very low precedence (lower than `=`).

```javascript
let a = (1 + 2, 3 + 4);  // a = 7 (1+2 thrown away, 3+4 returned)

// Without parentheses: a = 1 + 2, 3 + 4 → (a = 3), 7 (ignored)

// Used in complex constructs (e.g., for loops)
for (a = 1, b = 3, c = a * b; a < 10; a++) { ... }
```
