# Eval: Run a Code String

## Overview

`eval()` executes a string of JavaScript code. While powerful, it's rarely used in modern JavaScript and generally discouraged.

## Basic Syntax

```javascript
let result = eval(code);
```

### Examples

```javascript
eval('alert("Hello")'); // Hello

let value = eval("1+1");
alert(value); // 2

let value = eval("let i = 0; ++i");
alert(value); // 1
```

## Access to Outer Variables

`eval` can see and modify outer variables:

```javascript
let a = 1;

function f() {
  let a = 2;
  eval("alert(a)"); // 2 (sees local variable)
}

f();

let x = 5;
eval("x = 10");
alert(x); // 10 (modified)
```

### Strict Mode Behavior

In strict mode, `eval` has its own lexical environment:

```javascript
"use strict";

eval("let x = 5; function f() {}");
alert(typeof x); // undefined (not visible outside)
```

## Why `eval` Is Discouraged

1. **Security** – Executing arbitrary code strings can be dangerous
2. **Performance** – Minifiers can't rename variables used by `eval`
3. **Maintainability** – Harder to understand and debug
4. **Alternatives exist** – Modern JavaScript provides better solutions

## Safe Alternatives

### 1. Use `window.eval()` for Global Scope

```javascript
let x = 1;
{
  let x = 5;
  window.eval("alert(x)"); // 1 (global variable)
}
```

### 2. Use `new Function()` with Parameters

```javascript
// Instead of eval with local variables:
let localVar = 10;
// eval('alert(localVar)');  // Bad

// Use new Function (runs in global scope, pass data as arguments)
let f = new Function("a", "alert(a)");
f(5); // 5
```

## Summary

| Method                     | Scope                       | Use Case                        |
| -------------------------- | --------------------------- | ------------------------------- |
| `eval(code)`               | Current lexical environment | **Avoid**                       |
| `window.eval(code)`        | Global scope                | When global execution is needed |
| `new Function(args, body)` | Global scope                | Pass data as arguments          |

**Key Points:**

- `eval` executes code strings but is rarely needed in modern JS
- Prefer `new Function` or direct code over `eval`
- In strict mode, `eval` has its own scope (variables not visible outside)
- Consider security implications when using `eval`

## Task: Eval-Calculator

```javascript
function calculator() {
  let expr = prompt("Enter an arithmetic expression:", "2+2");
  alert(eval(expr));
}

calculator(); // Shows result of entered expression
```
