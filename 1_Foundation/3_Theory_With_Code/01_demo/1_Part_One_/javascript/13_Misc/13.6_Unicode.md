# Unicode, String Internals

## Overview

JavaScript strings are based on Unicode. Characters can be represented using different notations and may use 1-4 bytes.

## Unicode Notations

### 1. `\xXX` – 2 Hex Digits (0-255)

```javascript
alert("\x7A"); // z (U+007A)
alert("\xA9"); // © (copyright)
```

### 2. `\uXXXX` – 4 Hex Digits (0000-FFFF)

```javascript
alert("\u00A9"); // ©
alert("\u044F"); // я (Cyrillic)
alert("\u2191"); // ↑ (arrow up)
```

### 3. `\u{X…XXXXXX}` – 1-6 Hex Digits (0-10FFFF)

```javascript
alert("\u{20331}"); // 佫 (rare Chinese character)
alert("\u{1F60D}"); // 😍 (smiling face)
```

## Surrogate Pairs

Rare symbols (beyond U+FFFF) use **surrogate pairs** – two 2-byte characters. This affects string length and character access.

```javascript
alert("𝒳".length); // 2 (MATHEMATICAL SCRIPT CAPITAL X)
alert("😂".length); // 2 (FACE WITH TEARS OF JOY)
alert("𩷶".length); // 2 (rare Chinese character)

// Accessing pieces gives garbage
alert("𝒳"[0]); // Strange symbol (first half)
alert("𝒳"[1]); // Strange symbol (second half)
```

### Detecting Surrogate Pairs

```javascript
// First part: 0xd800..0xdbff
// Second part: 0xdc00..0xdfff

// charCodeAt (not surrogate-aware)
alert("𝒳".charCodeAt(0).toString(16)); // d835 (first part)
alert("𝒳".charCodeAt(1).toString(16)); // dcb3 (second part)

// codePointAt (surrogate-aware)
alert("𝒳".codePointAt(0).toString(16)); // 1d4b3 (full character)
```

### Safe String Splitting

```javascript
// Danger: splitting at arbitrary position can break surrogate pairs
alert("hi 😂".slice(0, 4)); // "hi [?]" (garbage)

// Use spread operator or Array.from for safe iteration
let chars = [..."hi 😂"];
alert(chars.slice(0, 4).join("")); // "hi 😂" (works!)
```

## Diacritical Marks and Normalization

Characters can be composed of a base character plus mark characters.

```javascript
// Base + dot above
alert("S\u0307"); // Ṡ

// Base + dot above + dot below
alert("S\u0307\u0323"); // Ṩ

// Different order – visually same, but not equal
let s1 = "S\u0307\u0323"; // S + dot above + dot below
let s2 = "S\u0323\u0307"; // S + dot below + dot above

alert(s1 == s2); // false (different composition)

// Normalize to make them equal
alert(s1.normalize() == s2.normalize()); // true
alert(s1.normalize().length); // 1 (combined into single character)
```

## Useful Methods

| Method                          | Description                                               |
| ------------------------------- | --------------------------------------------------------- |
| `str.codePointAt(pos)`          | Returns full Unicode code point (handles surrogate pairs) |
| `String.fromCodePoint(code)`    | Creates character from code point                         |
| `str.normalize()`               | Normalizes string to canonical form                       |
| `[...str]` or `Array.from(str)` | Splits string into characters (handles surrogate pairs)   |

## Summary

| Concept             | Key Points                                          |
| ------------------- | --------------------------------------------------- |
| **Notations**       | `\xXX`, `\uXXXX`, `\u{X...}`                        |
| **Surrogate pairs** | Rare symbols use 2 characters; `length` shows 2     |
| **Safe iteration**  | Use `[...str]` or `Array.from(str)`                 |
| **Normalization**   | `str.normalize()` makes visually same strings equal |

**Key Points:**

- JavaScript uses UTF-16 internally
- Surrogate pairs cause string length to be 2 for single symbols
- Use `codePointAt()` and `String.fromCodePoint()` for surrogate-aware operations
- Use `normalize()` to compare visually identical strings with different compositions
- Spread operator (`...`) safely splits strings into characters
