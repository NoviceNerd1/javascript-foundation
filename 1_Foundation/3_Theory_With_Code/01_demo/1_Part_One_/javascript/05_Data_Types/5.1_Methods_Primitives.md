# Methods of Primitives

## Overview

Primitives (strings, numbers, etc.) can be used like objects with methods, but they remain lightweight primitives.

## Primitives vs Objects

| Primitives                                                        | Objects                                  |
| ----------------------------------------------------------------- | ---------------------------------------- |
| Single value                                                      | Multiple properties                      |
| 7 types: string, number, bigint, boolean, symbol, null, undefined | Created with `{}`, functions are objects |
| Lightweight                                                       | Heavier, require more resources          |

## How Primitive Methods Work

```javascript
let str = "Hello";
alert(str.toUpperCase()); // HELLO
```

**What happens behind the scenes:**

1. A special **object wrapper** (String) is created with the primitive value
2. The method runs and returns a result
3. The wrapper is destroyed

> JavaScript engines optimize this process heavily.

## Built-in Wrapper Constructors

| Primitive | Wrapper Object |
| --------- | -------------- |
| string    | `String`       |
| number    | `Number`       |
| boolean   | `Boolean`      |
| symbol    | `Symbol`       |
| bigint    | `BigInt`       |

## Number Methods Example

```javascript
let n = 1.23456;
alert(n.toFixed(2)); // 1.23
```

## Important: Don't Use `new` with Wrappers

```javascript
alert(typeof 0); // "number"
alert(typeof new Number(0)); // "object"

let zero = new Number(0);
if (zero) {
  // Objects are always truthy
  alert("zero is truthy!?!"); // This runs!
}
```

**But using wrappers without `new` is fine (type conversion):**

```javascript
let num = Number("123"); // Converts to number primitive
let str = String(123); // Converts to string primitive
let bool = Boolean(1); // Converts to boolean primitive
```

## `null` and `undefined` Have No Methods

```javascript
alert(null.test); // Error!
alert(undefined.test); // Error!
```

## Task: Can I Add a String Property?

```javascript
let str = "Hello";
str.test = 5;
alert(str.test); // undefined
```

**Explanation:** When accessing `str.test`, a temporary wrapper object is created, the property is added to it, then the wrapper is destroyed. The next line creates a new wrapper with no `test` property, so it returns `undefined`.

## Summary

- Primitives provide methods via temporary object wrappers
- Wrappers are destroyed after use
- Never use `new` with `String`/`Number`/`Boolean`
- `null` and `undefined` have no methods
