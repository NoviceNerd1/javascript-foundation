# Backreferences: `\N` and `\k<name>` Summary

Backreferences allow using the content of capturing groups **inside the pattern itself**—useful for matching paired quotes, tags, or repeating patterns.

## Backreference by Number: `\N`

`\N` refers to the content of the Nth capturing group (groups numbered left to right by opening parenthesis).

```javascript
// Without backreference: matches mixed quotes
let str = `He said: "She's the one!".`;
let wrong = /['"](.*?)['"]/g;
console.log(str.match(wrong)); // ["She'] (wrong!)

// With backreference: closing quote matches opening quote
let correct = /(['"])(.*?)\1/g;
console.log(str.match(correct)); // ["She's the one!"] (correct!)
```

**How it works:**

1. `(['"])` captures the opening quote (`"` or `'`)
2. `(.*?)` matches the content lazily
3. `\1` matches exactly the same quote as captured in group 1

## Group Numbering

Groups are numbered left to right:

```javascript
let regexp = /(a)(b)\2\1/; // matches "abba"
console.log(regexp.test("abba")); // true

// Group 1 = "a"
// Group 2 = "b"
// \2 matches "b", \1 matches "a"
```

## Backreference by Name: `\k<name>`

For named groups, use `\k<name>`:

```javascript
let str = `He said: "She's the one!".`;
let regexp = /(?<quote>['"])(.*?)\k<quote>/g;

console.log(str.match(regexp)); // ["She's the one!"]
```

## Practical Examples

### Matching HTML Tags

```javascript
let str = "<div>Hello</div> <span>World</span>";
let regexp = /<(\w+)>.*?<\/\1>/g;

console.log(str.match(regexp));
// ["<div>Hello</div>", "<span>World</span>"]
```

### Matching Nested Patterns

```javascript
// Find repeated words
let str = "The the sky is is blue";
let regexp = /\b(\w+)\s+\1\b/gi;

console.log(str.match(regexp)); // ["The the", "is is"]
```

### Validating Balanced Quotes

```javascript
function hasBalancedQuotes(str) {
  return /^(['"]).*\1$/.test(str);
}

console.log(hasBalancedQuotes('"Hello"')); // true
console.log(hasBalancedQuotes("\"Hello'")); // false
```

### Matching Multiple Groups

```javascript
// Pattern with multiple backreferences
let regexp = /(\d{2})-(\d{2})-(\d{4}) \1-\2-\3/;
// matches "12-12-2012 12-12-2012"

console.log(regexp.test("12-12-2012 12-12-2012")); // true
console.log(regexp.test("12-12-2012 13-13-2013")); // false
```

## Important Notes

### 1. Non-capturing groups `(?:...)` are not available

```javascript
let regexp = /(?:a)(b)\1/; // \1 refers to (b), not (?:a)
console.log(regexp.test("abb")); // true
```

### 2. Pattern vs Replacement Syntax

| Context        | Syntax                 |
| -------------- | ---------------------- |
| In pattern     | `\1`, `\2`, `\k<name>` |
| In replacement | `$1`, `$2`, `$<name>`  |

```javascript
let str = "John Bull";
let regexp = /(\w+) (\w+)/;

// In pattern: \1, \2 (not used here)
// In replacement: $1, $2
console.log(str.replace(regexp, "$2, $1")); // "Bull, John"
```

### 3. Backreference Order

```javascript
let regexp = /(a)(b)(c)\2\1\3/; // matches "abccba"
console.log(regexp.test("abccba")); // true
// \2 = "b", \1 = "a", \3 = "c"
```

## Common Use Cases

### Validate Code Blocks

````javascript
let str = "```js\nconsole.log('hi');\n```";
let regexp = /```(\w*)\n[\s\S]*?\n```/;
// Matches code block with consistent backticks
````

### Find Repeating Words

```javascript
function findDuplicates(text) {
  return text.match(/\b(\w+)\s+\1\b/gi) || [];
}

console.log(findDuplicates("This this is a test test"));
// ["This this", "test test"]
```

### Validate XML/HTML Tags

```javascript
function isValidTag(tag) {
  return /^<(\w+)>.*<\/\1>$/.test(tag);
}

console.log(isValidTag("<div>content</div>")); // true
console.log(isValidTag("<div>content</span>")); // false
```

## Summary

| Feature                 | Syntax          | Example                  |
| ----------------------- | --------------- | ------------------------ |
| Backreference by number | `\N`            | `/(['"])(.*?)\1/`        |
| Backreference by name   | `\k<name>`      | `/(?<q>['"])(.*?)\k<q>/` |
| Replacement reference   | `$N`, `$<name>` | `'$2, $1'`               |

**Key Points:**

- Backreferences refer to captured group content, not the pattern itself
- Groups numbered by opening parentheses order
- Non-capturing groups `(?:...)` cannot be referenced
- Use `\1` in pattern, `$1` in replacement
- Named groups use `\k<name>` in pattern, `$<name>` in replacement
- Useful for matching paired delimiters, repeated words, balanced structures
