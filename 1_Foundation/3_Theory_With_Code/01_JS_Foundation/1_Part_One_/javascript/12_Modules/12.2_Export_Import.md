# Export and Import

## Overview

JavaScript modules use `export` and `import` directives to share code between files. There are multiple syntax variants for different use cases.

## Export Syntax

### Export Before Declarations

```javascript
// export array
export let months = ["Jan", "Feb", "Mar"];

// export constant
export const YEAR = 2015;

// export class
export class User {
  constructor(name) {
    this.name = name;
  }
}

// export function
export function sayHi(user) {
  alert(`Hello, ${user}!`);
}
```

### Export Apart from Declarations

```javascript
// 📁 say.js
function sayHi(user) {
  alert(`Hello, ${user}!`);
}
function sayBye(user) {
  alert(`Bye, ${user}!`);
}

export { sayHi, sayBye }; // List exports
```

### Export with Aliases

```javascript
export { sayHi as hi, sayBye as bye };
```

## Import Syntax

### Import Named Exports

```javascript
import { sayHi, sayBye } from "./say.js";

// With aliases
import { sayHi as hi, sayBye as bye } from "./say.js";
```

### Import Everything as an Object

```javascript
import * as say from "./say.js";

say.sayHi("John");
say.sayBye("John");
```

### Import Default Export

```javascript
// 📁 user.js
export default class User {
  constructor(name) {
    this.name = name;
  }
}

// 📁 main.js
import User from "./user.js"; // No curly braces
new User("John");
```

### Import Both Default and Named

```javascript
// 📁 user.js
export default class User { ... }
export function sayHi(user) { ... }

// 📁 main.js
import User, { sayHi } from './user.js';
```

### Import Without Assignment

```javascript
import "./say.js"; // Runs the module code, doesn't assign exports
```

## Default Export Details

### Default Export Can Be Anonymous

```javascript
export default class { ... }           // No class name
export default function(user) { ... }  // No function name
export default ['Jan', 'Feb'];         // Single value
```

### Referencing Default Export

```javascript
// Export as default from inside
export { sayHi as default };

// Import default with alias
import { default as User } from "./user.js";

// Using import * with default
import * as user from "./user.js";
let User = user.default;
```

## Re-export

### Basic Re-export

```javascript
export { sayHi } from "./say.js";
export { default as User } from "./user.js";
```

### Re-export Everything (Named Only)

```javascript
export * from "./user.js"; // Ignores default export
```

### Re-export Both Named and Default

```javascript
export * from "./user.js"; // Named exports
export { default } from "./user.js"; // Default export
```

## Practical Package Structure Example

```
auth/
    index.js          // Main entry point
    user.js
    helpers.js
    providers/
        github.js
        facebook.js
```

```javascript
// 📁 auth/index.js (main entry)
export { login, logout } from "./helpers.js";
export { default as User } from "./user.js";
export { default as Github } from "./providers/github.js";
```

Usage:

```javascript
import { login, logout, User, Github } from "auth/index.js";
```

## Key Differences

| Feature            | Named Export         | Default Export         |
| ------------------ | -------------------- | ---------------------- |
| Syntax             | `export class User`  | `export default class` |
| Import             | `import {User} from` | `import User from`     |
| Multiple per file  | Yes                  | Only one               |
| Must match name    | Yes                  | No (any name works)    |
| Re-export with `*` | Included             | Excluded               |

## Important Notes

### import/export Must Be at Top Level

```javascript
// Error: cannot be inside blocks
if (something) {
  import { sayHi } from "./say.js";
}
```

### Order Doesn't Matter

```javascript
sayHi(); // Works even though import is below

import { sayHi } from "./say.js";
```

### No Semicolons After export class/function

```javascript
export function sayHi() {} // No semicolon needed
```

## Summary

| Operation               | Syntax                                      |
| ----------------------- | ------------------------------------------- |
| Named export            | `export {x, y}` or `export function f() {}` |
| Default export          | `export default class {}`                   |
| Named import            | `import {x, y} from 'module'`               |
| Default import          | `import X from 'module'`                    |
| Import all              | `import * as obj from 'module'`             |
| Import with alias       | `import {x as y} from 'module'`             |
| Export with alias       | `export {x as y}`                           |
| Re-export               | `export {x} from 'module'`                  |
| Import for side effects | `import 'module'`                           |
