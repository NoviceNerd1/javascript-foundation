# Optional Chaining `?.`

## Overview

Optional chaining `?.` safely accesses nested object properties without errors when intermediate properties don't exist (returning `undefined`).

## The Problem

```javascript
let user = {}; // No address property

alert(user.address.street); // Error!
```

## Traditional Solutions (Inelegant)

```javascript
// Using if/ternary (repetitive)
alert(user.address ? user.address.street : undefined);

// Using && (still repetitive)
alert(user.address && user.address.street && user.address.street.name);
```

## Optional Chaining Solution

```javascript
let user = {};

alert(user?.address?.street); // undefined (no error)
```

### With Existing Properties

```javascript
let user = {
  address: {
    street: "Main St",
  },
};

alert(user?.address?.street); // "Main St"
```

## Key Rules

### 1. Checks Only What's Before `?.`

```javascript
let user = null;
alert(user?.address.street); // undefined (user is null/undefined)
// address.street not evaluated
```

### 2. Don't Overuse

```javascript
// User must exist, address is optional
let user = { address: null };
alert(user.address?.street); // Correct

// Bad - would hide bug if user is missing
alert(user?.address?.street); // Avoid if user should exist
```

### 3. Variable Must Be Declared

```javascript
// ReferenceError: user is not defined
user?.address;

// Must be declared first
let user;
user?.address; // undefined (no error)
```

## Short-Circuiting

```javascript
let user = null;
let x = 0;

user?.sayHi(x++); // Stops before calling sayHi

alert(x); // 0 (not incremented)
```

## Other Variants

### `?.()` – Call Method That May Not Exist

```javascript
let userAdmin = {
  admin() {
    alert("I am admin");
  },
};
let userGuest = {};

userAdmin.admin?.(); // "I am admin"
userGuest.admin?.(); // Nothing (no error)
```

### `?.[]` – Access Property with Brackets

```javascript
let key = "firstName";
let user1 = { firstName: "John" };
let user2 = null;

alert(user1?.[key]); // John
alert(user2?.[key]); // undefined
```

### With `delete`

```javascript
delete user?.name; // Deletes if user exists
```

## Important Restriction: No Writing

```javascript
let user = null;
user?.name = "John"; // Error! Cannot use ?. on left side of assignment
```

## Summary

| Form             | Purpose                            |
| ---------------- | ---------------------------------- |
| `obj?.prop`      | Safe property access               |
| `obj?.[prop]`    | Safe property access with brackets |
| `obj.method?.()` | Safe method call                   |

**Use carefully:** Only where missing properties are acceptable, not to hide programming errors.
