# Class Checking: instanceof

## Overview

The `instanceof` operator checks if an object belongs to a certain class, considering inheritance.

## Basic Syntax

```javascript
obj instanceof Class;
```

### Examples

```javascript
class Rabbit {}
let rabbit = new Rabbit();
alert(rabbit instanceof Rabbit); // true

// Works with constructor functions
function Rabbit() {}
alert(new Rabbit() instanceof Rabbit); // true

// Works with built-in classes
let arr = [1, 2, 3];
alert(arr instanceof Array); // true
alert(arr instanceof Object); // true (Array inherits from Object)
```

## How `instanceof` Works

### 1. With `Symbol.hasInstance` (Custom Logic)

```javascript
class Animal {
  static [Symbol.hasInstance](obj) {
    if (obj.canEat) return true;
  }
}

let obj = { canEat: true };
alert(obj instanceof Animal); // true (custom check)
```

### 2. Default Logic (Prototype Chain Check)

If no `Symbol.hasInstance`, it checks if `Class.prototype` appears in `obj`'s prototype chain:

```javascript
class Animal {}
class Rabbit extends Animal {}

let rabbit = new Rabbit();
alert(rabbit instanceof Animal); // true

// Checks: rabbit.__proto__ === Animal.prototype? (no)
//         rabbit.__proto__.__proto__ === Animal.prototype? (yes)
```

This is equivalent to: `Class.prototype.isPrototypeOf(obj)`

### Important: Constructor Doesn't Participate

Only the prototype chain matters, not the constructor itself:

```javascript
function Rabbit() {}
let rabbit = new Rabbit();

// Change prototype after object creation
Rabbit.prototype = {};

alert(rabbit instanceof Rabbit); // false (prototype changed)
```

## Object.prototype.toString for Type Checking

### Basic Usage

```javascript
let objectToString = Object.prototype.toString;

alert(objectToString.call(123)); // [object Number]
alert(objectToString.call(true)); // [object Boolean]
alert(objectToString.call(null)); // [object Null]
alert(objectToString.call(undefined)); // [object Undefined]
alert(objectToString.call([1, 2, 3])); // [object Array]
alert(objectToString.call(/regex/)); // [object RegExp]
```

### Customizing with `Symbol.toStringTag`

```javascript
let user = {
  [Symbol.toStringTag]: "User",
};

alert({}.toString.call(user)); // [object User]

// Built-in examples
alert(window[Symbol.toStringTag]); // Window
alert({}.toString.call(window)); // [object Window]
alert({}.toString.call(new XMLHttpRequest())); // [object XMLHttpRequest]
```

## Comparison Table

| Method               | Works For                     | Returns                |
| -------------------- | ----------------------------- | ---------------------- |
| `typeof`             | Primitives                    | String                 |
| `{}.toString.call()` | Primitives, built-ins, custom | String `[object Type]` |
| `instanceof`         | Objects (with inheritance)    | Boolean                |

## Summary

- `instanceof` checks prototype chain (or uses `Symbol.hasInstance`)
- `Class.prototype` must be in object's prototype chain, not the constructor
- `{}.toString.call()` is a more powerful type checker (works on primitives too)
- Use `Symbol.toStringTag` to customize `{}.toString` output

## Task: Strange instanceof

```javascript
function A() {}
function B() {}

A.prototype = B.prototype = {};

let a = new A();

alert(a instanceof B); // true
```

**Why?** `a.__proto__` is `A.prototype` which equals `B.prototype`. Since `B.prototype` is in `a`'s prototype chain, `instanceof B` returns `true`. The constructors themselves don't matter â€“ only the prototype chain.
