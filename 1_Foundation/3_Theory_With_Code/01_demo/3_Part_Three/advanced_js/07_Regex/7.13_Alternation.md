# Alternation (OR) `|` Summary

Alternation allows matching one of several expressions. It's like logical OR for regex patterns.

## Basic Syntax

```javascript
// Match "html", "php", "css", or "java(script)?"
let regexp = /html|php|css|java(script)?/gi;
let str = "First HTML appeared, then CSS, then JavaScript";
console.log(str.match(regexp)); // ['HTML', 'CSS', 'JavaScript']
```

## Alternation vs Character Classes

| Feature | Character Class `[...]` | Alternation `  | `   |
| ------- | ----------------------- | -------------- | --- |
| Scope   | Single characters       | Any expression |
| Example | `gr[ae]y`               | `gra           | ey` |
| Matches | `gray`, `grey`          | `gra` or `ey`  |

## Grouping with Parentheses

```javascript
// Without parentheses: matches "I love HTML" or "CSS"
console.log("I love HTML".match(/I love HTML|CSS/)); // ["I love HTML"]

// With parentheses: matches "I love HTML" or "I love CSS"
console.log("I love CSS".match(/I love (HTML|CSS)/)); // ["I love CSS", "CSS"]
```

## Example: Time Validation (hh:mm)

```javascript
// Hours: 00-23, Minutes: 00-59
let regexp = /([01]\d|2[0-3]):[0-5]\d/g;
let str = "00:00 10:10 23:59 25:99 1:2";
console.log(str.match(regexp)); // ['00:00', '10:10', '23:59']
```

**Breaking it down:**

- `([01]\d|2[0-3])` - hours: either `0x`/`1x` OR `2[0-3]`
- `:` - literal colon
- `[0-5]\d` - minutes: `0x`-`5x`

## Common Patterns with Alternation

### Match Multiple Words

```javascript
let str = "I like cats, dogs, and birds";
let regexp = /cat|dog|bird/gi;
console.log(str.match(regexp)); // ['cats', 'dogs', 'bird']
```

### Match Different Date Formats

```javascript
let str = "2024-01-15, 01/15/2024, 15.01.2024";
let regexp = /\d{4}-\d{2}-\d{2}|\d{2}\/\d{2}\/\d{4}|\d{2}\.\d{2}\.\d{4}/g;
console.log(str.match(regexp));
// ['2024-01-15', '01/15/2024', '15.01.2024']
```

### Match HTTP or HTTPS

```javascript
let regexp = /https?:\/\//;
console.log(regexp.test("http://example.com")); // true
console.log(regexp.test("https://example.com")); // true
```

### Match Email Domains

```javascript
let regexp = /@(gmail|yahoo|outlook)\.com$/i;
console.log(regexp.test("user@gmail.com")); // true
console.log(regexp.test("user@yahoo.com")); // true
console.log(regexp.test("user@hotmail.com")); // false
```

## Order Matters

Alternation tries options **left to right**:

```javascript
let regexp = /Java|JavaScript/;
console.log("JavaScript".match(regexp)); // ['Java'] (first match, not full)

// Better: put longer first
let better = /JavaScript|Java/;
console.log("JavaScript".match(better)); // ['JavaScript']
```

## Tasks

### Task 1: Find Programming Languages

```javascript
let regexp = /Java|JavaScript|PHP|C(\+\+)?|C/g;
// Better order: longer first
let regexp = /JavaScript|Java|PHP|C\+\+|C/g;

let str = "Java JavaScript PHP C++ C";
console.log(str.match(regexp)); // ['Java', 'JavaScript', 'PHP', 'C++', 'C']
```

### Task 2: Find BB-Tag Pairs

```javascript
let regexp = /\[(b|url|quote)\][\s\S]*?\[\/\1\]/g;
let str = "[url][b]http://google.com[/b][/url]";
console.log(str.match(regexp)); // ['[url][b]http://google.com[/b][/url]']
```

### Task 3: Find Quoted Strings with Escaping

```javascript
let regexp = /"(\\.|[^"\\])*"/g;
let str = ' .. "test me" .. "Say \\"Hello\\"!" .. "\\\\ \\"" .. ';
console.log(str.match(regexp));
// ['"test me"', '"Say \\"Hello\\"!"', '"\\\\ \\""']
```

**Explanation:**

- `"` - opening quote
- `(\\.|[^"\\])*` - content:
  - `\\.` - escaped character (backslash + any)
  - `[^"\\]` - any character except quote or backslash
- `"` - closing quote

### Task 4: Find Full `<style>` Tag

```javascript
let regexp = /<style(\s[^>]*)?>/g;
let str = '<style> <styler> <style test="...">';
console.log(str.match(regexp)); // ['<style>', '<style test="...">']
```

**Explanation:**

- `<style` - literal tag start
- `(\s[^>]*)?` - optional: space followed by any non-`>` characters
- `>` - closing bracket

## Summary

| Pattern  | Description     |
| -------- | --------------- | -------------------- | ---------------- |
| `A       | B`              | Match A OR B         |
| `(A      | B)C`            | Match A or B, then C |
| `A       | B               | C`                   | Match A, B, or C |
| `https?` | Short for `http | https`               |

**Key Points:**

- Alternation matches one of several patterns
- Use parentheses `(A|B)` to limit scope
- Order matters: first matching option wins (put longer variants first)
- Character classes `[...]` only work for single characters
- Alternation works with any length expressions
- Useful for validating formats (time, date, email) and matching alternatives
