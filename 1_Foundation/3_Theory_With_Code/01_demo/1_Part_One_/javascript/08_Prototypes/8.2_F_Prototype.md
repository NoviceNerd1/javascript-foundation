# F.prototype

## Overview

When creating objects with a constructor function `new F()`, the `F.prototype` property sets the `[[Prototype]]` of the new object.

```javascript
let animal = { eats: true };

function Rabbit(name) {
  this.name = name;
}

Rabbit.prototype = animal; // Sets prototype for new Rabbit objects

let rabbit = new Rabbit("White Rabbit");
alert(rabbit.eats); // true (inherited from animal)
```

## How It Works

- `F.prototype` is a **regular property** on the constructor function
- When `new F()` is called, the new object's `[[Prototype]]` is set to `F.prototype`
- This happens **only at creation time** – existing objects are unaffected by later changes to `F.prototype`

## Default `prototype` and `constructor`

Every function has a default `prototype` object with a `constructor` property pointing back to the function itself.

```javascript
function Rabbit() {}

// Default: Rabbit.prototype = { constructor: Rabbit }
alert(Rabbit.prototype.constructor === Rabbit); // true

let rabbit = new Rabbit();
alert(rabbit.constructor === Rabbit); // true (inherited)
```

### Using `constructor` to Create Similar Objects

```javascript
function Rabbit(name) {
  this.name = name;
}

let rabbit = new Rabbit("White Rabbit");
let rabbit2 = new rabbit.constructor("Black Rabbit"); // Works
```

## Important: `constructor` Is Not Guaranteed

If you **replace** the entire `prototype`, the `constructor` property is lost.

```javascript
function Rabbit() {}
Rabbit.prototype = { jumps: true }; // Replaces default prototype

let rabbit = new Rabbit();
alert(rabbit.constructor === Rabbit); // false
```

### Fix: Preserve `constructor`

```javascript
// Option 1: Add properties without replacing
Rabbit.prototype.jumps = true;

// Option 2: Add constructor manually
Rabbit.prototype = {
  jumps: true,
  constructor: Rabbit,
};
```

## Key Points

| Concept             | Description                                                            |
| ------------------- | ---------------------------------------------------------------------- |
| `F.prototype`       | Property on constructor function, sets `[[Prototype]]` for new objects |
| Default `prototype` | `{ constructor: F }`                                                   |
| `new F()`           | Uses `F.prototype` at creation time only                               |
| `constructor`       | Not automatically preserved if prototype is overwritten                |

## Tasks

### 1. Changing "prototype"

```javascript
function Rabbit() {}
Rabbit.prototype = { eats: true };
let rabbit = new Rabbit();

// Case 1: Replacing prototype after creation
Rabbit.prototype = {};
alert(rabbit.eats); // true (existing object still has old prototype)

// Case 2: Modifying prototype property
Rabbit.prototype.eats = false;
alert(rabbit.eats); // false (prototype object changed)

// Case 3: Deleting own property
delete rabbit.eats;
alert(rabbit.eats); // true (gets from prototype)

// Case 4: Deleting prototype property
delete Rabbit.prototype.eats;
alert(rabbit.eats); // undefined (property no longer exists)
```

### 2. Create Object with Same Constructor

```javascript
// Works correctly (constructor preserved)
function User(name) {
  this.name = name;
}
let user = new User("John");
let user2 = new user.constructor("Pete"); // Works

// Fails (constructor lost)
function Admin(name) {
  this.name = name;
}
Admin.prototype = {}; // Overwrites default prototype
let admin = new Admin("Alice");
let admin2 = new admin.constructor(); // admin.constructor is Object, not Admin
// Creates plain object {}, not Admin instance
```

## Summary

- `F.prototype` sets prototype for objects created with `new F()`
- Default `prototype` includes `constructor` property
- `constructor` can be used to create similar objects, but is lost if prototype is overwritten
- Preserve `constructor` by adding properties or manually setting it when replacing prototype
