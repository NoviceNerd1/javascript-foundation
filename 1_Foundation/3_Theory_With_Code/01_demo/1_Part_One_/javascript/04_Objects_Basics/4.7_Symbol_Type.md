# Symbol Type

## Overview

Symbols are primitive unique identifiers. Only **strings** and **symbols** can be object property keys.

## Creating Symbols

```javascript
let id = Symbol(); // Without description
let id = Symbol("id"); // With description (for debugging)

let id1 = Symbol("id");
let id2 = Symbol("id");
alert(id1 == id2); // false (always unique)
```

## Symbols Don't Auto-Convert to String

```javascript
let id = Symbol("id");
alert(id); // Error!
alert(id.toString()); // Symbol(id)
alert(id.description); // id
```

## Hidden Properties

Symbols create properties that other code can't accidentally access or overwrite.

```javascript
let user = { name: "John" }; // From third-party code

let id = Symbol("id");
user[id] = 1; // Safe addition

alert(user[id]); // 1

// Another script can use its own symbol without conflict
let id2 = Symbol("id");
user[id2] = "Their value";

alert(user[id]); // 1 (unaffected)
alert(user[id2]); // Their value
```

## Symbols in Object Literals

Use **square brackets** `[]` to use symbol as key:

```javascript
let id = Symbol("id");

let user = {
  name: "John",
  [id]: 123, // Not "id": 123
};
```

## Symbols Are Skipped by `for...in`

```javascript
let id = Symbol("id");
let user = {
  name: "John",
  age: 30,
  [id]: 123,
};

for (let key in user) alert(key); // name, age (no symbol)
alert(user[id]); // 123 (direct access works)
```

### `Object.assign()` Copies Symbols

```javascript
let id = Symbol("id");
let user = { [id]: 123 };
let clone = Object.assign({}, user);

alert(clone[id]); // 123 (symbols are copied)
```

## Global Symbols Registry

Use `Symbol.for(key)` to get/create global symbols. Same key returns same symbol.

```javascript
let id = Symbol.for("id");
let idAgain = Symbol.for("id");

alert(id === idAgain); // true
```

### `Symbol.keyFor()` – Get Name by Symbol

```javascript
let sym = Symbol.for("name");
let sym2 = Symbol("local");

alert(Symbol.keyFor(sym)); // name (global)
alert(Symbol.keyFor(sym2)); // undefined (not global)
alert(sym2.description); // local (description works for all)
```

## System Symbols (Well-Known)

JavaScript uses internal symbols that we can use to customize behavior:

| Symbol                      | Purpose                        |
| --------------------------- | ------------------------------ |
| `Symbol.iterator`           | Makes objects iterable         |
| `Symbol.toPrimitive`        | Object-to-primitive conversion |
| `Symbol.hasInstance`        | Custom `instanceof` behavior   |
| `Symbol.isConcatSpreadable` | Array concat behavior          |

## Summary

| Feature             | Description                            |
| ------------------- | -------------------------------------- |
| **Creation**        | `Symbol()` or `Symbol.for(key)`        |
| **Uniqueness**      | Always unique (except global registry) |
| **Hidden**          | Skipped by `for...in`, `Object.keys()` |
| **Use cases**       | Hidden properties, system symbols      |
| **Global registry** | `Symbol.for(key)` for shared symbols   |
