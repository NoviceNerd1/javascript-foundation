# Debugging in the Browser

## Overview

Debugging is finding and fixing errors in scripts. All modern browsers include developer tools with debugging capabilities.

## Opening Developer Tools

- **Windows/Linux**: `F12` or `Ctrl+Shift+I`
- **Mac**: `Cmd+Opt+I`
- Select the **Sources** panel

## Sources Panel Components

| Panel                    | Purpose                                               |
| ------------------------ | ----------------------------------------------------- |
| **File Navigator**       | Lists HTML, JS, CSS, and other files                  |
| **Code Editor**          | Displays source code                                  |
| **JavaScript Debugging** | Controls and information for debugging                |
| **Console**              | Appears with `Esc` – execute commands and view output |

## Breakpoints

### Setting Breakpoints

Click on a line number in the Sources panel. Execution pauses at that line.

```javascript
// Click line 4 to set breakpoint
function hello(name) {
  let phrase = `Hello, ${name}!`; // Breakpoint here
  say(phrase);
}
```

### Conditional Breakpoints

Right-click a line number → "Add conditional breakpoint" → enter expression. Breakpoint triggers only when expression is truthy.

### The `debugger` Command

```javascript
function hello(name) {
  let phrase = `Hello, ${name}!`;
  debugger; // Pauses execution here (dev tools must be open)
  say(phrase);
}
```

## Debugging Controls

| Button | Name           | Hotkey      | Action                                           |
| ------ | -------------- | ----------- | ------------------------------------------------ |
| ▶️     | Resume         | `F8`        | Continue execution until next breakpoint         |
| ⏭️     | Step           | `F9`        | Execute next statement                           |
| ⏭️➡️   | Step over      | `F10`       | Execute next statement (skip function internals) |
| ⬇️     | Step into      | `F11`       | Step into function call                          |
| ⬆️     | Step out       | `Shift+F11` | Continue to end of current function              |
| ⚪     | Breakpoints    | -           | Enable/disable all breakpoints                   |
| ⚠️     | Pause on error | -           | Auto-pause when error occurs                     |

## Information Panels (Right Side)

| Panel          | Description                                                    |
| -------------- | -------------------------------------------------------------- |
| **Watch**      | View values of custom expressions (add with `+`)               |
| **Call Stack** | Shows nested function call chain – click items to jump to code |
| **Scope**      | Displays `Local` (function) and `Global` variables             |

## Console Logging

```javascript
for (let i = 0; i < 5; i++) {
  console.log("value,", i); // Outputs to Console panel
}
```

- View output in **Console** panel or press `Esc` in any panel
- Regular users don't see console output

## Right-Click Context Menu Options

- **Continue to here** – resume execution until reaching the clicked line
- Remove/disable breakpoints

## Summary: Three Ways to Pause Script

1. **Breakpoint** – set in Sources panel
2. **`debugger` statement** – in code
3. **Error** – with "Pause on error" enabled

**Pro tip:** Explore dev tools by clicking around and using context menus – it's the fastest way to learn.
