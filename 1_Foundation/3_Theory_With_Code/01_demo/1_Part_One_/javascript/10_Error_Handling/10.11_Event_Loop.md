# Event Loop: Microtasks and Macrotasks

## Overview

JavaScript runs in an **event loop** that manages task execution. Understanding it helps with performance and avoiding blocking.

## Event Loop Algorithm (Simplified)

1. Run the oldest **macrotask** from the queue
2. Execute **all microtasks** (entire queue)
3. Render changes (if any)
4. Wait for next macrotask, repeat

## Macrotasks vs Microtasks

| Type           | Sources                                                                   | Queue Behavior                    |
| -------------- | ------------------------------------------------------------------------- | --------------------------------- |
| **Macrotasks** | `setTimeout`, `setInterval`, scripts, events, `setImmediate`              | One per loop iteration            |
| **Microtasks** | Promise handlers (`.then`/`.catch`/`.finally`), `queueMicrotask`, `await` | All executed after each macrotask |

## Execution Order Example

```javascript
setTimeout(() => alert("timeout")); // Macrotask

Promise.resolve().then(() => alert("promise")); // Microtask

alert("code"); // Synchronous

// Output:
// code
// promise
// timeout
```

## Visual Flow

```
Macrotask Queue: [script] [setTimeout] ...
Microtask Queue: [promise handlers] ...

1. Run script (macrotask)
2. Run ALL microtasks
3. Render
4. Run next macrotask (setTimeout)
5. Run ALL microtasks
6. Render...
```

## Practical Use Cases

### 1. Splitting CPU-Heavy Tasks

Prevent UI freeze by splitting work with `setTimeout`:

```javascript
let i = 0;

function count() {
  // Do chunk of work
  do {
    i++;
  } while (i % 1e6 != 0);

  if (i < 1e9) {
    setTimeout(count); // Schedule next chunk
  }
}

count(); // UI stays responsive
```

### 2. Progress Indication

Show progress between chunks:

```javascript
let i = 0;
let progress = document.getElementById("progress");

function count() {
  do {
    i++;
    progress.innerHTML = i; // Updates visible between chunks
  } while (i % 1000 != 0);

  if (i < 1e7) {
    setTimeout(count);
  }
}

count();
```

### 3. Postpone Action After Event

```javascript
menu.onclick = function () {
  // ... handle click ...

  let customEvent = new CustomEvent("menu-open", { bubbles: true });

  // Dispatch after click fully handled
  setTimeout(() => menu.dispatchEvent(customEvent));
};
```

## queueMicrotask

Schedule a microtask directly:

```javascript
queueMicrotask(() => alert("runs after current code, before macrotasks"));
```

## Key Differences

| Aspect        | Macrotasks                          | Microtasks                         |
| ------------- | ----------------------------------- | ---------------------------------- |
| Examples      | `setTimeout`, `setInterval`, events | Promise handlers, `queueMicrotask` |
| When executed | One per event loop iteration        | All after each macrotask           |
| Rendering     | Happens after microtasks            | Not between microtasks             |

## Task: What Will Be the Output?

```javascript
console.log(1);

setTimeout(() => console.log(2));

Promise.resolve().then(() => console.log(3));

Promise.resolve().then(() => setTimeout(() => console.log(4)));

Promise.resolve().then(() => console.log(5));

setTimeout(() => console.log(6));

console.log(7);
```

**Output:** `1, 7, 3, 5, 2, 6, 4`

**Explanation:**

1. Synchronous: `1, 7`
2. All microtasks run: `3, 5` (plus a `setTimeout` queued for later)
3. Macrotasks run: `2, 6`
4. Finally, the `setTimeout` from microtask runs: `4`

## Summary

- **Microtasks** run immediately after each macrotask, before rendering
- **Macrotasks** run one per loop iteration
- Use `setTimeout` to split heavy work and keep UI responsive
- Use `queueMicrotask` or promise handlers to run code after current task but before next macrotask
- Rendering occurs after microtasks are complete
