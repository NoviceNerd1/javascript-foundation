# Extending Built-in Classes

## Overview

Built-in classes like Array, Map, and Set can be extended, allowing custom functionality while preserving built-in methods.

## Basic Extension

```javascript
class PowerArray extends Array {
  isEmpty() {
    return this.length === 0;
  }
}

let arr = new PowerArray(1, 2, 5, 10, 50);
alert(arr.isEmpty()); // false

// Built-in methods return PowerArray instances
let filteredArr = arr.filter((item) => item >= 10);
alert(filteredArr); // 10, 50
alert(filteredArr.isEmpty()); // false (still PowerArray)
```

Built-in methods like `filter`, `map`, etc., return new objects of the **same type** because they use the object's `constructor` property internally.

## Customizing with `Symbol.species`

To make built-in methods return regular arrays instead of the extended type:

```javascript
class PowerArray extends Array {
  isEmpty() {
    return this.length === 0;
  }

  static get [Symbol.species]() {
    return Array; // Built-in methods use this constructor
  }
}

let arr = new PowerArray(1, 2, 5, 10, 50);
let filteredArr = arr.filter((item) => item >= 10);

alert(filteredArr instanceof PowerArray); // false
alert(filteredArr instanceof Array); // true
alert(filteredArr.isEmpty); // undefined (not a PowerArray)
```

## Other Built-in Collections

Map, Set, and other built-in collections work similarly with `Symbol.species`.

## Built-in Inheritance Limitation

Built-in classes **do not inherit static methods** from each other.

```javascript
class Animal {}
class Rabbit extends Animal {}

alert(Rabbit.__proto__ === Animal); // true (static inheritance works for custom classes)
```

But for built-ins:

```javascript
// Array inherits from Object
alert(Array.prototype.__proto__ === Object.prototype); // true (instance methods)

// But static methods are NOT inherited
alert(Array.__proto__ === Object); // false
alert(Array.keys); // undefined (no static inheritance)
```

### Structure of Built-in Inheritance

```
Object (constructor)
  ├── prototype: Object.prototype
  │     └── hasOwnProperty, toString, etc.
  │
  └── Array (constructor)
        ├── prototype: Array.prototype
        │     └── inherits from Object.prototype
        └── static methods: Array.isArray, etc.
```

**Key Point:** `Array` does not inherit static methods from `Object`. Only instance methods are inherited via the prototype chain.

## Summary

| Feature                           | Behavior                                         |
| --------------------------------- | ------------------------------------------------ |
| Extending built-ins               | `class MyArray extends Array`                    |
| Return type from built-in methods | Uses `this.constructor` by default               |
| Customize return type             | Use `Symbol.species` static getter               |
| Static inheritance                | Custom classes inherit statics; built-ins do not |

**Key Points:**

- Extended built-in classes return instances of the same class from methods like `filter`, `map`
- Use `Symbol.species` to change this behavior
- Built-in static methods are not inherited between built-in classes
- Only instance methods are inherited via the prototype chain
