# Automated Testing with Mocha

## Why Automated Testing?

Manual testing is error-prone. When you fix one thing, you may break another without noticing. **Automated tests** run after every change, ensuring everything still works.

## Behavior Driven Development (BDD)

BDD combines **tests**, **documentation**, and **examples** in one.

### Spec Structure

```javascript
describe("pow", function () {
  it("raises to n-th power", function () {
    assert.equal(pow(2, 3), 8);
  });
});
```

| Component  | Purpose                                   |
| ---------- | ----------------------------------------- |
| `describe` | Groups related tests (what functionality) |
| `it`       | Describes a specific use case             |
| `assert`   | Checks if result matches expected         |

## Development Flow

1. Write initial spec (failing tests)
2. Create minimal implementation
3. Run tests – see failures
4. Fix until all tests pass
5. Add more tests (they fail)
6. Improve implementation
7. Repeat until complete

## Testing Libraries Used

| Library   | Role                                                   |
| --------- | ------------------------------------------------------ |
| **Mocha** | Core framework – provides `describe`, `it`, runs tests |
| **Chai**  | Assertions – `assert.equal`, `assert.isNaN`, etc.      |
| **Sinon** | Spying, mocking (advanced)                             |

## Writing Tests

### One Test = One Assertion

**Better:**

```javascript
it("2 raised to power 3 is 8", function () {
  assert.equal(pow(2, 3), 8);
});

it("3 raised to power 4 is 81", function () {
  assert.equal(pow(3, 4), 81);
});
```

**Avoid:** Multiple assertions in one test (if first fails, you won't see others).

### Generating Tests Dynamically

```javascript
describe("pow", function () {
  function makeTest(x) {
    let expected = x * x * x;
    it(`${x} in power 3 is ${expected}`, function () {
      assert.equal(pow(x, 3), expected);
    });
  }

  for (let x = 1; x <= 5; x++) {
    makeTest(x);
  }
});
```

### Nested `describe` for Grouping

```javascript
describe("pow", function () {
  describe("raises x to power 3", function () {
    // tests for power 3
  });

  describe("raises x to power 4", function () {
    // tests for power 4
  });
});
```

## Hooks: before/after

```javascript
describe("test", function () {
  before(() => alert("Before all tests"));
  after(() => alert("After all tests"));
  beforeEach(() => alert("Before each test"));
  afterEach(() => alert("After each test"));

  it("test 1", () => alert(1));
  it("test 2", () => alert(2));
});
```

**Order:** before → beforeEach → test1 → afterEach → beforeEach → test2 → afterEach → after

## Common Chai Assertions

| Assertion                  | Checks            |
| -------------------------- | ----------------- |
| `assert.equal(a, b)`       | `a == b`          |
| `assert.strictEqual(a, b)` | `a === b`         |
| `assert.notEqual(a, b)`    | `a != b`          |
| `assert.isTrue(value)`     | `value === true`  |
| `assert.isFalse(value)`    | `value === false` |
| `assert.isNaN(value)`      | `isNaN(value)`    |

## Example: Complete `pow` Development

### Step 1 – Initial Spec

```javascript
describe("pow", function () {
  it("raises to n-th power", function () {
    assert.equal(pow(2, 3), 8);
  });
});
```

### Step 2 – Minimal Implementation

```javascript
function pow(x, n) {
  return 8; // Cheat – passes test
}
```

### Step 3 – Expand Spec

```javascript
it("2 raised to power 3 is 8", () => assert.equal(pow(2, 3), 8));
it("3 raised to power 4 is 81", () => assert.equal(pow(3, 4), 81));
```

### Step 4 – Real Implementation

```javascript
function pow(x, n) {
  if (n < 0) return NaN;
  if (Math.round(n) != n) return NaN;

  let result = 1;
  for (let i = 0; i < n; i++) {
    result *= x;
  }
  return result;
}
```

### Step 5 – Add Edge Cases

```javascript
it("for negative n returns NaN", () => assert.isNaN(pow(2, -1)));
it("for non-integer n returns NaN", () => assert.isNaN(pow(2, 1.5)));
```

## Benefits of Automated Testing

- **Tests** – Guarantee code works correctly
- **Docs** – `describe`/`it` titles explain functionality
- **Examples** – Tests show how to use functions
- **Safe refactoring** – Change code with confidence

## Task: What's Wrong?

```javascript
it("Raises x to the power n", function () {
  let x = 5;

  let result = x;
  assert.equal(pow(x, 1), result);

  result *= x;
  assert.equal(pow(x, 2), result);

  result *= x;
  assert.equal(pow(x, 3), result);
});
```

**Problem:** One test checks multiple values. If `pow(5, 1)` fails, we never see results for `pow(5, 2)` or `pow(5, 3)`. Should split into separate `it` blocks.
