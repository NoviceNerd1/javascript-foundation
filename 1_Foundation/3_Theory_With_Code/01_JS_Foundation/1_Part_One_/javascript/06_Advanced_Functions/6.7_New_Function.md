# The "new Function" Syntax

## Overview

`new Function` creates a function from a string at runtime. It's rarely used but essential for dynamic code generation (e.g., receiving code from a server).

## Syntax

```javascript
let func = new Function([arg1, arg2, ..., argN], functionBody);
```

### Examples

```javascript
// With arguments
let sum = new Function("a", "b", "return a + b");
alert(sum(1, 2)); // 3

// Without arguments
let sayHi = new Function('alert("Hello")');
sayHi(); // Hello

// Multiple arguments (comma-separated string also works)
let sum = new Function("a,b", "return a + b");
let sum = new Function("a , b", "return a + b"); // Same
```

## Dynamic Function Creation

```javascript
// Receive code from server
let str = // ... code from server dynamically
let func = new Function(str);
func();
```

## Important: No Closure Access

Functions created with `new Function` have their `[[Environment]]` set to the **global** Lexical Environment, not the current one.

```javascript
function getFunc() {
  let value = "test";

  let func = new Function("alert(value)");
  return func;
}

getFunc()(); // Error: value is not defined
```

Compare with regular function:

```javascript
function getFunc() {
  let value = "test";

  let func = function () {
    alert(value);
  };
  return func;
}

getFunc()(); // "test" (works)
```

## Why This Restriction?

1. **Minifiers** rename local variables (e.g., `userName` → `a`). If `new Function` could access outer variables, it would break because the variable name changed.

2. **Better architecture** – passing values explicitly via parameters is cleaner and avoids hidden dependencies.

## Summary

| Feature         | Description                                               |
| --------------- | --------------------------------------------------------- |
| **Syntax**      | `new Function(args, body)`                                |
| **Arguments**   | Can be separate parameters or comma-separated string      |
| **Closure**     | No access to outer variables (only global)                |
| **Use case**    | Dynamic code from server, templates                       |
| **Alternative** | Regular function declarations/expressions for static code |

**Best Practice:** Use `new Function` only when necessary (dynamic code generation). For static functions, use regular declarations or expressions.
