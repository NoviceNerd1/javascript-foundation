# Developer Console

Code is prone to errors. In browsers, users don't see errors by default. "Developer tools" are embedded in browsers to see errors and get useful information about scripts.

Most developers use **Chrome** or **Firefox** for development because they have the best developer tools.

## Google Chrome

**Open:** Press `F12` (or `Cmd+Opt+J` on Mac)

The developer tools open on the **Console tab** by default, showing:

- **Red-colored error messages** (e.g., unknown command "lalala")
- **Clickable link** to the source file with line number (e.g., `bug.html:12`)
- **Blue `>` symbol** - command line for typing JavaScript commands (press Enter to run)

### Multi-line Input

- Press `Enter` → executes the line
- Press `Shift+Enter` → insert multiple lines (for longer JavaScript code)

## Firefox, Edge, and Others

Most browsers use `F12` to open developer tools. The look and feel is similar across browsers.

## Safari (Mac only, not Windows/Linux)

**Setup required first:**

1. Open Settings → Advanced pane
2. Check "Show Develop menu in menu bar"

**Open console:** `Cmd+Opt+C`

A new top menu item named "Develop" appears with many commands and options.
