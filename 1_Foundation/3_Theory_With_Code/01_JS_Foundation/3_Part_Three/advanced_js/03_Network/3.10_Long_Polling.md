# Long Polling Summary

Long polling is a simple technique for maintaining a persistent connection with the server to receive real-time updates without delays.

## Regular Polling vs Long Polling

### Regular Polling

```javascript
// Request every 10 seconds
setInterval(async () => {
  let response = await fetch("/messages");
  let messages = await response.json();
  display(messages);
}, 10000);
```

**Problems:**

- Message delay up to 10 seconds
- Unnecessary requests when no data
- Server load even when user inactive

### Long Polling

```javascript
async function subscribe() {
  let response = await fetch("/subscribe");

  if (response.status == 502) {
    // Connection timeout - reconnect
    await subscribe();
  } else if (response.status != 200) {
    // Error - wait and retry
    await new Promise((resolve) => setTimeout(resolve, 1000));
    await subscribe();
  } else {
    // Got message - display and reconnect
    let message = await response.text();
    display(message);
    await subscribe(); // Immediately get next message
  }
}

subscribe();
```

**How it works:**

1. Client sends request to server
2. Server holds connection open until message available
3. Server responds with message, closes connection
4. Client immediately sends new request

## Complete Chat Example

### Client (browser.js)

```javascript
async function subscribe() {
  try {
    let response = await fetch("/subscribe");

    if (response.status === 502) {
      // Connection timeout, reconnect
      await subscribe();
    } else if (response.status !== 200) {
      console.log("Error:", response.statusText);
      await new Promise((r) => setTimeout(r, 1000));
      await subscribe();
    } else {
      let message = await response.text();
      displayMessage(message);
      await subscribe();
    }
  } catch (err) {
    console.log("Connection lost, reconnecting...");
    await new Promise((r) => setTimeout(r, 1000));
    await subscribe();
  }
}

function displayMessage(msg) {
  let div = document.createElement("div");
  div.textContent = msg;
  document.getElementById("messages").append(div);
}

async function sendMessage() {
  let input = document.getElementById("messageInput");
  let message = input.value;
  input.value = "";

  await fetch("/publish", {
    method: "POST",
    body: message,
  });
}

// Start polling
subscribe();
```

### Server (Node.js)

```javascript
const express = require("express");
const app = express();
const http = require("http");
const server = http.createServer(app);

// Store waiting clients
let clients = [];
let messages = [];

app.use(express.static("public"));

// Long polling endpoint
app.get("/subscribe", (req, res) => {
  // If there are pending messages, send immediately
  if (messages.length > 0) {
    res.send(messages.shift());
    return;
  }

  // Otherwise, store client and wait
  clients.push(res);

  // Timeout after 30 seconds to prevent hanging
  req.on("close", () => {
    clients = clients.filter((client) => client !== res);
  });

  res.setTimeout(30000, () => {
    res.status(502).end();
    clients = clients.filter((client) => client !== res);
  });
});

// Publish endpoint
app.post("/publish", express.text(), (req, res) => {
  let message = req.body;

  // Send to all waiting clients
  if (clients.length > 0) {
    clients.forEach((client) => client.send(message));
    clients = [];
  } else {
    // Store for next subscribers
    messages.push(message);
  }

  res.send("ok");
});

server.listen(3000);
```

## Handling Edge Cases

### Connection Timeout (502)

```javascript
if (response.status == 502) {
  // Server or proxy closed connection after timeout
  await subscribe(); // Just reconnect
}
```

### Network Errors

```javascript
try {
  let response = await fetch("/subscribe");
  // Handle response...
} catch (err) {
  // Network failure, reconnect after delay
  await new Promise((r) => setTimeout(r, 1000));
  await subscribe();
}
```

### Server Errors

```javascript
if (response.status != 200 && response.status != 502) {
  // Server error, wait before retry
  await new Promise((r) => setTimeout(r, 1000));
  await subscribe();
}
```

## Long Polling vs Alternatives

| Method                 | Pros                                  | Cons                                     |
| ---------------------- | ------------------------------------- | ---------------------------------------- |
| **Long Polling**       | Simple to implement, works everywhere | Overhead per message, many connections   |
| **WebSocket**          | Full-duplex, low overhead             | Requires WebSocket support, more complex |
| **Server-Sent Events** | Simple, automatic reconnect           | One-way only (server→client)             |
| **Regular Polling**    | Very simple                           | Delay, unnecessary requests              |

## When to Use Long Polling

**Good for:**

- Rare messages (notifications, alerts)
- Simple applications
- Environments without WebSocket support
- Quick implementation

**Not ideal for:**

- High-frequency messages
- Real-time games
- Applications with thousands of simultaneous connections

## Server Considerations

- Server must handle many pending connections
- Avoid per-connection process models (PHP, Ruby default config)
- Use asynchronous servers (Node.js, event-driven Python/Java)
- Set appropriate timeouts (30-60 seconds typical)

## Summary

1. **Client** sends request, waits for response
2. **Server** holds connection until message available
3. **Message arrives** → server responds, connection closes
4. **Client** immediately sends new request
5. **Handle timeouts** (502) with immediate reconnect
6. **Handle errors** with delay and retry
7. **Server** must handle many pending connections efficiently
