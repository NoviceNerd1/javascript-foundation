# Comparisons

## Comparison Operators

| Operator    | Meaning                       |
| ----------- | ----------------------------- |
| `>` , `<`   | Greater/less than             |
| `>=` , `<=` | Greater/less than or equals   |
| `==`        | Equality test (double equals) |
| `!=`        | Not equals                    |

> **Note:** Single `=` is assignment, not comparison.

## Boolean Result

All comparisons return `true` or `false`.

```javascript
alert(2 > 1); // true
alert(2 == 1); // false
alert(2 != 1); // true

let result = 5 > 4;
alert(result); // true
```

## String Comparison

Uses "dictionary" or "lexicographical" order - letter by letter.

```javascript
alert("Z" > "A"); // true
alert("Glow" > "Glee"); // true (G=G, l=l, o>e)
alert("Bee" > "Be"); // true (Be=Be, longer string is greater)
```

**Algorithm:**

1. Compare first character of both strings
2. If different → result determined
3. If same → compare next characters
4. Repeat until end of either string
5. If both end at same length → equal. Otherwise, longer string is greater

> **Important:** Case matters. Lowercase 'a' is greater than uppercase 'A' because of Unicode character indices.

## Comparison of Different Types

JavaScript converts values to numbers.

```javascript
alert("2" > 1); // true (string '2' → number 2)
alert("01" == 1); // true (string '01' → number 1)

alert(true == 1); // true (true → 1)
alert(false == 0); // true (false → 0)
```

### A Funny Consequence

```javascript
let a = 0;
alert(Boolean(a)); // false

let b = "0";
alert(Boolean(b)); // true

alert(a == b); // true! (because both become 0 numerically)
```

## Strict Equality (`===`)

Checks equality **without type conversion**. If types differ, returns `false` immediately.

```javascript
alert(0 === false); // false (number vs boolean)
alert(0 == false); // true (with type conversion)

alert(null === undefined); // false (different types)
alert(null == undefined); // true (special rule)
```

**Strict non-equality:** `!==`

## Comparing `null` and `undefined`

### Strict equality (`===`)

```javascript
alert(null === undefined); // false (different types)
```

### Non-strict equality (`==`)

```javascript
alert(null == undefined); // true (special rule - equal to each other but nothing else)
```

### Mathematical comparisons (`>`, `<`, `>=`, `<=`)

- `null` → converted to `0`
- `undefined` → converted to `NaN`

### Strange Result: `null` vs `0`

```javascript
alert(null > 0); // false (null → 0, 0 > 0? false)
alert(null == 0); // false (equality works differently)
alert(null >= 0); // true  (null → 0, 0 >= 0? true)
```

**Why?** Comparisons (`>`, `<`, `>=`, `<=`) and equality (`==`) work differently:

- Comparisons convert `null` to number (`0`)
- Equality has a special rule for `null`/`undefined`

### Incomparable `undefined`

```javascript
alert(undefined > 0); // false (undefined → NaN)
alert(undefined < 0); // false (NaN always returns false)
alert(undefined == 0); // false (undefined only equals null/undefined)
```

## Summary

| Comparison Type       | Behavior                                                                |
| --------------------- | ----------------------------------------------------------------------- |
| Returns boolean       | Always `true` or `false`                                                |
| Strings               | Letter-by-letter (Unicode order)                                        |
| Different types       | Converted to numbers                                                    |
| Strict equality `===` | No type conversion                                                      |
| `null` & `undefined`  | `==` equals each other only; `null` → `0` for math; `undefined` → `NaN` |

## Best Practice to Avoid Problems

1. Treat any comparison with `null`/`undefined` (except strict equality `===`) with care
2. Don't use `>=`, `>`, `<`, `<=` with variables that may be `null`/`undefined` unless certain
3. Check for `null`/`undefined` separately when needed
