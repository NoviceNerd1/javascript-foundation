# Anchors: String Start `^` and End `$` Summary

Anchors `^` and `$` are zero-width tests that match positions, not characters. `^` matches at the beginning of text, `$` matches at the end.

## Basic Usage

```javascript
// ^ matches start of string
let str1 = "Mary had a little lamb";
console.log(/^Mary/.test(str1)); // true (starts with Mary)

// $ matches end of string
let str2 = "its fleece was white as snow";
console.log(/snow$/.test(str2)); // true (ends with snow)
```

## Testing Full String Match

Use both anchors to ensure the entire string matches a pattern:

```javascript
// Check for time format HH:MM
let regexp = /^\d\d:\d\d$/;

console.log(regexp.test("12:34")); // true
console.log(regexp.test("12:345")); // false (extra character)
console.log(regexp.test(" 12:34")); // false (leading space)
console.log(regexp.test("12:34 ")); // false (trailing space)
```

## Real-World Examples

### Validate Time Format

```javascript
function isValidTime(str) {
  return /^([01]\d|2[0-3]):[0-5]\d$/.test(str);
}

console.log(isValidTime("09:30")); // true
console.log(isValidTime("23:59")); // true
console.log(isValidTime("24:00")); // false
```

### Validate Email (Basic)

```javascript
function isValidEmail(email) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}

console.log(isValidEmail("user@example.com")); // true
console.log(isValidEmail("user@example")); // false
console.log(isValidEmail("user@example.")); // false
```

### Validate Username

```javascript
function isValidUsername(username) {
  // Letters, numbers, underscore, 3-16 characters
  return /^[a-zA-Z0-9_]{3,16}$/.test(username);
}

console.log(isValidUsername("john_123")); // true
console.log(isValidUsername("jo")); // false (too short)
console.log(isValidUsername("user@name")); // false (invalid char)
```

### Validate Hex Color

```javascript
function isValidHexColor(color) {
  return /^#([0-9A-Fa-f]{3}|[0-9A-Fa-f]{6})$/.test(color);
}

console.log(isValidHexColor("#FFF")); // true
console.log(isValidHexColor("#123456")); // true
console.log(isValidHexColor("#1234")); // false
```

## Anchors Are Zero-Width

Anchors do not consume characters, they only test position:

```javascript
let str = "abc";
let regexp = /^abc$/;

// Each character is matched exactly once:
// ^ (position before 'a')
// a (consumes 'a')
// b (consumes 'b')
// c (consumes 'c')
// $ (position after 'c')
```

## With `m` Flag (Multiline)

Anchors work per line when `m` flag is used (covered in next chapter):

```javascript
let str = "1st line\n2nd line\n3rd line";

// Without m - only start of entire string
console.log(str.match(/^\d/g)); // ['1']

// With m - start of each line
console.log(str.match(/^\d/gm)); // ['1', '2', '3']
```

## Common Patterns

| Pattern | Description                 |
| ------- | --------------------------- |
| `^...`  | String starts with ...      |
| `...$`  | String ends with ...        |
| `^...$` | Exact match (entire string) |
| `^$`    | Empty string                |

## Task: Regexp `^$`

**Which string matches the pattern `^$`?**

Answer: An empty string.

```javascript
console.log(/^$/.test("")); // true
console.log(/^$/.test(" ")); // false (contains space)
console.log(/^$/.test("a")); // false
console.log(/^$/.test("\n")); // false (contains newline)
```

## Summary

| Anchor  | Position                      |
| ------- | ----------------------------- |
| `^`     | Start of string               |
| `$`     | End of string                 |
| `^...$` | Entire string matches exactly |

**Key Points:**

- Anchors are zero-width (test positions, not characters)
- `^...$` ensures the whole string matches the pattern
- Without `m` flag, anchors work on entire string
- Use for validation, format checking, exact matching
