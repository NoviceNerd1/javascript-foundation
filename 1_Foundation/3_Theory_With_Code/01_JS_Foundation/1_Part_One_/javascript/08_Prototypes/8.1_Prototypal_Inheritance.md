# Prototypal Inheritance

## Overview

Every JavaScript object has a hidden `[[Prototype]]` property that references another object (or `null`). When a property is missing, JavaScript automatically looks for it in the prototype – this is **prototypal inheritance**.

## Setting the Prototype

```javascript
let animal = { eats: true };
let rabbit = { jumps: true };

rabbit.__proto__ = animal; // Set prototype

alert(rabbit.eats); // true (inherited)
alert(rabbit.jumps); // true (own property)
```

## Prototype Chain

```javascript
let animal = {
  eats: true,
  walk() {
    alert("Animal walk");
  },
};

let rabbit = {
  jumps: true,
  __proto__: animal,
};

let longEar = {
  earLength: 10,
  __proto__: rabbit,
};

longEar.walk(); // Animal walk (from animal)
alert(longEar.jumps); // true (from rabbit)
```

## Reading vs Writing

- **Reading:** Looks up prototype chain if property not found
- **Writing/Deleting:** Works directly on the object (doesn't use prototype)

```javascript
let animal = { eats: true };
let rabbit = { __proto__: animal };

rabbit.eats = false; // Creates own property, doesn't modify animal
alert(rabbit.eats); // false (own)
alert(animal.eats); // true (unchanged)
```

## The `this` Value

`this` is always the object before the dot, not the prototype where the method was found.

```javascript
let animal = {
  sleep() {
    this.isSleeping = true; // this = rabbit
  },
};

let rabbit = {
  name: "White Rabbit",
  __proto__: animal,
};

rabbit.sleep();
alert(rabbit.isSleeping); // true
alert(animal.isSleeping); // undefined
```

## `for..in` Loop

Iterates over **both own and inherited** enumerable properties.

```javascript
let animal = { eats: true };
let rabbit = { jumps: true, __proto__: animal };

for (let prop in rabbit) {
  alert(prop); // jumps, then eats
}

// Filter inherited properties
for (let prop in rabbit) {
  let isOwn = rabbit.hasOwnProperty(prop);
  if (isOwn)
    alert(`Own: ${prop}`); // Own: jumps
  else alert(`Inherited: ${prop}`); // Inherited: eats
}
```

## Key Methods

| Method                    | Purpose                                           |
| ------------------------- | ------------------------------------------------- |
| `obj.hasOwnProperty(key)` | Returns `true` if property is own (not inherited) |
| `Object.keys(obj)`        | Returns only own enumerable keys                  |
| `Object.values(obj)`      | Returns only own enumerable values                |

## Summary

| Concept         | Description                                                    |
| --------------- | -------------------------------------------------------------- |
| `[[Prototype]]` | Hidden property referencing prototype object                   |
| `__proto__`     | Getter/setter for prototype (historical, but widely supported) |
| Inheritance     | Property lookup goes up the prototype chain                    |
| Write/delete    | Affects object directly, not prototype                         |
| `this`          | Always the object before the dot                               |
| `for..in`       | Includes inherited enumerable properties                       |

## Tasks

### 1. Working with Prototype

```javascript
let animal = { jumps: null };
let rabbit = { __proto__: animal, jumps: true };

alert(rabbit.jumps); // true (own property)
delete rabbit.jumps;
alert(rabbit.jumps); // null (from animal)
delete animal.jumps;
alert(rabbit.jumps); // undefined (no longer exists)
```

### 2. Searching Algorithm

```javascript
let head = { glasses: 1 };
let table = { pen: 3, __proto__: head };
let bed = { sheet: 1, pillow: 2, __proto__: table };
let pockets = { money: 2000, __proto__: bed };

alert(pockets.pen); // 3 (from table)
alert(bed.glasses); // 1 (from head)

// Performance: Modern engines optimize; accessing directly vs via chain
// has negligible difference. head.glasses is theoretically faster,
// but the optimization is so good it's practically the same.
```

### 3. Where Does It Write?

```javascript
let animal = {
  eat() {
    this.full = true;
  },
};

let rabbit = { __proto__: animal };

rabbit.eat();
alert(rabbit.full); // true (this = rabbit)
alert(animal.full); // undefined (unchanged)
```

### 4. Why Are Both Hamsters Full?

```javascript
// Problem: stomach array is shared via prototype
let hamster = { stomach: [] };
let speedy = { __proto__: hamster };
let lazy = { __proto__: hamster };

speedy.eat("apple");
alert(lazy.stomach); // "apple" (shared!)

// Fix: Give each hamster its own stomach
let hamster = { stomach: [] };
let speedy = { __proto__: hamster, stomach: [] };
let lazy = { __proto__: hamster, stomach: [] };

speedy.eat("apple");
alert(speedy.stomach); // ["apple"]
alert(lazy.stomach); // [] (empty)
```
