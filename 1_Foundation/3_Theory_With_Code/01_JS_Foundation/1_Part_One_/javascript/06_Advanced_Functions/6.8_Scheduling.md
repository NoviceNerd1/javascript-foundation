# Scheduling: setTimeout and setInterval

## Overview

JavaScript allows scheduling function execution at a later time using two methods:

- **`setTimeout`** – runs once after a delay
- **`setInterval`** – runs repeatedly at intervals

## setTimeout

### Syntax

```javascript
let timerId = setTimeout(func|code, [delay], [arg1], [arg2], ...);
```

### Examples

```javascript
// Basic
function sayHi() {
  alert("Hello");
}
setTimeout(sayHi, 1000);

// With arguments
function sayHi(phrase, who) {
  alert(phrase + ", " + who);
}
setTimeout(sayHi, 1000, "Hello", "John"); // Hello, John

// Arrow function
setTimeout(() => alert("Hello"), 1000);
```

### Important: Don't Call the Function

```javascript
// Wrong – executes immediately
setTimeout(sayHi(), 1000);

// Correct – passes reference
setTimeout(sayHi, 1000);
```

### Canceling with clearTimeout

```javascript
let timerId = setTimeout(() => alert("never happens"), 1000);
clearTimeout(timerId); // Cancels execution
```

## setInterval

Runs repeatedly at specified intervals.

```javascript
let timerId = setInterval(() => alert("tick"), 2000);

// Stop after 5 seconds
setTimeout(() => {
  clearInterval(timerId);
  alert("stop");
}, 5000);
```

## Nested setTimeout

A more flexible alternative to `setInterval`:

```javascript
let timerId = setTimeout(function tick() {
  alert("tick");
  timerId = setTimeout(tick, 2000); // Schedule next call
}, 2000);
```

### Advantages Over setInterval

- Sets delay **between** executions (not from start)
- Can adjust delay dynamically based on previous execution

```javascript
let delay = 5000;
let timerId = setTimeout(function request() {
  // send request...
  if (failed) delay *= 2; // Increase delay on failure
  timerId = setTimeout(request, delay);
}, delay);
```

### Delay Comparison

**setInterval:** Delay starts counting from function start

```
func run → [delay] → func run → [delay] → ...
```

**Nested setTimeout:** Delay starts from function end

```
[delay] → func run → [delay] → func run → ...
```

## Zero Delay setTimeout

```javascript
setTimeout(() => alert("World"));
alert("Hello");
// Output: Hello, then World
```

Executes after current script completes, not immediately (minimum delay ~4ms after 5 nested calls in browsers).

## Garbage Collection

Scheduled functions are kept in memory, preventing garbage collection of their outer lexical environment.

```javascript
// Function stays in memory until executed/canceled
setTimeout(function() {...}, 100);
```

**Best practice:** Cancel timers when no longer needed.

## Browser Limitations

After 5 nested timers, browsers enforce a **minimum 4ms delay**.

```javascript
let start = Date.now();
let times = [];

setTimeout(function run() {
  times.push(Date.now() - start);
  if (start + 100 < Date.now()) alert(times);
  else setTimeout(run);
});
// Output shows delays: 1,1,1,1,9,15,20...
```

## Summary

| Method                              | Purpose                            | Cancel              |
| ----------------------------------- | ---------------------------------- | ------------------- |
| `setTimeout(func, delay, ...args)`  | Run once after delay               | `clearTimeout(id)`  |
| `setInterval(func, delay, ...args)` | Run repeatedly                     | `clearInterval(id)` |
| Nested `setTimeout`                 | Run repeatedly with dynamic delays | `clearTimeout(id)`  |

**Key Points:**

- Delay is in milliseconds (1000ms = 1 second)
- Functions are referenced, not called with `()`
- Zero delay runs after current script completes
- Browser limits nested timers to minimum 4ms
- Cancel timers to free memory

## Tasks

### 1. Output Every Second

```javascript
// Using setInterval
function printNumbers(from, to) {
  let current = from;
  let timerId = setInterval(() => {
    alert(current);
    if (current === to) clearInterval(timerId);
    current++;
  }, 1000);
}

// Using nested setTimeout
function printNumbers(from, to) {
  let current = from;
  setTimeout(function run() {
    alert(current);
    if (current < to) setTimeout(run, 1000);
    current++;
  }, 1000);
}
```

### 2. What Will setTimeout Show?

```javascript
let i = 0;
setTimeout(() => alert(i), 100); // Shows 100000000

// Heavy computation takes >100ms
for (let j = 0; j < 100000000; j++) {
  i++;
}
// setTimeout runs after loop completes
```
