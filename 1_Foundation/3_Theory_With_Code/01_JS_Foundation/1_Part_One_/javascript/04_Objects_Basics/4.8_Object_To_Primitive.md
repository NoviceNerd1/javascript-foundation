# Object to Primitive Conversion

## Overview

When objects are used in operations like `+`, `-`, or `alert()`, JavaScript automatically converts them to primitives.

> **Important:** The result of `obj1 + obj2` cannot be another object â€“ it will be a primitive value.

## Conversion Rules

| Hint        | When Used                                                                      |
| ----------- | ------------------------------------------------------------------------------ |
| `"string"`  | `alert(obj)`, using object as property key                                     |
| `"number"`  | Maths operations: `Number(obj)`, `+obj`, `date1 - date2`, comparisons `>`, `<` |
| `"default"` | Binary `+`, `==` comparison (rare; usually same as `"number"`)                 |

**No boolean conversion:** All objects are `true` in boolean contexts.

## Conversion Methods (in priority order)

### 1. `Symbol.toPrimitive` (Preferred)

```javascript
let user = {
  name: "John",
  money: 1000,

  [Symbol.toPrimitive](hint) {
    alert(`hint: ${hint}`);
    return hint == "string" ? `{name: "${this.name}"}` : this.money;
  },
};

alert(user); // hint: string â†’ {name: "John"}
alert(+user); // hint: number â†’ 1000
alert(user + 500); // hint: default â†’ 1500
```

### 2. `toString()` and `valueOf()` (Fallback)

| Hint                     | Priority                   |
| ------------------------ | -------------------------- |
| `"string"`               | `toString()` â†’ `valueOf()` |
| `"number"` / `"default"` | `valueOf()` â†’ `toString()` |

```javascript
let user = {
  name: "John",
  money: 1000,

  toString() {
    return `{name: "${this.name}"}`;
  },
  valueOf() {
    return this.money;
  },
};

alert(user); // toString â†’ {name: "John"}
alert(+user); // valueOf â†’ 1000
alert(user + 500); // valueOf â†’ 1500
```

### Default `toString()` and `valueOf()`

```javascript
let user = { name: "John" };
alert(user); // [object Object] (default toString)
alert(user.valueOf() === user); // true (default valueOf returns object itself)
```

### Catch-All with `toString()` Only

```javascript
let user = {
  name: "John",
  toString() {
    return this.name;
  },
};

alert(user); // John
alert(user + 500); // John500
```

## Conversion Stages

1. **Object â†’ Primitive** (using methods above)
2. **Primitive â†’ Required Type** (if needed)

```javascript
let obj = {
  toString() {
    return "2";
  },
};

alert(obj * 2); // 4 (string "2" â†’ number 2)
alert(obj + 2); // "22" (string "2" + 2 â†’ concatenation)
```

## Key Points

| Rule                      | Description                                                |
| ------------------------- | ---------------------------------------------------------- |
| Must return **primitive** | Returning object causes error (or ignored for old methods) |
| `Symbol.toPrimitive`      | Handles all hints, most modern approach                    |
| `toString`                | Good for human-readable logging                            |
| `valueOf`                 | Used for maths operations                                  |
| All objects are `true`    | No boolean conversion for objects                          |

## Summary

- **Prefer `Symbol.toPrimitive`** for full control
- **Use `toString()`** as simple catch-all for debugging
- Methods must return **primitives**, not objects
- Conversion happens automatically in operators and built-in functions
