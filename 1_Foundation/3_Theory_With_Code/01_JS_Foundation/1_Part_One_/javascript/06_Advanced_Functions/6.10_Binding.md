# Function Binding

## The Problem: Losing `this`

When passing object methods as callbacks, `this` is lost.

```javascript
let user = {
  firstName: "John",
  sayHi() {
    alert(`Hello, ${this.firstName}!`);
  },
};

setTimeout(user.sayHi, 1000); // Hello, undefined!
// Equivalent to: let f = user.sayHi; setTimeout(f, 1000);
```

## Solution 1: Wrapper Function

```javascript
setTimeout(function () {
  user.sayHi(); // Hello, John!
}, 1000);

// Arrow function version
setTimeout(() => user.sayHi(), 1000);
```

**Vulnerability:** If `user` changes before the call, wrong object is used.

```javascript
setTimeout(() => user.sayHi(), 1000);
user = {
  sayHi() {
    alert("Another user!");
  },
};
// "Another user!" after 1 second
```

## Solution 2: `bind`

`func.bind(context)` returns a bound function with fixed `this`.

```javascript
let user = {
  firstName: "John",
};

function func() {
  alert(this.firstName);
}

let funcUser = func.bind(user);
funcUser(); // John

// With arguments
function func(phrase) {
  alert(phrase + ", " + this.firstName);
}
let funcUser = func.bind(user);
funcUser("Hello"); // Hello, John
```

### For Object Methods

```javascript
let user = {
  firstName: "John",
  sayHi() {
    alert(`Hello, ${this.firstName}!`);
  },
};

let sayHi = user.sayHi.bind(user);
sayHi(); // Hello, John!
setTimeout(sayHi, 1000); // Hello, John! (works even if user changes)
```

### Bind All Methods

```javascript
for (let key in user) {
  if (typeof user[key] == "function") {
    user[key] = user[key].bind(user);
  }
}
```

## Partial Functions

`bind` can also fix arguments:

```javascript
let bound = func.bind(context, [arg1], [arg2], ...);
```

```javascript
function mul(a, b) {
  return a * b;
}

let double = mul.bind(null, 2);
alert(double(3)); // 6
alert(double(4)); // 8

let triple = mul.bind(null, 3);
alert(triple(3)); // 9
```

### Manual Partial (without binding `this`)

```javascript
function partial(func, ...argsBound) {
  return function (...args) {
    return func.call(this, ...argsBound, ...args);
  };
}

let user = {
  firstName: "John",
  say(time, phrase) {
    alert(`[${time}] ${this.firstName}: ${phrase}`);
  },
};

user.sayNow = partial(
  user.say,
  new Date().getHours() + ":" + new Date().getMinutes(),
);
user.sayNow("Hello"); // [10:00] John: Hello
```

## Summary

| Method                          | Purpose                                   |
| ------------------------------- | ----------------------------------------- |
| `func.bind(context)`            | Fixes `this` to `context`                 |
| `func.bind(context, arg1, ...)` | Fixes `this` AND arguments                |
| Partial application             | Creating a function with preset arguments |

**Key Points:**

- `bind` creates a new function that calls the original with fixed `this`
- Once bound, `this` cannot be changed by another `bind`
- Use `bind` when passing object methods as callbacks
- Partial functions are useful for creating specialized versions of generic functions

## Tasks

### 1. Bound Function as a Method

```javascript
function f() {
  alert(this);
}
let user = { g: f.bind(null) };
user.g(); // null (bind fixed this to null)
```

### 2. Second Bind

```javascript
function f() {
  alert(this.name);
}
f = f.bind({ name: "John" }).bind({ name: "Ann" });
f(); // John (only first bind matters)
```

### 3. Function Property After Bind

```javascript
function sayHi() {
  alert(this.name);
}
sayHi.test = 5;
let bound = sayHi.bind({ name: "John" });
alert(bound.test); // undefined (bind returns new function, test not copied)
```

### 4. Fix Function That Loses "this"

```javascript
// Problem:
askPassword(user.loginOk, user.loginFail); // this = undefined

// Fix with bind:
askPassword(user.loginOk.bind(user), user.loginFail.bind(user));

// Or with wrapper:
askPassword(
  () => user.loginOk(),
  () => user.loginFail(),
);
```

### 5. Partial Application for Login

```javascript
askPassword(
  () => user.login(true),
  () => user.login(false),
);

// Or using bind:
askPassword(user.login.bind(user, true), user.login.bind(user, false));
```
