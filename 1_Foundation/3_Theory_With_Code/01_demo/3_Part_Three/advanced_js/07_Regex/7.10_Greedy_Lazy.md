# Greedy and Lazy Quantifiers Summary

Quantifiers are **greedy by default**—they match as many characters as possible. Adding `?` after a quantifier makes it **lazy**—matching as few characters as possible.

## Greedy Search (Default)

The regexp engine:

1. Takes as many characters as possible
2. If the rest of pattern fails, backtracks (reduces count) and retries

```javascript
// Greedy: finds ONE match covering both quotes
let regexp = /".+"/g;
let str = 'a "witch" and her "broom" is one';
console.log(str.match(regexp)); // ["witch" and her "broom"]
```

### How Greedy Works

```
Step 1: " (found at position 3)
Step 2: .+ consumes all remaining characters (greedy)
Step 3: " (no match at end, backtrack)
Step 4: Reduce .+ by 1 character, check for " → repeat until found
Result: "witch" and her "broom"
```

## Lazy Search (Add `?`)

Add `?` after quantifier: `+?`, `*?`, `??`, `{n,m}?`

```javascript
// Lazy: finds each quote separately
let regexp = /".+?"/g;
let str = 'a "witch" and her "broom" is one';
console.log(str.match(regexp)); // ["witch", "broom"]
```

### How Lazy Works

```
Step 1: " (found at position 3)
Step 2: .+? takes minimal (1 character), try to match "
Step 3: If " not found, increase .+? by 1 character
Step 4: Repeat until " found
Result: "witch"
Then continue from end of match → "broom"
```

## Lazy Quantifier Examples

```javascript
// Lazy vs greedy with digits
console.log("123 456".match(/\d+ \d+?/g)); // ["123 4"]
// \d+ (greedy) → "123"
// space matches
// \d+? (lazy) → "4" (minimal)
```

```javascript
// Lazy star
console.log("aab".match(/a*?b/)); // ["aab"]? Actually "b"? Let's check
console.log("aab".match(/a*?b/)); // ["aab"]? Wait, lazy a*? matches 0 first, then b? No b at start.
// Actually: a*? (lazy) matches 0 a's, then b fails → try 1 a, then b → "ab"? Let's test.
```

## Alternative: Excluding Set

Often a better solution than lazy quantifiers:

```javascript
// Using negated set instead of lazy
let regexp = /"[^"]+"/g;
let str = 'a "witch" and her "broom" is one';
console.log(str.match(regexp)); // ["witch", "broom"]
```

## Real-World Example: HTML Tags

### Problem: Greedy matches too much

```javascript
let str = '<a href="link1" class="doc">... <a href="link2" class="doc">';
let greedy = /<a href=".*" class="doc">/g;
console.log(str.match(greedy));
// ONE match: '<a href="link1" class="doc">... <a href="link2" class="doc">'
```

### Problem: Lazy also fails with nested/incorrect structure

```javascript
let str = '<a href="link1" class="wrong">... <p style="" class="doc">';
let lazy = /<a href=".*?" class="doc">/g;
console.log(str.match(lazy));
// WRONG: matches across tags: '<a href="link1" class="wrong">... <p style="" class="doc">'
```

### Solution: Excluding set `[^"]*`

```javascript
let correct = /<a href="[^"]*" class="doc">/g;
let str1 = '<a href="link1" class="wrong">... <p style="" class="doc">';
let str2 = '<a href="link1" class="doc">... <a href="link2" class="doc">';

console.log(str1.match(correct)); // null (correct, no match)
console.log(str2.match(correct));
// ['<a href="link1" class="doc">', '<a href="link2" class="doc">']
```

## Quantifier Comparison

| Mode   | Syntax                     | Behavior                  |
| ------ | -------------------------- | ------------------------- |
| Greedy | `+`, `*`, `?`, `{n,m}`     | Match as many as possible |
| Lazy   | `+?`, `*?`, `??`, `{n,m}?` | Match as few as possible  |

## Common Patterns

| Pattern   | Matches                   | Type                    |
| --------- | ------------------------- | ----------------------- |
| `".*"`    | `"witch" and her "broom"` | Greedy (bad for quotes) |
| `".*?"`   | `"witch"`, `"broom"`      | Lazy (works)            |
| `"[^"]*"` | `"witch"`, `"broom"`      | Excluding set (better)  |

## Tasks

### Task 1: `/d+? d+?/`

```javascript
console.log("123 456".match(/\d+? \d+?/g)); // ["123 4"]
// \d+? → lazy, takes "1" then space? No space after "1".
// Actually: first \d+? takes "1", tries space → fails? Let's simulate.
// Better: greedy \d+ takes "123", space matches, lazy \d+? takes "4"
```

### Task 2: Find HTML Comments

```javascript
let regexp = /<!--.*?-->/gs; // 's' flag for dot to match newline
let str = `... <!-- My -- comment
 test --> ..  <!----> ..
`;
console.log(str.match(regexp)); // ['<!-- My -- comment \n test -->', '<!---->']
```

### Task 3: Find HTML Tags

```javascript
let regexp = /<[^<>]+>/g;
let str = '<> <a href="/"> <input type="radio" checked> <b>';
console.log(str.match(regexp)); // ['<a href="/">', '<input type="radio" checked>', '<b>']
```

## Summary

| Concept                | Key Point                                                      |
| ---------------------- | -------------------------------------------------------------- |
| **Greedy**             | Default. Matches maximum, backtracks if needed                 |
| **Lazy**               | Add `?` after quantifier. Matches minimum                      |
| **Backtracking**       | Greedy reduces repetitions when rest of pattern fails          |
| **Excluding sets**     | Often better than lazy: `[^"]*` instead of `.*?` for quotes    |
| **When to use lazy**   | When you want minimal match but can't predict delimiter        |
| **When to avoid lazy** | When pattern has nested structures; use excluding sets instead |

**Key Principle:** Greedy quantifiers are powerful but can overreach. Lazy quantifiers are a tool, but not always the best. Excluding sets (`[^...]*`) often provide a cleaner solution.
