# Error Handling with Promises

## Overview

Promise chains have built-in error handling. When a promise rejects, control jumps to the nearest rejection handler (`.catch`). This works like `try..catch` for asynchronous code.

## Basic Error Handling

```javascript
fetch("https://no-such-server.blabla") // rejects
  .then((response) => response.json())
  .catch((err) => alert(err)); // TypeError: failed to fetch
```

A single `.catch` at the end handles errors from any previous promise in the chain:

```javascript
fetch("/user.json")
  .then((response) => response.json())
  .then((user) => fetch(`https://api.github.com/users/${user.name}`))
  .then((response) => response.json())
  .then((githubUser) => {
    // process user
  })
  .catch((error) => alert(error.message)); // Catches ANY error above
```

## Implicit try..catch

Promise executors and handlers have an invisible `try..catch`. Any thrown error becomes a rejection:

```javascript
// These are equivalent:

// Throwing error
new Promise((resolve, reject) => {
  throw new Error("Whoops!");
}).catch(alert); // Error: Whoops!

// Explicit reject
new Promise((resolve, reject) => {
  reject(new Error("Whoops!"));
}).catch(alert); // Error: Whoops!
```

### In .then Handlers

```javascript
new Promise((resolve, reject) => {
  resolve("ok");
})
  .then((result) => {
    throw new Error("Whoops!"); // Rejects the promise
  })
  .catch(alert); // Error: Whoops!

// Programming errors also caught
new Promise((resolve, reject) => {
  resolve("ok");
})
  .then((result) => {
    blabla(); // No such function
  })
  .catch(alert); // ReferenceError: blabla is not defined
```

## Rethrowing

Like `try..catch`, we can rethrow errors that we can't handle:

```javascript
new Promise((resolve, reject) => {
  throw new Error("Whoops!");
})
  .catch(function (error) {
    if (error instanceof URIError) {
      // handle URIError
    } else {
      alert("Can't handle such error");
      throw error; // Rethrow to next catch
    }
  })
  .then(() => {
    /* doesn't run */
  })
  .catch((error) => {
    alert(`Unknown error: ${error}`);
  });
```

## Successful Error Handling

If `.catch` handles the error normally (without throwing), execution continues to the next `.then`:

```javascript
new Promise((resolve, reject) => {
  throw new Error("Whoops!");
})
  .catch((error) => {
    alert("Error handled, continue normally");
  })
  .then(() => alert("Next successful handler runs"));
```

## Unhandled Rejections

If no `.catch` handles a rejection, it becomes an **unhandled rejection**:

```javascript
new Promise(() => {
  noSuchFunction(); // Error, no .catch
}).then(() => {});
// Script doesn't die, but error appears in console
```

### Global Handler (Browser)

```javascript
window.addEventListener("unhandledrejection", function (event) {
  alert(event.promise); // The rejected promise
  alert(event.reason); // The error object
});

new Promise(() => {
  throw new Error("Whoops!");
}); // Triggers unhandledrejection
```

## Summary

| Concept               | Description                                           |
| --------------------- | ----------------------------------------------------- |
| `.catch`              | Handles rejections from any previous promise in chain |
| Implicit `try..catch` | Errors in executor/handlers become rejections         |
| Rethrowing            | Throw in `.catch` to pass error to next handler       |
| Unhandled rejection   | Rejection with no handler – triggers global event     |

**Key Points:**

- Place `.catch` at the end of chain to catch all errors
- Throwing an error in `.then` or executor becomes a rejection
- Re-throw in `.catch` to let it propagate
- Always handle rejections (with `.catch` or global handler)

## Task: Error in setTimeout

```javascript
new Promise(function (resolve, reject) {
  setTimeout(() => {
    throw new Error("Whoops!");
  }, 1000);
}).catch(alert);
```

**Answer:** The `.catch` will **not** trigger.

**Explanation:** The error is thrown inside `setTimeout`, not in the promise executor itself. The executor has already completed before the timeout runs. The error is a regular synchronous error in a callback, not a promise rejection. The promise has already resolved (with no value) and cannot be rejected after it's settled.
