# Custom Errors, Extending Error

## Overview

Custom error classes allow us to represent specific error types in our application. Inheriting from `Error` ensures compatibility with `instanceof` checks and standard error properties.

## Basic Error Class Structure

Built-in `Error` class (pseudocode):

```javascript
class Error {
  constructor(message) {
    this.message = message;
    this.name = "Error";
    this.stack = <call stack>;  // non-standard but widely supported
  }
}
```

## Extending Error

```javascript
class ValidationError extends Error {
  constructor(message) {
    super(message); // Call parent constructor
    this.name = "ValidationError"; // Set proper name
  }
}

function test() {
  throw new ValidationError("Whoops!");
}

try {
  test();
} catch (err) {
  alert(err.message); // Whoops!
  alert(err.name); // ValidationError
  alert(err.stack); // Stack trace
}
```

## Practical Example: Validation Errors

```javascript
class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}

function readUser(json) {
  let user = JSON.parse(json);

  if (!user.age) {
    throw new ValidationError("No field: age");
  }
  if (!user.name) {
    throw new ValidationError("No field: name");
  }

  return user;
}

try {
  let user = readUser('{ "age": 25 }');
} catch (err) {
  if (err instanceof ValidationError) {
    alert("Invalid data: " + err.message); // Invalid data: No field: name
  } else if (err instanceof SyntaxError) {
    alert("JSON Syntax Error: " + err.message);
  } else {
    throw err; // Unknown error, rethrow
  }
}
```

## Creating Error Hierarchy

```javascript
class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}

class PropertyRequiredError extends ValidationError {
  constructor(property) {
    super("No property: " + property);
    this.name = "PropertyRequiredError";
    this.property = property;
  }
}

// Usage
try {
  let user = readUser('{ "age": 25 }');
} catch (err) {
  if (err instanceof PropertyRequiredError) {
    alert(err.message); // No property: name
    alert(err.property); // name
  }
}
```

## Simplifying with Base Error Class

```javascript
class MyError extends Error {
  constructor(message) {
    super(message);
    this.name = this.constructor.name; // Auto-set name
  }
}

class ValidationError extends MyError {}

class PropertyRequiredError extends ValidationError {
  constructor(property) {
    super("No property: " + property);
    this.property = property;
  }
}

alert(new PropertyRequiredError("field").name); // PropertyRequiredError
```

## Wrapping Exceptions

Create a generic error that wraps lower-level errors:

```javascript
class ReadError extends Error {
  constructor(message, cause) {
    super(message);
    this.cause = cause;
    this.name = "ReadError";
  }
}

function readUser(json) {
  let user;
  try {
    user = JSON.parse(json);
  } catch (err) {
    if (err instanceof SyntaxError) {
      throw new ReadError("Syntax Error", err);
    } else {
      throw err;
    }
  }

  try {
    validateUser(user);
  } catch (err) {
    if (err instanceof ValidationError) {
      throw new ReadError("Validation Error", err);
    } else {
      throw err;
    }
  }
}

try {
  readUser("{bad json}");
} catch (e) {
  if (e instanceof ReadError) {
    alert(e); // ReadError: Syntax Error
    alert("Original: " + e.cause); // Original: SyntaxError: ...
  } else {
    throw e;
  }
}
```

## Summary

| Concept             | Description                                                   |
| ------------------- | ------------------------------------------------------------- |
| `extends Error`     | Inherit from built-in Error class                             |
| `super(message)`    | Must call parent constructor                                  |
| `this.name`         | Set to custom error name                                      |
| `instanceof`        | Works with inheritance chain                                  |
| Wrapping exceptions | Catch low-level errors, throw higher-level error with `cause` |

**Key Points:**

- Always call `super(message)` in custom error constructors
- Set `this.name` to the error class name
- Use `instanceof` for error type checking (future-proof)
- Wrap exceptions to simplify error handling at higher levels

## Task: Inherit from SyntaxError

```javascript
class FormatError extends SyntaxError {
  constructor(message) {
    super(message);
    this.name = "FormatError";
  }
}

let err = new FormatError("formatting error");

alert(err.message); // formatting error
alert(err.name); // FormatError
alert(err.stack); // stack trace
alert(err instanceof FormatError); // true
alert(err instanceof SyntaxError); // true
```
