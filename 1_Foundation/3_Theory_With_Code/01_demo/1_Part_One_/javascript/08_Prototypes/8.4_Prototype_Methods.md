# Prototype Methods, Objects Without **proto**

## Modern Prototype Methods

| Method                                | Purpose                                  |
| ------------------------------------- | ---------------------------------------- |
| `Object.getPrototypeOf(obj)`          | Returns `[[Prototype]]` of `obj`         |
| `Object.setPrototypeOf(obj, proto)`   | Sets `[[Prototype]]` of `obj` to `proto` |
| `Object.create(proto[, descriptors])` | Creates new object with given prototype  |

## Using `Object.create`

```javascript
let animal = { eats: true };

// Create object with animal as prototype
let rabbit = Object.create(animal);
alert(rabbit.eats); // true
alert(Object.getPrototypeOf(rabbit) === animal); // true

// With property descriptors
let rabbit = Object.create(animal, {
  jumps: { value: true },
});
alert(rabbit.jumps); // true
```

### Perfect Cloning with `Object.create`

```javascript
let clone = Object.create(
  Object.getPrototypeOf(obj),
  Object.getOwnPropertyDescriptors(obj),
);
// Copies all properties (enumerable/non-enumerable) and correct prototype
```

## The `__proto__` History

| Year    | Event                                                                        |
| ------- | ---------------------------------------------------------------------------- |
| Ancient | `prototype` property of constructor functions                                |
| 2012    | `Object.create` introduced                                                   |
| 2015    | `Object.setPrototypeOf` / `Object.getPrototypeOf` added                      |
| 2022    | `__proto__` in object literals `{ __proto__: ... }` allowed (out of Annex B) |

> **Recommendation:** Use modern methods. `__proto__` as getter/setter is discouraged (slow, breaks optimizations).

## Performance Warning

Changing prototype on existing objects (`Object.setPrototypeOf` or `obj.__proto__ =`) is **slow**. Avoid unless necessary.

## "Very Plain" Objects (Prototype-less)

Regular objects have `__proto__` as an accessor property from `Object.prototype`, which can cause issues when using objects as dictionaries:

```javascript
let obj = {};
let key = "__proto__";
obj[key] = "some value";
alert(obj[key]); // [object Object] (not "some value"!) - assignment ignored
```

### Solution: Create Objects Without Prototype

```javascript
let obj = Object.create(null);
// or: obj = { __proto__: null }

obj["__proto__"] = "some value";
alert(obj["__proto__"]); // "some value" (works as regular property)

// No built-in methods
alert(obj); // Error: no toString
```

### Dictionary with Null Prototype

```javascript
let dict = Object.create(null);
dict.hello = "你好";
dict.bye = "再见";

alert(Object.keys(dict)); // ["hello", "bye"] (Object.keys still works)
```

## Summary

| Method                              | Purpose                                        |
| ----------------------------------- | ---------------------------------------------- |
| `Object.create(proto)`              | Create object with given prototype             |
| `Object.getPrototypeOf(obj)`        | Get prototype                                  |
| `Object.setPrototypeOf(obj, proto)` | Set prototype (slow)                           |
| `{ __proto__: proto }`              | Create object with prototype (literal syntax)  |
| `Object.create(null)`               | Create prototype-less object (pure dictionary) |

**Key Points:**

- Use `Object.create` for setting prototype at creation
- Avoid modifying prototype of existing objects (performance)
- Use `Object.create(null)` for dictionaries to avoid `__proto__` conflicts
- `Object.keys`, `Object.values`, etc. work on prototype-less objects

## Tasks

### 1. Add toString to Dictionary

```javascript
let dictionary = Object.create(null);

dictionary.toString = function () {
  return Object.keys(this).join(",");
};

// Works as expected
dictionary.apple = "Apple";
dictionary.__proto__ = "test";

for (let key in dictionary) {
  alert(key); // "apple", then "__proto__"
}

alert(dictionary); // "apple,__proto__" (toString called)
```

### 2. The Difference Between Calls

```javascript
function Rabbit(name) {
  this.name = name;
}
Rabbit.prototype.sayHi = function () {
  alert(this.name);
};

let rabbit = new Rabbit("Rabbit");

rabbit.sayHi(); // "Rabbit" (this = rabbit)
Rabbit.prototype.sayHi(); // undefined (this = Rabbit.prototype)
Object.getPrototypeOf(rabbit).sayHi(); // undefined (same as above)
rabbit.__proto__.sayHi(); // undefined (same as above)
```

Only the first call uses `rabbit` as `this`. The others call with `Rabbit.prototype` as `this`, which has no `name` property.
