# Variables

Variables are "named storage" for data. Use `let` to create a variable.

## Basic Variable Usage

```javascript
let message; // declare variable
message = "Hello"; // assign value
alert(message); // shows: Hello

let message = "Hello!"; // declare + assign together
alert(message); // Hello!
```

## Multiple Variables

```javascript
// One line (not recommended - harder to read)
let user = "John",
  age = 25,
  message = "Hello";

// Multi-line (recommended - easier to read)
let user = "John";
let age = 25;
let message = "Hello";

// Alternative multiline styles
let user = "John",
  age = 25,
  message = "Hello";

let user = "John",
  age = 25,
  message = "Hello";
```

## Old Keyword: `var`

```javascript
var message = "Hello"; // old-school, slightly different from let
```

## Real-life Analogy

Think of a variable as a **box** with a unique sticker (name) that holds data. You can change the value anytime:

```javascript
let message;
message = "Hello!";
message = "World!"; // old value removed
alert(message); // World!
```

## Copying Data Between Variables

```javascript
let hello = "Hello world!";
let message;

message = hello; // copy value

alert(hello); // Hello world!
alert(message); // Hello world!
```

## Important Rule: No Repeated Declaration

```javascript
let message = "This";
let message = "That"; // SyntaxError: 'message' already declared
```

## Variable Naming Rules

**Allowed:** letters, digits, `$`, `_` (first character cannot be a digit)

```javascript
let userName; // camelCase (recommended for multiple words)
let test123;
let $ = 1; // $ is allowed
let _ = 2; // _ is allowed
```

**Not allowed:**

```javascript
let 1a;            // cannot start with digit
let my-name;       // hyphen '-' not allowed
```

**Case matters:** `apple` and `APPLE` are different variables.

**Reserved words** cannot be used: `let`, `class`, `return`, `function`, etc.

```javascript
let let = 5;       // error!
let return = 5;    // error!
```

## Assignment Without `let` (Bad Practice)

```javascript
// Without "use strict" - creates variable automatically (bad)
num = 5;
alert(num); // 5

// With "use strict" - error
("use strict");
num = 5; // error: num is not defined
```

## Constants: `const`

```javascript
const myBirthday = "18.04.1982";
myBirthday = "01.01.2001"; // error! can't reassign
```

### Uppercase Constants (for hard-coded values)

```javascript
const COLOR_RED = "#F00";
const COLOR_GREEN = "#0F0";
const COLOR_BLUE = "#00F";
const COLOR_ORANGE = "#FF7F00";

let color = COLOR_ORANGE;
alert(color); // #FF7F00
```

**When to use uppercase:** Only for constants known before execution (hard-coded values).

**Normal case for constants:** Values calculated at runtime but never change after assignment.

```javascript
const pageLoadTime = /* time taken to load */;  // normal case, not uppercase
```

## Naming Best Practices

- Use human-readable names: `userName`, `shoppingCart`
- Avoid abbreviations: `a`, `b`, `c` (unless you know what you're doing)
- Make names descriptive: `data` and `value` are too vague
- Be consistent with terms within your team
- Don't reuse variables for different purposes (extra variables are good, not evil)

## Summary

| Keyword | Description                                   |
| ------- | --------------------------------------------- |
| `let`   | Modern variable declaration                   |
| `var`   | Old-school (avoid; differences covered later) |
| `const` | Like `let` but value cannot change            |
