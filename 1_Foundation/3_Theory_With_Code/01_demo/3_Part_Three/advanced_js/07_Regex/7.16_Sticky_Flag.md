# Sticky Flag "y" Summary

The `y` flag (sticky) forces a regular expression to match **exactly at the position** specified by `lastIndex`, not somewhere after it.

## The Problem: Searching at a Specific Position

```javascript
let str = 'let varName = "value"';

// Want to find the word starting at position 4
// Expected: "varName"

// Without flags - finds first word only
console.log(str.match(/\w+/)); // "let" (wrong position)

// With 'g' flag - finds all words
console.log(str.match(/\w+/g)); // ["let", "varName", "value"] (too many)
```

## Using `lastIndex` with `g` Flag

The `g` flag allows controlling search start position via `lastIndex`:

```javascript
let str = 'let varName = "value"';
let regexp = /\w+/g;

regexp.lastIndex = 4;
let word = regexp.exec(str);
console.log(word); // "varName" (found at position 4)
console.log(word.index); // 4

// BUT: searches from position, not exactly at it
regexp.lastIndex = 3;
word = regexp.exec(str);
console.log(word); // "varName" (found at position 4, not exactly at 3)
```

## Sticky Flag `y` – Exact Position Match

The `y` flag ensures match occurs **exactly** at `lastIndex`:

```javascript
let str = 'let varName = "value"';
let regexp = /\w+/y;

regexp.lastIndex = 3;
console.log(regexp.exec(str)); // null (no word at position 3)

regexp.lastIndex = 4;
console.log(regexp.exec(str)); // ["varName"] (exact match)
```

## How `g` vs `y` Differ

| Flag | Behavior                                                              |
| ---- | --------------------------------------------------------------------- |
| `g`  | Searches from `lastIndex`; can find match later                       |
| `y`  | Matches only at `lastIndex`; fails if no match at that exact position |

```javascript
let str = "abc123";
let regexpG = /\d+/g;
let regexpY = /\d+/y;

regexpG.lastIndex = 2;
console.log(regexpG.exec(str)); // ["123"] (found at position 3)

regexpY.lastIndex = 2;
console.log(regexpY.exec(str)); // null (no digit at position 2)

regexpY.lastIndex = 3;
console.log(regexpY.exec(str)); // ["123"] (exact match)
```

## Practical Example: Lexical Analysis

Parsing code token by token:

```javascript
function tokenize(str) {
  let tokens = [];
  let regexp = /\w+|\d+|\s+|[()=]/y;
  let pos = 0;

  regexp.lastIndex = pos;
  let match;

  while ((match = regexp.exec(str))) {
    tokens.push(match[0]);
    pos = regexp.lastIndex;
  }

  return tokens;
}

let code = "let x = 42";
console.log(tokenize(code));
// ["let", " ", "x", " ", "=", " ", "42"]
```

## Performance Benefits

With `y` flag, regexp only checks the exact position, not scanning forward:

```javascript
let str = "x".repeat(10000) + "!";
let regexpG = /!/g;
let regexpY = /!/y;

console.time("g flag");
regexpG.lastIndex = 0;
regexpG.exec(str);
console.timeEnd("g flag"); // Scans entire string

console.time("y flag");
regexpY.lastIndex = 0;
regexpY.exec(str);
console.timeEnd("y flag"); // Checks only position 0, fails quickly
```

## Using `sticky` Property

Check if regexp has `y` flag:

```javascript
let regexp = /pattern/y;
console.log(regexp.sticky); // true

let regexp2 = /pattern/g;
console.log(regexp2.sticky); // false
```

## Complete Lexer Example

```javascript
class Lexer {
  constructor(str) {
    this.str = str;
    this.pos = 0;
    this.tokens = [];
  }

  tokenize() {
    const tokenPatterns = [
      { type: "NUMBER", regexp: /\d+/y },
      { type: "STRING", regexp: /"[^"]*"/y },
      { type: "IDENTIFIER", regexp: /[a-zA-Z_]\w*/y },
      { type: "OPERATOR", regexp: /[+\-*/=]/y },
      { type: "WHITESPACE", regexp: /\s+/y },
      { type: "PAREN", regexp: /[()]/y },
    ];

    while (this.pos < this.str.length) {
      let matched = false;

      for (let pattern of tokenPatterns) {
        pattern.regexp.lastIndex = this.pos;
        let match = pattern.regexp.exec(this.str);

        if (match) {
          if (pattern.type !== "WHITESPACE") {
            this.tokens.push({
              type: pattern.type,
              value: match[0],
              position: this.pos,
            });
          }
          this.pos = pattern.regexp.lastIndex;
          matched = true;
          break;
        }
      }

      if (!matched) {
        throw new Error(`Unexpected character at position ${this.pos}`);
      }
    }

    return this.tokens;
  }
}

let lexer = new Lexer("x = 42 + y");
console.log(lexer.tokenize());
/* [
  { type: 'IDENTIFIER', value: 'x', position: 0 },
  { type: 'OPERATOR', value: '=', position: 2 },
  { type: 'NUMBER', value: '42', position: 4 },
  { type: 'OPERATOR', value: '+', position: 7 },
  { type: 'IDENTIFIER', value: 'y', position: 9 }
] */
```

## Summary

| Flag | Behavior                                  | Use Case                                        |
| ---- | ----------------------------------------- | ----------------------------------------------- |
| `g`  | Global, searches from `lastIndex` forward | Finding all matches in string                   |
| `y`  | Sticky, matches exactly at `lastIndex`    | Lexical analysis, parsing at specific positions |

**Key Points:**

- `y` flag forces match at exact `lastIndex` position
- Returns `null` if no match at that exact position
- More performant when scanning at known positions
- Essential for writing parsers and tokenizers
- Use `regexp.sticky` property to check if `y` flag is set
- Works with `regexp.exec()` and `regexp.test()` methods
