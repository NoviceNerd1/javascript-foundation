# Garbage Collection

## Overview

JavaScript automatically manages memory through **garbage collection**. Unreachable values are removed automatically.

## Reachability

A value is **reachable** if accessible from **roots** (global variables, currently executing function variables, etc.)

### Roots (Always Reachable)

- Current function's local variables/parameters
- Variables/parameters of nested calls on the stack
- Global variables

## Examples

### Simple Reference

```javascript
let user = { name: "John" };
// Object is reachable via 'user' reference

user = null;
// Object becomes unreachable → garbage collected
```

### Two References

```javascript
let user = { name: "John" };
let admin = user;

user = null;
// Object still reachable via 'admin' → stays in memory

admin = null;
// Now unreachable → garbage collected
```

### Interlinked Objects

```javascript
function marry(man, woman) {
  woman.husband = man;
  man.wife = woman;
  return {
    father: man,
    mother: woman,
  };
}

let family = marry({ name: "John" }, { name: "Ann" });

// Remove references
delete family.father;
delete family.mother.husband;
// John becomes unreachable → garbage collected
```

### Unreachable Island

```javascript
family = null;
// Entire object structure becomes unreachable
// Even though John and Ann reference each other,
// no root references them → all removed
```

## Mark-and-Sweep Algorithm

1. **Mark** roots
2. **Follow** references and mark all reachable objects
3. **Remove** unmarked objects

```
Roots → follow references → mark reachable → delete unmarked
```

## Optimization Techniques

| Technique                   | Description                                                                           |
| --------------------------- | ------------------------------------------------------------------------------------- |
| **Generational collection** | New objects tracked separately (most die young); old objects examined less frequently |
| **Incremental collection**  | Splits marking into parts to reduce delays                                            |
| **Idle-time collection**    | Runs only when CPU is idle                                                            |

## Key Points

- Garbage collection is **automatic** – cannot force or prevent
- Objects stay in memory while **reachable**
- **Being referenced ≠ being reachable** – interconnected objects can become unreachable together
- Modern engines use advanced algorithms with optimizations

> **Note:** In-depth engine knowledge (like V8 internals) is useful for low-level optimizations after mastering the language.
