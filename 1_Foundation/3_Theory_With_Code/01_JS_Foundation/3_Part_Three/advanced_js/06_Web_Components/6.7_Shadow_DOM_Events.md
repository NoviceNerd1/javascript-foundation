# Shadow DOM and Events Summary

Events in shadow DOM are retargeted to maintain encapsulation, with the host element appearing as the target when caught outside.

## Event Retargeting

When an event originates inside shadow DOM, the outer document sees the **shadow host** as the target:

```html
<user-card></user-card>

<script>
  customElements.define(
    "user-card",
    class extends HTMLElement {
      connectedCallback() {
        this.attachShadow({ mode: "open" });
        this.shadowRoot.innerHTML = `<button>Click me</button>`;

        // Inner handler sees the actual button
        this.shadowRoot.firstChild.onclick = (e) => {
          console.log("Inner target:", e.target.tagName); // BUTTON
        };
      }
    },
  );

  // Outer handler sees the host element
  document.onclick = (e) => {
    console.log("Outer target:", e.target.tagName); // USER-CARD
  };
</script>
```

## When Retargeting Does NOT Occur

Events on **slotted elements** (from light DOM) are not retargeted:

```html
<user-card>
  <span slot="username">John Smith</span>
  <!-- Click here -->
</user-card>

<script>
  customElements.define(
    "user-card",
    class extends HTMLElement {
      connectedCallback() {
        this.attachShadow({ mode: "open" });
        this.shadowRoot.innerHTML = `<slot name="username"></slot>`;

        this.shadowRoot.onclick = (e) => {
          console.log("Inner target:", e.target.tagName); // SPAN
        };
      }
    },
  );

  userCard.onclick = (e) => {
    console.log("Outer target:", e.target.tagName); // SPAN (no retargeting)
  };
</script>
```

## event.composedPath()

Returns the full event path through flattened DOM:

```javascript
element.addEventListener("click", (e) => {
  console.log(e.composedPath());
  // [span, slot, div, shadow-root, user-card, body, html, document, window]
});
```

**For closed shadow DOM (`mode: 'closed'`):** path starts from host, internal details hidden.

## composed Event Property

Controls whether events cross shadow DOM boundary:

| `composed` | Behavior                            |
| ---------- | ----------------------------------- |
| `true`     | Event crosses shadow boundary       |
| `false`    | Event only caught inside shadow DOM |

### Built-in Events

| `composed: true`               | `composed: false`          |
| ------------------------------ | -------------------------- |
| click, dblclick                | mouseenter, mouseleave     |
| mousedown, mouseup             | load, unload, abort, error |
| mousemove, mouseout, mouseover | select                     |
| wheel                          | slotchange                 |
| blur, focus, focusin, focusout |                            |
| keydown, keyup                 |                            |
| input, beforeinput             |                            |
| touch events, pointer events   |                            |

### Custom Events

Must explicitly set `composed: true` to cross shadow boundary:

```javascript
let inner = document.createElement("div");
outer.shadowRoot.append(inner);

// This event reaches document
inner.dispatchEvent(
  new CustomEvent("test", {
    bubbles: true,
    composed: true,
    detail: "crosses boundary",
  }),
);

// This event does NOT reach document
inner.dispatchEvent(
  new CustomEvent("test", {
    bubbles: true,
    composed: false,
    detail: "stays inside",
  }),
);
```

## Event Bubbling Through Shadow DOM

Events bubble through the flattened DOM:

```
Flattened DOM:
<user-card>                    ← Event reaches here (retargeted)
  #shadow-root
    <div>                      ← Bubbles up
      <slot>                   ← Bubbles up
        <span>                 ← Original target (slotted)
```

## Complete Example

```html
<custom-panel id="panel">
  <button slot="action">Click Me</button>
</custom-panel>

<script>
  class CustomPanel extends HTMLElement {
    connectedCallback() {
      this.attachShadow({ mode: "open" });
      this.shadowRoot.innerHTML = `
      <style>
        .panel { border: 1px solid #ccc; padding: 10px; }
        .title { background: #eee; padding: 5px; }
      </style>
      <div class="panel">
        <div class="title"><slot name="title">Panel</slot></div>
        <div class="content"><slot></slot></div>
        <div class="actions"><slot name="action"></slot></div>
      </div>
    `;

      // Track clicks inside shadow DOM
      this.shadowRoot.addEventListener("click", (e) => {
        console.log("Shadow click - target:", e.target.tagName);
        console.log(
          "Composed path:",
          e.composedPath().map((el) => el.tagName || el.nodeName),
        );
      });
    }
  }

  customElements.define("custom-panel", CustomPanel);

  // Track clicks from document
  panel.addEventListener("click", (e) => {
    console.log("Document click - target:", e.target.tagName);
    console.log("Retargeted?", e.target === panel ? "Yes" : "No");
  });

  // Simulate click on slotted button after 1 second
  setTimeout(() => {
    panel.querySelector("button").click();
  }, 1000);
</script>
```

## Custom Event with composed: false (Component-Only)

```javascript
class Modal extends HTMLElement {
  close() {
    // Dispatch event that only affects the component
    this.dispatchEvent(
      new CustomEvent("modal-close", {
        bubbles: true,
        composed: false, // Won't bubble out of component
        detail: { timestamp: Date.now() },
      }),
    );
  }

  connectedCallback() {
    this.attachShadow({ mode: "open" });
    // Listen for internal events
    this.addEventListener("modal-close", () => {
      this.remove(); // Remove component from DOM
    });
  }
}
```

## Summary

| Concept             | Description                                                  |
| ------------------- | ------------------------------------------------------------ |
| **Retargeting**     | Events from shadow DOM show host as target in outer document |
| **No retargeting**  | Events on slotted elements keep original target              |
| **composedPath()**  | Full event path through flattened DOM                        |
| **composed flag**   | `true` = crosses shadow boundary, `false` = stays inside     |
| **Custom events**   | Must set `composed: true` to cross boundary                  |
| **Built-in events** | Most UI events have `composed: true`                         |

**Key Points:**

- Events from shadow DOM are retargeted to the host for external listeners
- Slotted element events are not retargeted
- `composedPath()` reveals the full event chain including shadow DOM
- Set `composed: true` on custom events to bubble out of shadow DOM
- Some events (mouseenter, slotchange) never cross shadow boundary
