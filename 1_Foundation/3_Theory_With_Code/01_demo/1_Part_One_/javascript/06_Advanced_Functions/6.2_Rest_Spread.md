# Rest Parameters and Spread Syntax

## Overview

The `...` syntax has two opposite uses:

- **Rest parameters**: Gather remaining arguments into an array (function definition)
- **Spread syntax**: Expand an array into a list of arguments (function call)

## Rest Parameters `...`

### Gather Remaining Arguments

```javascript
function sumAll(...args) {
  let sum = 0;
  for (let arg of args) sum += arg;
  return sum;
}

alert(sumAll(1)); // 1
alert(sumAll(1, 2, 3)); // 6
```

### First Parameters as Variables, Rest as Array

```javascript
function showName(firstName, lastName, ...titles) {
  alert(firstName + " " + lastName); // Julius Caesar
  alert(titles[0]); // Consul
  alert(titles.length); // 2
}

showName("Julius", "Caesar", "Consul", "Imperator");
```

**Important:** Rest parameters must be **last** in the parameter list.

```javascript
// Error: rest must be last
function f(arg1, ...rest, arg2) { }
```

## The `arguments` Object (Old Style)

```javascript
function showName() {
  alert(arguments.length);
  alert(arguments[0]);
  alert(arguments[1]);
}

showName("Julius", "Caesar"); // 2, Julius, Caesar
```

**Downsides:**

- Array-like (not a real array) – no array methods like `map()`
- Captures **all** arguments (cannot partially capture)
- Arrow functions don't have `arguments`

## Spread Syntax `...`

### Expand Array into Function Arguments

```javascript
let arr = [3, 5, 1];
alert(Math.max(...arr)); // 5 (instead of NaN)
```

### Combine Multiple Arrays and Values

```javascript
let arr1 = [1, -2, 3, 4];
let arr2 = [8, 3, -8, 1];

alert(Math.max(...arr1, ...arr2)); // 8
alert(Math.max(1, ...arr1, 2, ...arr2, 25)); // 25
```

### Merge Arrays

```javascript
let arr = [3, 5, 1];
let arr2 = [8, 9, 15];

let merged = [0, ...arr, 2, ...arr2];
alert(merged); // 0,3,5,1,2,8,9,15
```

### Convert String to Array

```javascript
let str = "Hello";
alert([...str]); // H,e,l,l,o

// Array.from also works (more universal)
alert(Array.from(str)); // H,e,l,l,o
```

**Difference:** `Array.from` works with both iterables and array-likes; spread works only with iterables.

### Copy Arrays and Objects

```javascript
// Copy array
let arr = [1, 2, 3];
let arrCopy = [...arr];
arr.push(4);
alert(arr); // 1,2,3,4
alert(arrCopy); // 1,2,3

// Copy object
let obj = { a: 1, b: 2, c: 3 };
let objCopy = { ...obj };
obj.d = 4;
alert(JSON.stringify(obj)); // {"a":1,"b":2,"c":3,"d":4}
alert(JSON.stringify(objCopy)); // {"a":1,"b":2,"c":3}
```

## Rest vs Spread – Quick Guide

| Context             | Syntax                | Purpose                     |
| ------------------- | --------------------- | --------------------------- |
| Function definition | `function f(...args)` | Gather arguments into array |
| Function call       | `f(...arr)`           | Expand array into arguments |
| Array literal       | `[...arr1, ...arr2]`  | Merge arrays                |
| Object literal      | `{ ...obj }`          | Copy object                 |

## Summary

| Feature             | Description                                                            |
| ------------------- | ---------------------------------------------------------------------- |
| **Rest parameters** | `...` in function parameters – collects remaining arguments into array |
| **Spread syntax**   | `...` in function calls/array literals – expands iterable into list    |
| **`arguments`**     | Old array-like object (avoid in modern code)                           |
| **Copying**         | `[...arr]` and `{...obj}` create shallow copies                        |

**Key Points:**

- Rest parameters must be **last** in function definition
- Spread works with **any iterable** (strings, arrays, maps, sets)
- Arrow functions **don't have** their own `arguments`
