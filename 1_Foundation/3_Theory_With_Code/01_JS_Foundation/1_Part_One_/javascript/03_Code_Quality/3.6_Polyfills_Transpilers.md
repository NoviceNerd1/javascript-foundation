# Polyfills and Transpilers

## Overview

JavaScript evolves constantly. New features may not be supported in older browsers. **Transpilers** and **polyfills** bridge this gap, allowing modern code to run on outdated engines.

## Transpilers

A **transpiler** converts modern syntax into older equivalent code.

### Example: Nullish Coalescing (`??`)

```javascript
// Modern code
height = height ?? 100;

// After transpilation
height = height !== undefined && height !== null ? height : 100;
```

### Popular Transpilers

- **Babel** – Most prominent transpiler
- Integrated with build tools like **webpack** (via babel-loader)

## Polyfills

A **polyfill** adds missing built-in functions (not syntax). It "fills in" the gap.

### Example: `Math.trunc`

```javascript
if (!Math.trunc) {
  Math.trunc = function (number) {
    return number < 0 ? Math.ceil(number) : Math.floor(number);
  };
}
```

### Popular Polyfill Library

- **core-js** – Supports a wide range of features, allows selective inclusion

## When to Use What

| Feature Type                                                  | Solution                     |
| ------------------------------------------------------------- | ---------------------------- |
| New syntax (operators, `class`, etc.)                         | **Transpiler** (e.g., Babel) |
| New built-in functions (`Math.trunc`, `Array.includes`, etc.) | **Polyfill** (e.g., core-js) |

## Development Flow

1. Write modern JavaScript
2. Run transpiler (converts syntax to older version)
3. Include polyfills (add missing functions)
4. Deploy to production

## Summary

- **Transpilers** – Convert new syntax to old syntax
- **Polyfills** – Add missing built-in functions
- Use both to ensure code works everywhere
- **Babel** + **core-js** is the standard toolchain
