# Recursion and Stack

## Overview

Recursion occurs when a function calls **itself**. It's useful for tasks that can be split into simpler versions of the same task.

## Two Approaches: Iterative vs Recursive

### Iterative (Loop)

```javascript
function pow(x, n) {
  let result = 1;
  for (let i = 0; i < n; i++) {
    result *= x;
  }
  return result;
}
```

### Recursive

```javascript
function pow(x, n) {
  if (n == 1) {
    return x; // Base case
  } else {
    return x * pow(x, n - 1); // Recursive step
  }
}
```

## How Recursion Works

### Components

- **Base case**: Simplest case that returns immediately (e.g., `n == 1`)
- **Recursive step**: Reduces problem to simpler version + additional operation

### Execution Context Stack

When a function makes a nested call:

1. Current function is **paused**
2. Its execution context is **stored** in a stack
3. Nested call **executes**
4. After nested call finishes, previous context is **restored**

**Example: `pow(2, 3)` call stack:**

```
Context: { x: 2, n: 1 }  → returns 2
Context: { x: 2, n: 2 }  → returns 4
Context: { x: 2, n: 3 }  → returns 8
```

### Recursion Depth Limit

- Max recursion depth is **~10,000** (engine-dependent)
- Each nested call consumes memory
- Recursive functions can be rewritten iteratively for better performance

## Recursive Traversal

### Example: Summing Nested Structure

```javascript
let company = {
  sales: [
    { name: "John", salary: 1000 },
    { name: "Alice", salary: 1600 },
  ],
  development: {
    sites: [
      { name: "Peter", salary: 2000 },
      { name: "Alex", salary: 1800 },
    ],
    internals: [{ name: "Jack", salary: 1300 }],
  },
};

function sumSalaries(department) {
  if (Array.isArray(department)) {
    // Base case
    return department.reduce((sum, p) => sum + p.salary, 0);
  } else {
    // Recursive case
    let sum = 0;
    for (let subdep of Object.values(department)) {
      sum += sumSalaries(subdep);
    }
    return sum;
  }
}

alert(sumSalaries(company)); // 7700
```

## Recursive Data Structures

### Linked List

```javascript
let list = {
  value: 1,
  next: {
    value: 2,
    next: {
      value: 3,
      next: {
        value: 4,
        next: null,
      },
    },
  },
};

// Alternative creation
let list = { value: 1 };
list.next = { value: 2 };
list.next.next = { value: 3 };
list.next.next.next = { value: 4 };

// Prepend
list = { value: "new", next: list };

// Remove from middle
list.next = list.next.next;
```

**Linked List Advantages:**

- Fast insertion/deletion (no mass renumbering)
- Easy to split and join

**Linked List Disadvantages:**

- Cannot access element by index directly

## Tasks

### 1. Sum All Numbers (3 Variants)

```javascript
// Loop
function sumTo(n) {
  let sum = 0;
  for (let i = 1; i <= n; i++) sum += i;
  return sum;
}

// Recursion
function sumTo(n) {
  if (n == 1) return 1;
  return n + sumTo(n - 1);
}

// Arithmetic formula
function sumTo(n) {
  return (n * (n + 1)) / 2;
}
```

**Performance:** Formula is fastest, loop is second, recursion is slowest. Recursion fails for large n due to stack overflow.

### 2. Calculate Factorial

```javascript
function factorial(n) {
  return n == 1 ? 1 : n * factorial(n - 1);
}

alert(factorial(5)); // 120
```

### 3. Fibonacci Numbers

```javascript
function fib(n) {
  let a = 1,
    b = 1;
  for (let i = 3; i <= n; i++) {
    let c = a + b;
    a = b;
    b = c;
  }
  return b;
}

alert(fib(77)); // 5527939700884757 (fast)
```

### 4. Output Single-Linked List

```javascript
// Loop
function printList(list) {
  let current = list;
  while (current) {
    alert(current.value);
    current = current.next;
  }
}

// Recursion
function printList(list) {
  alert(list.value);
  if (list.next) printList(list.next);
}
```

**Better:** Loop avoids recursion depth limits.

### 5. Output Single-Linked List in Reverse

```javascript
// Loop (using array)
function printReverseList(list) {
  let arr = [];
  let current = list;
  while (current) {
    arr.push(current.value);
    current = current.next;
  }
  for (let i = arr.length - 1; i >= 0; i--) {
    alert(arr[i]);
  }
}

// Recursion
function printReverseList(list) {
  if (list.next) printReverseList(list.next);
  alert(list.value);
}
```

## Summary

| Concept                      | Description                          |
| ---------------------------- | ------------------------------------ |
| **Recursion**                | Function calling itself              |
| **Base case**                | Stops recursion (simplest scenario)  |
| **Recursive step**           | Reduces problem and calls self       |
| **Recursion depth**          | Max nested calls (~10,000)           |
| **Execution stack**          | Stores paused function contexts      |
| **Recursive data structure** | Self-referential (linked list, tree) |

**When to use recursion:**

- Tree/nested structure traversal
- Problems naturally defined recursively (factorial, Fibonacci)
- When code readability is more important than micro-optimizations

**When to use loops:**

- Simple iterations
- When recursion depth may be large
- When performance is critical
