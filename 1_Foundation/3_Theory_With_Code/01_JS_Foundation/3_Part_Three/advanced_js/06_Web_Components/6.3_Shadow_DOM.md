# Shadow DOM Summary

Shadow DOM provides encapsulation by allowing a component to have its own hidden DOM tree with isolated styles and IDs, inaccessible from the main document.

## What is Shadow DOM?

Shadow DOM is a hidden DOM subtree that:

- Is separate from the main document (light DOM)
- Has its own styles (no leakage from/to outer page)
- Has its own ID namespace
- Is rendered instead of regular children

## Creating Shadow DOM

```javascript
// Create shadow root (must be open or closed)
let shadowRoot = element.attachShadow({ mode: "open" });
```

### mode Options

| Mode       | `elem.shadowRoot`   | Accessibility                          |
| ---------- | ------------------- | -------------------------------------- |
| `"open"`   | Returns shadow root | Can be accessed from outside           |
| `"closed"` | Returns `null`      | Only accessible via returned reference |

### Host Element Requirements

Only certain elements can host shadow DOM:

- Custom elements
- Elements: `article`, `aside`, `blockquote`, `body`, `div`, `footer`, `h1`-`h6`, `header`, `main`, `nav`, `p`, `section`, `span`

**Cannot host:** `<img>`, `<input>`, `<button>`, etc.

## Basic Example

```html
<show-hello name="John"></show-hello>

<script>
  customElements.define(
    "show-hello",
    class extends HTMLElement {
      connectedCallback() {
        const shadow = this.attachShadow({ mode: "open" });
        shadow.innerHTML = `
      <style>
        p { font-weight: bold; color: blue; }
      </style>
      <p>Hello, ${this.getAttribute("name")}!</p>
    `;
      }
    },
  );
</script>
```

## Built-in Shadow DOM

Browser native controls also use shadow DOM:

```html
<input type="range" />
```

In DevTools (enable "Show user agent shadow DOM"):

```
#shadow-root (user-agent)
  <div pseudo="-webkit-slider-runnable-track">
    ...
  </div>
```

### Styling Built-in Shadow DOM

```css
/* Style browser's native slider track (non-standard) */
input::-webkit-slider-runnable-track {
  background: red;
}
```

## Encapsulation Rules

### 1. No Access from Light DOM

```javascript
// Shadow DOM elements not found by document selectors
document.querySelectorAll("p").length; // 0
shadowRoot.querySelectorAll("p").length; // 1 (inside shadow)
```

### 2. Isolated Styles

```html
<style>
  /* Does NOT affect shadow DOM */
  p {
    color: red;
  }
</style>

<div id="elem"></div>
<script>
  elem.attachShadow({ mode: "open" });
  elem.shadowRoot.innerHTML = `
    <style>p { font-weight: bold; }</style>
    <p>Bold only, not red</p>
  `;
</script>
```

### 3. Separate ID Namespace

```javascript
// Same ID can exist in light and shadow DOM
lightDiv.id = "same-id";
shadowDiv.id = "same-id"; // No conflict
```

## Shadow Root Properties

```javascript
let shadow = element.attachShadow({ mode: "open" });

// Access host element
console.log(shadow.host === element); // true

// Access shadow root (if mode = 'open')
console.log(element.shadowRoot); // returns shadow

// Check if element has shadow root
console.log(element.shadowRoot); // null if not attached
```

## Closed Shadow DOM

```javascript
class ClosedComponent extends HTMLElement {
  constructor() {
    super();
    const shadow = this.attachShadow({ mode: "closed" });
    shadow.innerHTML = "<p>Hidden</p>";
    this._shadow = shadow; // Store reference internally
  }

  getContent() {
    return this._shadow.innerHTML; // Controlled access
  }
}

customElements.define("closed-comp", ClosedComponent);

let elem = document.querySelector("closed-comp");
console.log(elem.shadowRoot); // null
console.log(elem.getContent()); // '<p>Hidden</p>'
```

## Complete Custom Element with Shadow DOM

```javascript
class Tooltip extends HTMLElement {
  constructor() {
    super();
    this._tooltipIcon = null;
    this.attachShadow({ mode: "open" });
  }

  connectedCallback() {
    this.shadowRoot.innerHTML = `
      <style>
        :host {
          position: relative;
          display: inline-block;
        }
        .tooltip-text {
          position: absolute;
          background: #333;
          color: white;
          padding: 5px;
          border-radius: 3px;
          font-size: 12px;
          display: none;
          white-space: nowrap;
          z-index: 10;
        }
        :host(:hover) .tooltip-text {
          display: block;
        }
        .icon {
          cursor: pointer;
          font-weight: bold;
          background: #eee;
          padding: 0 5px;
          border-radius: 3px;
        }
      </style>
      <span class="icon">?</span>
      <span class="tooltip-text">
        <slot></slot>
      </span>
    `;
  }
}

customElements.define("custom-tooltip", Tooltip);
```

```html
<custom-tooltip>This is the tooltip text</custom-tooltip>
```

## :host Pseudo-class

Styles the shadow host element from inside:

```javascript
shadowRoot.innerHTML = `
  <style>
    /* Styles the host element itself */
    :host {
      display: inline-block;
      border: 1px solid #ccc;
      padding: 5px;
    }
    
    /* Styles host with specific attribute */
    :host([theme="dark"]) {
      background: #333;
      color: white;
    }
    
    /* Styles host when in specific context */
    :host-context(.dark-theme) {
      background: #222;
    }
  </style>
`;
```

## Event Retargeting

Events from shadow DOM appear to come from the host element:

```javascript
class ButtonWithShadow extends HTMLElement {
  connectedCallback() {
    let shadow = this.attachShadow({ mode: "open" });
    shadow.innerHTML = `<button>Click me</button>`;

    shadow.querySelector("button").onclick = (e) => {
      console.log(e.target); // <button> (actual target)
      console.log(e.composedPath()); // [button, shadow-root, button-with-shadow]
    };
  }
}

customElements.define("btn-shadow", ButtonWithShadow);
```

```html
<btn-shadow></btn-shadow>
<script>
  document.querySelector("btn-shadow").onclick = (e) => {
    console.log(e.target); // <btn-shadow> (retargeted!)
  };
</script>
```

## Browser Support & Polyfill

- Modern browsers: Chrome, Firefox, Safari, Edge all support Shadow DOM v1
- Polyfill: https://github.com/webcomponents/polyfills/tree/master/packages/shadow-dom

## Summary

| Feature      | Light DOM                | Shadow DOM                 |
| ------------ | ------------------------ | -------------------------- |
| **Access**   | Global                   | Encapsulated               |
| **Styles**   | Global                   | Local only                 |
| **IDs**      | Global namespace         | Component scope            |
| **Query**    | `document.querySelector` | `shadowRoot.querySelector` |
| **Creation** | Regular HTML             | `attachShadow()`           |

**Key Points:**

- `elem.attachShadow({mode: 'open/closed'})` creates shadow DOM
- Only one shadow root per element
- Shadow DOM styles don't affect outside; outside styles don't affect shadow DOM
- Events from shadow DOM are retargeted to host element
- Use `:host` to style the host element from inside shadow DOM
- Custom elements often use shadow DOM for component encapsulation
