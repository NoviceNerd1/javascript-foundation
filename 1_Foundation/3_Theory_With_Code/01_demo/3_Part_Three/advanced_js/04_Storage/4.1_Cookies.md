# Cookies, document.cookie Summary

Cookies are small strings stored in the browser, sent automatically with every request to the same domain. They are commonly used for authentication and user tracking.

## Reading Cookies

```javascript
// Read all cookies (name=value pairs separated by ;)
console.log(document.cookie); // "cookie1=value1; cookie2=value2"
```

## Writing Cookies

```javascript
// Set a cookie (only updates the named cookie)
document.cookie = "user=John";

// Encode special characters
let name = "my name";
let value = "John Smith";
document.cookie = encodeURIComponent(name) + "=" + encodeURIComponent(value);
```

## Cookie Attributes

Attributes are appended after `name=value`, separated by `;`:

```javascript
document.cookie =
  "user=John; path=/; expires=Tue, 19 Jan 2038 03:14:07 GMT; secure";
```

### domain

Controls which domains can access the cookie:

```javascript
// Default: only current domain
document.cookie = "user=John";

// Allow all subdomains (*.site.com)
document.cookie = "user=John; domain=site.com";
```

### path

URL path prefix for cookie access:

```javascript
// Accessible under /admin and /admin/*
document.cookie = "user=John; path=/admin";

// Accessible everywhere (recommended)
document.cookie = "user=John; path=/";
```

### expires & max-age

```javascript
// Session cookie (deleted when browser closes)
document.cookie = "user=John";

// Expires in 1 day
let date = new Date(Date.now() + 86400e3);
document.cookie = `user=John; expires=${date.toUTCString()}`;

// Expires in 1 hour (max-age in seconds)
document.cookie = "user=John; max-age=3600";

// Delete cookie
document.cookie = "user=John; max-age=0";
```

### secure

Only send cookie over HTTPS:

```javascript
// Only sent over HTTPS connections
document.cookie = "user=John; secure";
```

### samesite

Protects against XSRF attacks:

| Value    | Behavior                                     |
| -------- | -------------------------------------------- |
| `strict` | Never sent for cross-site requests           |
| `lax`    | Sent for safe GET top-level navigations only |
| `none`   | Always sent (requires `secure`)              |

```javascript
// Strict protection
document.cookie = "user=John; samesite=strict";

// Lax (default for modern browsers)
document.cookie = "user=John; samesite=lax";

// No protection (requires secure)
document.cookie = "user=John; samesite=none; secure";
```

### httpOnly

Not accessible from JavaScript (server-set only):

```http
Set-Cookie: session=secret; httpOnly
```

## Cookie Helper Functions

```javascript
// Get cookie by name
function getCookie(name) {
  let matches = document.cookie.match(
    new RegExp(
      "(?:^|; )" +
        name.replace(/([\.$?*|{}\(\)\[\]\\\/\+^])/g, "\\$1") +
        "=([^;]*)",
    ),
  );
  return matches ? decodeURIComponent(matches[1]) : undefined;
}

// Set cookie with attributes
function setCookie(name, value, attributes = {}) {
  attributes = { path: "/", ...attributes };

  if (attributes.expires instanceof Date) {
    attributes.expires = attributes.expires.toUTCString();
  }

  let updatedCookie =
    encodeURIComponent(name) + "=" + encodeURIComponent(value);

  for (let key in attributes) {
    updatedCookie += "; " + key;
    if (attributes[key] !== true) {
      updatedCookie += "=" + attributes[key];
    }
  }

  document.cookie = updatedCookie;
}

// Delete cookie
function deleteCookie(name) {
  setCookie(name, "", { "max-age": -1 });
}

// Usage
setCookie("user", "John", { secure: true, "max-age": 3600 });
console.log(getCookie("user")); // John
deleteCookie("user");
```

## XSRF Attack Protection

### How XSRF Works

1. User logged into `bank.com` (has auth cookie)
2. User visits `evil.com`
3. `evil.com` submits form to `bank.com/pay`
4. Browser sends auth cookie → unauthorized payment

### Protection with samesite

```javascript
// Server sets cookie with samesite
Set-Cookie: auth=secret; samesite=strict

// Cross-site requests won't send this cookie
// XSRF attack fails
```

## Third-Party Cookies

Cookies set by domains other than the current page:

```html
<!-- Banner from ads.com sets cookie -->
<img src="https://ads.com/banner.png" />
```

- Used for cross-site tracking
- Safari blocks by default
- Firefox uses blocklists
- Chrome phasing out support

## Cookie Limitations

| Limitation             | Value                   |
| ---------------------- | ----------------------- |
| Max size per cookie    | 4KB                     |
| Max cookies per domain | ~20 (browser dependent) |
| Can set/update         | One cookie at a time    |

## GDPR Considerations

- Tracking/authentication cookies require user consent in EU
- Two approaches:
  1. Consent checkbox on registration
  2. Splash screen requiring agreement
- Non-tracking cookies (preferences, settings) are generally allowed

## Cookie Attributes Quick Reference

| Attribute  | Purpose                    |
| ---------- | -------------------------- |
| `domain`   | Which domains can access   |
| `path`     | URL path restriction       |
| `expires`  | Expiration date            |
| `max-age`  | Expiration in seconds      |
| `secure`   | HTTPS only                 |
| `samesite` | XSRF protection            |
| `httpOnly` | No JS access (server only) |

## Summary

1. **Read**: `document.cookie` returns all cookies
2. **Write**: `document.cookie = "name=value; attributes"`
3. **Encode**: Use `encodeURIComponent()` for special characters
4. **Delete**: Set `max-age=0` or `expires` in past
5. **Attributes**: `domain`, `path`, `expires`/`max-age`, `secure`, `samesite`
6. **Security**: `httpOnly` prevents JS access, `samesite` prevents XSRF
7. **GDPR**: Tracking cookies require user consent in EU
