# Promise

## Overview

A Promise is an object that links **producing code** (which takes time) and **consuming code** (which waits for the result). It represents a future value.

## Promise States

| State       | Description                                         |
| ----------- | --------------------------------------------------- |
| `pending`   | Initial state, neither fulfilled nor rejected       |
| `fulfilled` | Operation completed successfully (`resolve` called) |
| `rejected`  | Operation failed (`reject` called)                  |

## Promise Constructor

```javascript
let promise = new Promise(function (resolve, reject) {
  // Executor (producing code)
  // Call resolve(value) on success
  // Call reject(error) on failure
});
```

### Basic Example

```javascript
let promise = new Promise(function (resolve, reject) {
  setTimeout(() => resolve("done"), 1000);
});

promise.then(
  (result) => alert(result), // "done" after 1 second
  (error) => alert(error), // doesn't run
);
```

### Error Example

```javascript
let promise = new Promise(function (resolve, reject) {
  setTimeout(() => reject(new Error("Whoops!")), 1000);
});

promise.then(
  (result) => alert(result), // doesn't run
  (error) => alert(error), // "Error: Whoops!" after 1 second
);
```

## Important Rules

- **Only one result/error**: Once `resolve` or `reject` is called, further calls are ignored
- **Use Error objects**: Recommended for `reject`
- **Immediate resolution**: `resolve`/`reject` can be called immediately (no need to wait)

```javascript
let promise = new Promise(function (resolve, reject) {
  resolve("done");
  reject(new Error("...")); // ignored
  setTimeout(() => resolve("...")); // ignored
});
```

## Consumers: then, catch, finally

### then

```javascript
promise.then(
  (result) => {
    /* handle success */
  },
  (error) => {
    /* handle error */
  },
);

// Only success handler
promise.then((result) => alert(result));

// Only error handler (using catch is better)
promise.then(null, (error) => alert(error));
```

### catch

```javascript
promise.catch((error) => alert(error));
// Equivalent to: promise.then(null, error => alert(error))
```

### finally

Runs always when promise is settled (fulfilled or rejected). Used for cleanup.

```javascript
new Promise((resolve, reject) => {
  setTimeout(() => resolve("value"), 2000);
})
  .finally(() => alert("Promise ready")) // Runs first
  .then((result) => alert(result)); // Shows "value"

// With error
new Promise((resolve, reject) => {
  throw new Error("error");
})
  .finally(() => alert("Promise ready")) // Runs first
  .catch((err) => alert(err)); // Shows error
```

**finally features:**

- No arguments (doesn't know success/failure)
- Passes through result/error to next handler
- Return value ignored
- If throws error, it goes to next error handler

## Adding Handlers to Settled Promises

Handlers run immediately if promise is already settled:

```javascript
let promise = new Promise((resolve) => resolve("done!"));
promise.then(alert); // "done!" (immediately)
```

## Practical Example: loadScript with Promise

```javascript
function loadScript(src) {
  return new Promise(function (resolve, reject) {
    let script = document.createElement("script");
    script.src = src;

    script.onload = () => resolve(script);
    script.onerror = () => reject(new Error(`Script load error for ${src}`));

    document.head.append(script);
  });
}

// Usage
let promise = loadScript(
  "https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.11/lodash.js",
);

promise.then(
  (script) => alert(`${script.src} is loaded!`),
  (error) => alert(`Error: ${error.message}`),
);

promise.then((script) => alert("Another handler...")); // Multiple handlers allowed
```

## Promise vs Callbacks

| Promises                                      | Callbacks                         |
| --------------------------------------------- | --------------------------------- |
| Natural order: first load, then handle result | Must know callback before calling |
| Multiple handlers allowed                     | Only one callback                 |
| Better error handling with `.catch`           | Error handling manual             |

## Tasks

### 1. Re-resolve a Promise?

```javascript
let promise = new Promise(function (resolve, reject) {
  resolve(1);
  setTimeout(() => resolve(2), 1000);
});

promise.then(alert); // Shows 1 (second resolve ignored)
```

### 2. Delay with a Promise

```javascript
function delay(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

delay(3000).then(() => alert("runs after 3 seconds"));
```

### 3. Animated Circle with Promise

```javascript
function showCircle(cx, cy, radius) {
  return new Promise((resolve) => {
    let div = document.createElement("div");
    div.style.width = 0;
    div.style.height = 0;
    div.style.left = cx + "px";
    div.style.top = cy + "px";
    div.className = "circle";
    document.body.append(div);

    setTimeout(() => {
      div.style.width = radius * 2 + "px";
      div.style.height = radius * 2 + "px";
      div.addEventListener("transitionend", function handler() {
        div.removeEventListener("transitionend", handler);
        resolve(div);
      });
    }, 0);
  });
}

// Usage
showCircle(150, 150, 100).then((div) => {
  div.classList.add("message-ball");
  div.append("Hello, world!");
});
```

## Summary

| Method                           | Purpose                    |
| -------------------------------- | -------------------------- |
| `new Promise(executor)`          | Create promise             |
| `resolve(value)`                 | Fulfill promise with value |
| `reject(error)`                  | Reject promise with error  |
| `.then(onFulfilled, onRejected)` | Handle result/error        |
| `.catch(onRejected)`             | Handle error only          |
| `.finally(onFinally)`            | Always runs (cleanup)      |

**Key Points:**

- Promise state changes only once (pending → fulfilled or pending → rejected)
- Handlers can be added anytime (run immediately if already settled)
- Promises enable cleaner asynchronous code than callbacks
- Chaining (next chapter) allows sequential async operations without nesting
