# Private and Protected Properties and Methods

## Overview

Encapsulation separates internal interface from external interface, making code more reliable and maintainable.

## Internal vs External Interface

- **Internal interface** – methods/properties accessible only inside the class (for internal workings)
- **External interface** – methods/properties accessible from outside

## Protected Properties (Convention)

Protected fields start with `_` (underscore). This is a **convention**, not enforced by the language.

```javascript
class CoffeeMachine {
  _waterAmount = 0;

  set waterAmount(value) {
    if (value < 0) value = 0;
    this._waterAmount = value;
  }

  get waterAmount() {
    return this._waterAmount;
  }

  constructor(power) {
    this._power = power;
  }

  get power() {
    return this._power; // Read-only (no setter)
  }
}

let machine = new CoffeeMachine(100);
machine.waterAmount = -10; // Becomes 0
alert(machine.power); // 100
machine.power = 25; // Error (no setter)
```

### Getter/Setter Functions (Alternative)

```javascript
class CoffeeMachine {
  _waterAmount = 0;

  setWaterAmount(value) {
    if (value < 0) value = 0;
    this._waterAmount = value;
  }

  getWaterAmount() {
    return this._waterAmount;
  }
}
```

### Protected Fields Are Inherited

```javascript
class MegaCoffeeMachine extends CoffeeMachine {
  method() {
    alert(this._waterAmount); // Accessible in child class
  }
}
```

## Private Properties (`#` – Language-Enforced)

Private fields start with `#`. They are only accessible **inside the class** (not even in inheriting classes).

```javascript
class CoffeeMachine {
  #waterLimit = 200;

  #fixWaterAmount(value) {
    if (value < 0) return 0;
    if (value > this.#waterLimit) return this.#waterLimit;
    return value;
  }

  setWaterAmount(value) {
    this.#waterLimit = this.#fixWaterAmount(value);
  }
}

let machine = new CoffeeMachine();
machine.#waterLimit = 1000; // Error
machine.#fixWaterAmount(123); // Error
```

### Private Fields with Public Accessors

```javascript
class CoffeeMachine {
  #waterAmount = 0;

  get waterAmount() {
    return this.#waterAmount;
  }

  set waterAmount(value) {
    if (value < 0) value = 0;
    this.#waterAmount = value;
  }
}

let machine = new CoffeeMachine();
machine.waterAmount = 100; // Works (uses setter)
alert(machine.#waterAmount); // Error (private)
```

### Private Fields Are Not Inherited

```javascript
class MegaCoffeeMachine extends CoffeeMachine {
  method() {
    alert(this.#waterAmount); // Error: not accessible
    // Must use public getter instead
    alert(this.waterAmount); // OK
  }
}
```

### Private Fields Cannot Be Accessed via `this[name]`

```javascript
class User {
  #name = "John";

  sayHi() {
    let fieldName = "name";
    alert(this[fieldName]); // Works for public
    alert(this["#name"]); // Doesn't work for private
  }
}
```

## Comparison

| Type      | Naming  | Access             | Inheritance | Language Enforced |
| --------- | ------- | ------------------ | ----------- | ----------------- |
| Public    | `name`  | Anywhere           | Yes         | No                |
| Protected | `_name` | Class + subclasses | Yes         | No (convention)   |
| Private   | `#name` | Only class         | No          | Yes               |

## Benefits of Encapsulation

- **Protection** – Users can't accidentally break internal logic
- **Maintainability** – Internal code can be changed without affecting users
- **Simplicity** – External interface is clear and simple

## Summary

| Feature                | Syntax      | Access             |
| ---------------------- | ----------- | ------------------ |
| Public                 | `property`  | Anywhere           |
| Protected (convention) | `_property` | Class + subclasses |
| Private (language)     | `#property` | Only class         |

**Key Points:**

- Protected uses `_` convention (not enforced)
- Private uses `#` (language-enforced, recent addition)
- Private fields are not inherited
- Use encapsulation to hide internal complexity and prevent misuse
