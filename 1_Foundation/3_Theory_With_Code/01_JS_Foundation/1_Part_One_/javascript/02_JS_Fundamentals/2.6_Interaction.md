# Interaction: alert, prompt, confirm

Three browser-specific functions for interacting with users. All are **modal** - they pause script execution and prevent interaction with the rest of the page until dismissed.

## alert

Shows a message and waits for the user to press "OK".

```javascript
alert("Hello");
```

## prompt

Shows a message with an input field, OK button, and Cancel button.

```javascript
result = prompt(title, [default]);
```

- `title` - text to show the visitor
- `default` (optional) - initial value for the input field

**Returns:**

- Text from input field if OK is pressed
- `null` if Cancel or Esc is pressed

```javascript
let age = prompt("How old are you?", 100);
alert(`You are ${age} years old!`); // You are 100 years old!
```

### IE Compatibility Note

Internet Explorer inserts `"undefined"` if no default is provided:

```javascript
let test = prompt("Test"); // In IE: shows "undefined" as default
let test = prompt("Test", ""); // Recommended: always provide second argument
```

## confirm

Shows a message with OK and Cancel buttons.

```javascript
result = confirm(question);
```

**Returns:**

- `true` if OK is pressed
- `false` if Cancel or Esc is pressed

```javascript
let isBoss = confirm("Are you the boss?");
alert(isBoss); // true if OK pressed
```

## Summary Table

| Function                 | Purpose          | Return Value     |
| ------------------------ | ---------------- | ---------------- |
| `alert(message)`         | Show message     | (none)           |
| `prompt(title, default)` | Get text input   | Text or `null`   |
| `confirm(question)`      | Get confirmation | `true` / `false` |

## Limitations

- **Location:** Determined by browser (usually center)
- **Appearance:** Cannot be modified (browser-dependent)

These methods are simple and work well when "bells and whistles" don't matter. For richer interfaces, other methods exist.
