# Web Components Summary

Web Components are a set of browser standards for creating reusable, encapsulated custom HTML elements with their own DOM and styles.

## Core Standards

| Standard            | Purpose                                          |
| ------------------- | ------------------------------------------------ |
| **Custom Elements** | Define new HTML tags with custom behavior        |
| **Shadow DOM**      | Encapsulated DOM tree, hidden from main document |
| **CSS Scoping**     | Styles that only apply inside Shadow DOM         |
| **HTML Templates**  | Declarative fragments of reusable markup         |

## Why Web Components?

Component architecture splits complex UI into independent, reusable pieces:

- Each component has its own JavaScript class
- Own DOM structure (encapsulated)
- Own CSS styles (scoped)
- Clear API for interaction

**Benefits:**

- Reusability across projects
- Encapsulation (no style/selector conflicts)
- Maintainability
- Framework-agnostic

## Component Structure Example

```html
<twitter-user-info name="John" avatar="john.jpg">
  <!-- Component internals hidden in Shadow DOM -->
</twitter-user-info>
```

```javascript
class TwitterUserInfo extends HTMLElement {
  constructor() {
    super();
    // Attach Shadow DOM
    const shadow = this.attachShadow({ mode: "open" });

    // Component template
    shadow.innerHTML = `
      <style>
        .card { border: 1px solid #ccc; padding: 10px; }
        img { width: 50px; border-radius: 50%; }
      </style>
      <div class="card">
        <img src="${this.getAttribute("avatar")}">
        <span>${this.getAttribute("name")}</span>
      </div>
    `;
  }
}

// Register custom element
customElements.define("twitter-user-info", TwitterUserInfo);
```

## Component Architecture Principles

1. **Splitting** - Break complex UI into simpler parts
2. **Encapsulation** - Internal details hidden from outside
3. **Composition** - Components can contain subcomponents
4. **Clear API** - Defined methods, properties, and events
