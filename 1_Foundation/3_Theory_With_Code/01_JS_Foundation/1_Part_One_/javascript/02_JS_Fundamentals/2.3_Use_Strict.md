# The Modern Mode: "use strict"

JavaScript evolved without breaking existing code, but imperfect decisions got stuck forever. In 2009, ECMAScript 5 (ES5) added new features and modified some existing ones. To keep old code working, most modifications are off by default - you enable them with `"use strict"`.

## The Directive

```javascript
"use strict";

// This code works the modern way
...
```

- Looks like a string: `"use strict"` or `'use strict'`
- When at the top of a script, the whole script works the "modern" way
- Can also be placed at the beginning of a function (enables strict mode for that function only)

## Important Rules

### Must be at the top

```javascript
alert("some code");
// "use strict" below is ignored - it must be at the top

("use strict");

// strict mode is NOT activated
```

**Only comments may appear above `"use strict"`.**

### No way to cancel

There is no `"no use strict"` directive. Once you enter strict mode, there's no going back.

## Browser Console

The developer console does NOT use `use strict` by default.

**To use strict in the console:**

Method 1 (works in Firefox and Chrome):

```javascript
'use strict'; <Shift+Enter for a newline>
// ...your code...
<Enter to run>
```

Method 2 (old browser fallback):

```javascript
(function () {
  "use strict";

  // ...your code here...
})();
```

## Should We Use "use strict"?

Modern JavaScript supports **classes** and **modules** - advanced structures that enable `use strict` automatically. So you don't need the directive if using them.

| Approach        | Need "use strict"? |
| --------------- | ------------------ |
| Regular scripts | Yes (recommended)  |
| Classes/Modules | No (automatic)     |

## Summary

- `"use strict"` enables modern JavaScript mode
- Must be at the top of script (only comments above allowed)
- Cannot be canceled once enabled
- Browser console doesn't use strict by default (use workarounds)
- All examples in this tutorial assume strict mode unless specified otherwise
- For now, use it. Later with classes/modules, you may omit it
