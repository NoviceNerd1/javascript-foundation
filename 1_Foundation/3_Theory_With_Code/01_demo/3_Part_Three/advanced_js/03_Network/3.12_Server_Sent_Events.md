# Server Sent Events (EventSource) Summary

`EventSource` provides a persistent one-way connection from server to client over regular HTTP. It's simpler than WebSocket and includes built-in auto-reconnection.

## Basic Usage

```javascript
// Create connection
let eventSource = new EventSource("/events/subscribe");

// Receive messages
eventSource.onmessage = function (event) {
  console.log("Message:", event.data);
};

// Connection opened
eventSource.onopen = function () {
  console.log("Connection established");
};

// Error handling
eventSource.onerror = function () {
  console.log("Connection error (will auto-reconnect)");
};

// Close connection
eventSource.close();
```

## Server Response Format

Server must respond with `Content-Type: text/event-stream` and send messages in this format:

```
data: Message 1

data: Message 2

data: Message 3
data: of two lines

data: {"user":"John","message":"Hello"}
```

**Rules:**

- Each message starts with `data:`
- Messages separated by double newline `\n\n`
- Multiple `data:` lines become a single message with `\n` between lines
- Space after colon is optional

## Message ID for Reconnection

```javascript
// Server sends with ID
data: Message 1
id: 1

data: Message 2
id: 2

// Browser tracks last ID
console.log(eventSource.lastEventId); // "2"
// On reconnect, browser sends: Last-Event-ID: 2
```

## Custom Events

```javascript
// Server sends custom event type
event: join;
data: Bob;

event: leave;
data: Alice;

// Client handles custom events
eventSource.addEventListener("join", (event) => {
  console.log(`${event.data} joined`);
});

eventSource.addEventListener("leave", (event) => {
  console.log(`${event.data} left`);
});

// Default message handler still works
eventSource.onmessage = (event) => {
  console.log("Regular message:", event.data);
};
```

## Reconnection Control

```javascript
// Server sets custom retry delay (in milliseconds)
retry: 15000
data: Connection will retry in 15 seconds if lost

// Stop reconnecting (server returns HTTP 204)
```

## Cross-Origin Requests

```javascript
// Basic cross-origin
let source = new EventSource("https://another-site.com/events");

// With credentials
let source = new EventSource("https://another-site.com/events", {
  withCredentials: true,
});
```

Server must respond with `Access-Control-Allow-Origin` header.

## Connection States

```javascript
console.log(eventSource.readyState);
// 0 = CONNECTING - connecting or reconnecting
// 1 = OPEN - connected
// 2 = CLOSED - connection closed

// Check state
if (eventSource.readyState === EventSource.CONNECTING) {
  console.log("Connecting...");
}
```

## Complete Example

### Server (Node.js)

```javascript
const express = require("express");
const app = express();

app.get("/events", (req, res) => {
  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");
  res.setHeader("Connection", "keep-alive");

  let id = 0;

  // Send message every 2 seconds
  const interval = setInterval(() => {
    id++;
    res.write(`id: ${id}\n`);
    res.write(
      `data: ${JSON.stringify({
        time: new Date().toISOString(),
        value: Math.random(),
      })}\n\n`,
    );
  }, 2000);

  // Stop on client disconnect
  req.on("close", () => {
    clearInterval(interval);
    res.end();
  });
});

app.listen(3000);
```

### Client (HTML + JS)

```html
<div id="messages"></div>

<script>
  let eventSource = new EventSource("/events");

  eventSource.onopen = function () {
    console.log("Connected");
  };

  eventSource.onmessage = function (event) {
    let data = JSON.parse(event.data);
    let div = document.createElement("div");
    div.textContent = `[${data.time}] ${data.value}`;
    document.getElementById("messages").prepend(div);
  };

  eventSource.onerror = function () {
    console.log("Connection lost, auto-reconnecting...");
  };

  // Optional: manual close after 30 seconds
  setTimeout(() => {
    eventSource.close();
    console.log("Connection closed");
  }, 30000);
</script>
```

## EventSource vs WebSocket

| Feature         | EventSource          | WebSocket                |
| --------------- | -------------------- | ------------------------ |
| Direction       | Server → Client only | Bidirectional            |
| Data types      | Text only            | Text + Binary            |
| Protocol        | Regular HTTP         | WebSocket protocol       |
| Auto-reconnect  | ✅ Built-in          | ❌ Manual implementation |
| Message IDs     | ✅ Built-in          | ❌ Manual                |
| Browser support | Modern (no IE)       | Modern                   |
| Complexity      | Simple               | More complex             |

## EventSource Properties & Methods

| Property/Method | Description                      |
| --------------- | -------------------------------- |
| `readyState`    | Current connection state (0,1,2) |
| `lastEventId`   | Last received message ID         |
| `close()`       | Close the connection             |
| `onmessage`     | Handler for messages             |
| `onopen`        | Handler for connection open      |
| `onerror`       | Handler for errors               |

## Server Response Fields

| Field    | Description                                     |
| -------- | ----------------------------------------------- |
| `data:`  | Message content (can be multiple lines)         |
| `id:`    | Message ID (sent in Last-Event-ID on reconnect) |
| `retry:` | Reconnection delay in milliseconds              |
| `event:` | Custom event name (default "message")           |

## When to Use EventSource

**Good for:**

- Live feeds (news, stock prices)
- Notifications
- Log streaming
- Simple real-time updates
- Applications where server → client communication is sufficient

**Not ideal for:**

- Two-way communication (games, chat with user input)
- Binary data transmission
- Applications requiring IE support

## Summary

1. **Create**: `new EventSource(url, [options])`
2. **Receive**: `onmessage` or `addEventListener` for custom events
3. **Server format**: `data: content\n\n` with optional `id:`, `event:`, `retry:`
4. **Auto-reconnect**: Built-in, controlled by `retry:` or HTTP 204
5. **State**: `readyState` property (0/1/2)
6. **Close**: `eventSource.close()`
7. **Cross-origin**: Supported with CORS, optional `withCredentials`
