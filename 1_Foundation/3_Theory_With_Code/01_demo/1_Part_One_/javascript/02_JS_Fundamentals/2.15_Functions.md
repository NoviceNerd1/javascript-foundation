# Functions in JavaScript

## Function Declaration

```javascript
function showMessage() {
  alert("Hello everyone!");
}

showMessage(); // Call the function
```

## Local Variables

Variables declared inside a function are only visible inside it.

```javascript
function showMessage() {
  let message = "Hello!"; // Local variable
  alert(message);
}

showMessage();
alert(message); // Error: message is not defined
```

## Outer Variables

Functions can access and modify outer (global) variables.

```javascript
let userName = "John";

function showMessage() {
  userName = "Bob"; // Modifies outer variable
  alert("Hello, " + userName);
}

showMessage(); // Hello, Bob
alert(userName); // Bob
```

### Variable Shadowing

If a local variable has the same name as an outer variable, it shadows the outer one.

```javascript
let userName = "John";

function showMessage() {
  let userName = "Bob"; // Local variable shadows outer
  alert("Hello, " + userName); // Bob
}

showMessage();
alert(userName); // John (unchanged)
```

## Parameters (Arguments)

```javascript
function showMessage(from, text) {
  alert(from + ": " + text);
}

showMessage("Ann", "Hello!"); // Ann: Hello!
```

### Parameters are Copies

Functions receive copies of values; changes don't affect outer variables.

```javascript
function showMessage(from, text) {
  from = "*" + from + "*";
  alert(from + ": " + text);
}

let from = "Ann";
showMessage(from, "Hello"); // *Ann*: Hello
alert(from); // Ann (unchanged)
```

## Default Parameters

```javascript
function showMessage(from, text = "no text given") {
  alert(from + ": " + text);
}

showMessage("Ann"); // Ann: no text given
showMessage("Ann", undefined); // Ann: no text given
showMessage("Ann", "Hello"); // Ann: Hello
```

### Alternative Default Values

```javascript
// Using ||
function showMessage(text) {
  text = text || "empty";
  alert(text);
}

// Using ?? (better for 0, false, etc.)
function showCount(count) {
  alert(count ?? "unknown");
}
showCount(0); // 0
showCount(null); // unknown
```

## Returning Values

```javascript
function sum(a, b) {
  return a + b;
}

let result = sum(1, 2);
alert(result); // 3
```

### Multiple Returns

```javascript
function checkAge(age) {
  if (age >= 18) {
    return true;
  } else {
    return confirm("Do you have permission?");
  }
}
```

### Empty Return (returns undefined)

```javascript
function doNothing() {
  return; // Returns undefined
}

alert(doNothing() === undefined); // true
```

> **Important:** Don't add a newline between `return` and its value:
>
> ```javascript
> return; // Wrong - becomes empty return
> some + expression;
>
> return (
>   // Correct
>   some + expression
> );
> ```

## Function Naming

- Use verbs: `showMessage`, `getAge`, `calcSum`
- Common prefixes:
  - `show...` – displays something
  - `get...` – returns a value
  - `calc...` – calculates something
  - `create...` – creates something
  - `check...` – returns boolean

## One Function = One Action

Functions should do exactly what their name suggests.

**Bad:**

```javascript
function getAge() {
  let age = prompt("Enter age");
  alert(age); // Should only get, not display
  return age;
}
```

**Good:**

```javascript
function getAge() {
  return prompt("Enter age");
}
```

## Functions as Comments (Self-Documenting Code)

```javascript
// Without helper function - harder to read
function showPrimes(n) {
  nextPrime: for (let i = 2; i < n; i++) {
    for (let j = 2; j < i; j++) {
      if (i % j == 0) continue nextPrime;
    }
    alert(i);
  }
}

// With helper function - self-describing
function showPrimes(n) {
  for (let i = 2; i < n; i++) {
    if (!isPrime(i)) continue;
    alert(i);
  }
}

function isPrime(n) {
  for (let i = 2; i < n; i++) {
    if (n % i == 0) return false;
  }
  return true;
}
```

## Tasks

### 1. Is "else" Required?

```javascript
// Both variants work identically
function checkAge(age) {
  if (age > 18) {
    return true;
  }
  return confirm("Did parents allow you?");
}
// else is not required because return exits the function
```

### 2. Rewrite Using `?` or `||`

```javascript
// Using ? operator
function checkAge(age) {
  return age > 18 ? true : confirm("Did parents allow you?");
}

// Using || operator
function checkAge(age) {
  return age > 18 || confirm("Did parents allow you?");
}
```

### 3. Function `min(a, b)`

```javascript
function min(a, b) {
  return a < b ? a : b;
}

min(2, 5) == 2;
min(3, -1) == -1;
min(1, 1) == 1;
```

### 4. Function `pow(x, n)`

```javascript
function pow(x, n) {
  let result = 1;
  for (let i = 0; i < n; i++) {
    result *= x;
  }
  return result;
}

let x = prompt("x?", "");
let n = prompt("n?", "");

if (n >= 1 && n % 1 == 0) {
  alert(pow(x, n));
} else {
  alert("Power n must be a positive integer");
}
```

## Summary

| Concept            | Description                         |
| ------------------ | ----------------------------------- |
| `function name()`  | Declares a function                 |
| Parameters         | Variables listed in declaration     |
| Arguments          | Values passed when calling          |
| Local variables    | Visible only inside function        |
| Global variables   | Visible everywhere (minimize usage) |
| `return`           | Returns value, exits function       |
| Default parameters | Set with `=` in declaration         |
