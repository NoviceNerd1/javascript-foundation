# Cross-Window Communication Summary

The **Same Origin Policy** restricts windows/frames from different origins from accessing each other's content to prevent information theft.

## Same Origin Definition

Two URLs share the same origin when they have identical:

- Protocol (`http` vs `https`)
- Domain (`site.com` vs `www.site.com`)
- Port (`:8080` vs default)

## Iframe Access

### Getting References

```javascript
// Access iframe content
let iframeWindow = iframe.contentWindow; // window object
let iframeDoc = iframe.contentDocument; // document object

// Alternative via frames collection
window.frames[0]; // by index
window.frames.iframeName; // by name
window.parent; // parent window
window.top; // topmost parent
```

### Cross-Origin Restrictions

```javascript
<iframe src="https://other-site.com" id="iframe"></iframe>

<script>
  iframe.onload = function() {
    let ref = iframe.contentWindow;     // ✅ Allowed
    iframe.contentDocument;              // ❌ Error
    iframe.contentWindow.location.href;  // ❌ Cannot read

    // ✅ Can write to location (redirect)
    iframe.contentWindow.location = '/';
  };
</script>
```

## Same-Origin Techniques

### document.domain (Subdomains)

```javascript
// On both pages: john.site.com and peter.site.com
document.domain = "site.com"; // Now they can interact freely
```

⚠️ Deprecated but still supported. Use `postMessage` instead.

### iframe Loading Pitfall

```javascript
<iframe src="/" id="iframe"></iframe>

<script>
  let oldDoc = iframe.contentDocument;

  // Check for document change
  let timer = setInterval(() => {
    let newDoc = iframe.contentDocument;
    if (newDoc !== oldDoc) {
      alert('Document loaded!');
      clearInterval(timer);
    }
  }, 100);
</script>
```

## Sandbox Attribute

Restricts iframe capabilities for security:

```html
<!-- Strictest restrictions -->
<iframe sandbox src="untrusted.html"></iframe>

<!-- Relax specific restrictions -->
<iframe sandbox="allow-scripts allow-forms" src="page.html"></iframe>
```

**Common sandbox flags:**

- `allow-same-origin` - treat as same origin
- `allow-scripts` - run JavaScript
- `allow-forms` - submit forms
- `allow-popups` - use `window.open`
- `allow-top-navigation` - change `parent.location`

## Cross-Window Messaging (postMessage)

Allows communication between any windows regardless of origin.

### Sending Messages

```javascript
// Send to target window
targetWindow.postMessage(data, targetOrigin);

// Examples
let iframeWin = document.querySelector("iframe").contentWindow;
iframeWin.postMessage("Hello!", "https://example.com");

// Allow any origin (less secure)
iframeWin.postMessage("Hello!", "*");
```

### Receiving Messages

```javascript
window.addEventListener("message", function (event) {
  // Verify sender origin for security
  if (event.origin !== "https://trusted-site.com") return;

  console.log("Received:", event.data);
  console.log("From:", event.origin);

  // Reply using source
  event.source.postMessage("Got it!", event.origin);
});
```

**Event properties:**

- `data` - the sent data
- `origin` - sender's origin
- `source` - reference to sender window

## Window Hierarchy References

```javascript
// Check if in iframe
if (window === top) {
  console.log("Topmost window");
} else {
  console.log("Inside a frame");
}
```

## Summary Table

| Access Type      | Same Origin    | Different Origin             |
| ---------------- | -------------- | ---------------------------- |
| Read content     | ✅ Full access | ❌ Blocked                   |
| Write content    | ✅ Full access | ❌ Blocked (except location) |
| Change location  | ✅ Yes         | ✅ Yes (write only)          |
| Send messages    | ✅ Yes         | ✅ Yes (via postMessage)     |
| Reference window | ✅ Yes         | ✅ Yes                       |
