# Function Expressions in JavaScript

## Overview

In JavaScript, functions are **values**. They can be assigned, copied, and passed around like any other value.

## Function Declaration vs Function Expression

### Function Declaration

```javascript
function sayHi() {
  alert("Hello");
}
```

### Function Expression

```javascript
let sayHi = function () {
  alert("Hello");
};
```

## Function is a Value

```javascript
function sayHi() {
  alert("Hello");
}

alert(sayHi); // Shows function code (not executed)

// Copy function to another variable
let func = sayHi;
func(); // Hello
sayHi(); // Hello
```

> **Note:** No parentheses means no execution. `sayHi` references the function; `sayHi()` calls it.

### Semicolon in Function Expression

```javascript
let sayHi = function () {
  alert("Hello");
}; // ← Semicolon ends the assignment statement
```

## Callback Functions

Passing functions as arguments to be "called back" later.

```javascript
function ask(question, yes, no) {
  if (confirm(question)) yes();
  else no();
}

// Using named functions
function showOk() {
  alert("You agreed.");
}
function showCancel() {
  alert("You canceled.");
}

ask("Do you agree?", showOk, showCancel);

// Using anonymous functions directly
ask(
  "Do you agree?",
  function () {
    alert("You agreed.");
  },
  function () {
    alert("You canceled.");
  },
);
```

## Key Differences

| Feature       | Function Declaration            | Function Expression       |
| ------------- | ------------------------------- | ------------------------- |
| Syntax        | Separate statement              | Part of expression        |
| Creation time | Before code execution (hoisted) | When execution reaches it |
| Availability  | Anywhere in scope               | Only after definition     |
| Block scope   | Block-scoped (strict mode)      | Follows variable scope    |

### Hoisting Example

```javascript
// Works - Function Declaration is hoisted
sayHi("John"); // Hello, John

function sayHi(name) {
  alert(`Hello, ${name}`);
}
```

```javascript
// Fails - Function Expression not hoisted
sayHi("John"); // Error!

let sayHi = function (name) {
  alert(`Hello, ${name}`);
};
```

### Block Scope Example

```javascript
let age = prompt("What is your age?", 18);

// Function Declaration is block-scoped
if (age < 18) {
  function welcome() {
    alert("Hello!");
  }
} else {
  function welcome() {
    alert("Greetings!");
  }
}

welcome(); // Error: welcome is not defined
```

### Solution: Use Function Expression

```javascript
let age = prompt("What is your age?", 18);
let welcome;

if (age < 18) {
  welcome = function () {
    alert("Hello!");
  };
} else {
  welcome = function () {
    alert("Greetings!");
  };
}

welcome(); // Works

// Or using ternary operator
let welcome =
  age < 18
    ? function () {
        alert("Hello!");
      }
    : function () {
        alert("Greetings!");
      };
```

## Summary

| Aspect      | Function Declaration     | Function Expression                |
| ----------- | ------------------------ | ---------------------------------- |
| Definition  | `function name() {}`     | `let name = function() {};`        |
| Hoisting    | Yes (fully hoisted)      | No (only variable hoisted)         |
| Block scope | Yes (in strict mode)     | No (follows variable scope)        |
| Use case    | Preferred for most cases | Needed for conditional definitions |

**Best Practice:** Use Function Declaration by default. Use Function Expression when conditional declaration or runtime creation is required.
