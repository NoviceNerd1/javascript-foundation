# Shadow DOM Slots & Composition Summary

Slots allow mixing light DOM content into shadow DOM at specific insertion points, enabling flexible component composition.

## Basic Slot Usage

```html
<user-card>
  <span slot="username">John Smith</span>
  <span slot="birthday">01.01.2001</span>
</user-card>

<script>
  customElements.define(
    "user-card",
    class extends HTMLElement {
      connectedCallback() {
        this.attachShadow({ mode: "open" });
        this.shadowRoot.innerHTML = `
      <div>Name: <slot name="username"></slot></div>
      <div>Birthday: <slot name="birthday"></slot></div>
    `;
      }
    },
  );
</script>
```

## Slot Types

| Slot Type    | Syntax                 | Description                             |
| ------------ | ---------------------- | --------------------------------------- |
| **Named**    | `<slot name="X">`      | Gets light DOM elements with `slot="X"` |
| **Default**  | `<slot>`               | Gets all unslotted light DOM children   |
| **Fallback** | `<slot>default</slot>` | Shown when no matching content exists   |

## Flattened DOM

The browser creates a "flattened" DOM for rendering by inserting slotted elements into slots:

```html
<user-card>
  #shadow-root
  <div>
    Name:
    <slot name="username">
      <span slot="username">John Smith</span>
      <!-- inserted here -->
    </slot>
  </div>
</user-card>
```

**Important:** Nodes are not physically moved. JavaScript still sees them in their original positions.

## Slot Rules

### 1. Only Top-Level Children

```html
<user-card>
  <span slot="username">Works</span>
  <!-- ✓ Valid -->
  <div>
    <span slot="birthday">Ignored</span>
    <!-- ✗ Not top-level -->
  </div>
</user-card>
```

### 2. Multiple Elements for Same Slot

```html
<user-card>
  <span slot="username">John</span>
  <span slot="username">Smith</span>
</user-card>
<!-- Both appear in the slot, one after another -->
```

### 3. Default Slot

```html
<user-card>
  <div>Unslotted content goes to default slot</div>
  <span slot="username">John</span>
</user-card>
```

```javascript
this.shadowRoot.innerHTML = `
  <slot name="username"></slot>
  <slot></slot>  <!-- Gets unslotted content -->
`;
```

## Fallback Content

```html
<slot name="username">Anonymous</slot>
<!-- Shows "Anonymous" if no slot="username" in light DOM -->
```

## Complete Menu Example

```html
<template id="menu-tmpl">
  <style>
    .menu {
      border: 1px solid #ccc;
    }
    .menu[closed] ul {
      display: none;
    }
    .menu-title {
      cursor: pointer;
      background: #eee;
      padding: 5px;
    }
    ul {
      margin: 0;
      padding: 5px 20px;
    }
  </style>
  <div class="menu" closed>
    <div class="menu-title"><slot name="title">Menu</slot></div>
    <ul>
      <slot name="item"></slot>
    </ul>
  </div>
</template>

<script>
  customElements.define(
    "custom-menu",
    class extends HTMLElement {
      connectedCallback() {
        this.attachShadow({ mode: "open" });
        let tmpl = document.getElementById("menu-tmpl");
        this.shadowRoot.append(tmpl.content.cloneNode(true));

        this.shadowRoot.querySelector(".menu-title").onclick = () => {
          let menu = this.shadowRoot.querySelector(".menu");
          menu.toggleAttribute("closed");
        };
      }
    },
  );
</script>

<custom-menu>
  <span slot="title">Candy menu</span>
  <li slot="item">Lollipop</li>
  <li slot="item">Fruit Toast</li>
  <li slot="item">Cup Cake</li>
</custom-menu>
```

## Slotchange Event

Triggers when slotted content changes:

```javascript
this.shadowRoot.querySelector("slot").addEventListener("slotchange", (e) => {
  console.log("Slot changed:", e.target.name);
  let slot = e.target;
  let assigned = slot.assignedElements();
  console.log("Assigned elements:", assigned);
});
```

**Note:** Does NOT trigger when content inside slotted element changes (use MutationObserver for that).

## Slot API Methods

| Method                                | Description                        |
| ------------------------------------- | ---------------------------------- |
| `slot.assignedNodes()`                | DOM nodes assigned to slot         |
| `slot.assignedElements()`             | Element nodes assigned to slot     |
| `slot.assignedNodes({flatten: true})` | Includes nested slots and fallback |
| `node.assignedSlot`                   | Returns slot node is assigned to   |

### Example: Tracking Slot Items

```javascript
class CustomMenu extends HTMLElement {
  items = [];

  connectedCallback() {
    this.attachShadow({ mode: "open" });
    this.shadowRoot.innerHTML = `
      <div class="menu">
        <slot name="title"></slot>
        <ul><slot name="item"></slot></ul>
      </div>
    `;

    this.shadowRoot.addEventListener("slotchange", (e) => {
      let slot = e.target;
      if (slot.name === "item") {
        this.items = slot.assignedElements().map((el) => el.textContent);
        console.log("Items:", this.items);
      }
    });
  }
}
```

## Dynamic Slot Updates

Slots automatically update when light DOM changes:

```javascript
let menu = document.querySelector("custom-menu");

// Auto-updates rendering
setTimeout(() => {
  menu.insertAdjacentHTML("beforeend", '<li slot="item">New Item</li>');
}, 1000);
```

## Multiple Unnamed Slots

Only the first unnamed slot works; subsequent ones are ignored:

```html
<!-- Works: first unnamed slot gets all unslotted content -->
<slot></slot>
<slot></slot>
<!-- Ignored -->
```

## Summary

| Concept           | Description                                                        |
| ----------------- | ------------------------------------------------------------------ |
| **Named slots**   | `<slot name="X">` with matching `slot="X"` in light DOM            |
| **Default slot**  | First unnamed `<slot>` gets unslotted content                      |
| **Fallback**      | Content inside `<slot>` shows when no matching elements            |
| **Flattened DOM** | Virtual DOM for rendering; actual DOM unchanged                    |
| **slotchange**    | Event when slot content changes (add/remove, not internal changes) |
| **API**           | `assignedNodes()`, `assignedElements()`, `assignedSlot`            |

**Key Points:**

- Slots enable component composition without moving DOM nodes
- Only direct children of the host can be slotted
- Multiple elements with same slot name are appended in order
- Default slot captures all remaining unslotted content
- Use `slotchange` event to react to slot content changes
- Use MutationObserver for deeper content changes inside slotted elements
