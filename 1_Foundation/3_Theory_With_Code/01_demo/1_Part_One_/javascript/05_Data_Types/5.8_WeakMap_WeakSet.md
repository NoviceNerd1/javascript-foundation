# WeakMap and WeakSet

## Overview

WeakMap and WeakSet are collections that hold **weak references** to objects, allowing garbage collection when objects are no longer reachable.

## WeakMap

### Key Characteristics

- **Keys must be objects** (not primitives)
- Keys are **weakly referenced** – if an object has no other references, it's removed from memory and the WeakMap
- **Not iterable** – no `keys()`, `values()`, `entries()`, `forEach()`
- No `size` property

### Methods

| Method                    | Description          |
| ------------------------- | -------------------- |
| `weakMap.set(key, value)` | Stores value by key  |
| `weakMap.get(key)`        | Returns value by key |
| `weakMap.delete(key)`     | Removes element      |
| `weakMap.has(key)`        | Checks existence     |

### Example

```javascript
let weakMap = new WeakMap();

let obj = {};
weakMap.set(obj, "ok"); // Works (object key)

// Error: string key not allowed
weakMap.set("test", "Whoops");

// Auto-removal
let john = { name: "John" };
weakMap.set(john, "secret");
john = null; // john is now removed from WeakMap automatically
```

### Use Case 1: Additional Data Storage

Store data for objects managed by external code – data auto-clears when object is gone.

```javascript
// 📁 visitsCount.js
let visitsCountMap = new WeakMap();

function countUser(user) {
  let count = visitsCountMap.get(user) || 0;
  visitsCountMap.set(user, count + 1);
}

// 📁 main.js
let john = { name: "John" };
countUser(john); // Count his visits

john = null; // User data auto-removed from WeakMap
```

### Use Case 2: Caching

```javascript
let cache = new WeakMap();

function process(obj) {
  if (!cache.has(obj)) {
    let result = /* heavy calculation */ obj;
    cache.set(obj, result);
  }
  return cache.get(obj);
}

let obj = { data: "some data" };
let result = process(obj);
obj = null; // Cache entry auto-removed
```

## WeakSet

### Key Characteristics

- **Only objects** can be added (not primitives)
- Objects are **weakly referenced**
- **Not iterable** – no loops, `size`, or `keys()`
- No `size` property

### Methods

| Method                | Description        |
| --------------------- | ------------------ |
| `weakSet.add(obj)`    | Adds object to set |
| `weakSet.delete(obj)` | Removes object     |
| `weakSet.has(obj)`    | Checks existence   |

### Example: Track Visits

```javascript
let visitedSet = new WeakSet();

let john = { name: "John" };
let pete = { name: "Pete" };
let mary = { name: "Mary" };

visitedSet.add(john);
visitedSet.add(pete);
visitedSet.add(john); // Duplicate ignored

alert(visitedSet.has(john)); // true
alert(visitedSet.has(mary)); // false

john = null; // John removed from WeakSet automatically
```

## Comparison

| Feature            | Map      | WeakMap                       | Set      | WeakSet                |
| ------------------ | -------- | ----------------------------- | -------- | ---------------------- |
| Key type           | Any      | Objects only                  | Any      | Objects only           |
| Value type         | Any      | Any                           | Any      | -                      |
| Garbage collection | Prevents | Allows                        | Prevents | Allows                 |
| Iterable           | Yes      | No                            | Yes      | No                     |
| `size` property    | Yes      | No                            | Yes      | No                     |
| Methods            | All      | `set`, `get`, `has`, `delete` | All      | `add`, `has`, `delete` |

## When to Use

| Scenario                                 | Recommended        |
| ---------------------------------------- | ------------------ |
| Store extra data tied to object lifetime | **WeakMap**        |
| Cache results that expire with objects   | **WeakMap**        |
| Track "yes/no" facts about objects       | **WeakSet**        |
| Need iteration or size                   | **Map** or **Set** |

## Tasks

### 1. Store "Unread" Flags

```javascript
// Use WeakSet to track read messages
let readMessages = new WeakSet();

function markAsRead(message) {
  readMessages.add(message);
}

function isRead(message) {
  return readMessages.has(message);
}

let messages = [
  { text: "Hello", from: "John" },
  { text: "How goes?", from: "John" },
  { text: "See you soon", from: "Alice" },
];

markAsRead(messages[0]);
alert(isRead(messages[0])); // true
alert(isRead(messages[1])); // false

// When message is removed from array, it auto-disappears from WeakSet
```

### 2. Store Read Dates

```javascript
// Use WeakMap to store read dates
let readDates = new WeakMap();

function markAsReadWithDate(message) {
  readDates.set(message, new Date());
}

function getReadDate(message) {
  return readDates.get(message);
}

let messages = [
  { text: "Hello", from: "John" },
  { text: "How goes?", from: "John" },
  { text: "See you soon", from: "Alice" },
];

markAsReadWithDate(messages[0]);
alert(getReadDate(messages[0])); // Date object
alert(getReadDate(messages[1])); // undefined

// When message is removed from array, its read date auto-disappears
```

## Summary

| Collection  | Purpose                              | Key Feature                      |
| ----------- | ------------------------------------ | -------------------------------- |
| **WeakMap** | Store additional data tied to object | Keys are objects, auto-cleaned   |
| **WeakSet** | Track object membership              | Values are objects, auto-cleaned |

**Key Points:**

- Weak references allow garbage collection
- No iteration, no `size` property
- Perfect for caches and data tied to object lifetimes
- Prevents memory leaks when managing external objects
