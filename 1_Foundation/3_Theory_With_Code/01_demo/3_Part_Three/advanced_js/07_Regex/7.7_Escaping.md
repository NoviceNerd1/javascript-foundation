# Escaping Special Characters Summary

Special characters in regular expressions must be escaped with a backslash `\` to be treated as literal characters.

## Special Characters

The following characters have special meaning in regexp and need escaping:

```
[ ] { } ( ) \ ^ $ . | ? * +
```

Also `/` needs escaping when using `/.../` syntax.

## Escaping with `/.../` Syntax

```javascript
// Find a literal dot
console.log("Chapter 5.1".match(/\d\.\d/)); // ['5.1']
console.log("Chapter 511".match(/\d\.\d/)); // null

// Find parentheses
console.log("function g()".match(/g\(\)/)); // ['g()']

// Find backslash
console.log("1\\2".match(/\\/)); // ['\']

// Find slash
console.log("/".match(/\//)); // ['/']
```

## Escaping with `new RegExp()`

When using `new RegExp()`, backslashes must be **doubled** because string quotes consume one backslash.

```javascript
// ❌ Wrong - backslashes are consumed
let regexp = new RegExp("\d\.\d");
console.log("Chapter 5.1".match(regexp)); // null
console.log("\d\.\d"); // "d.d" (backslashes removed)

// ✅ Correct - double backslashes
let regStr = "\\d\\.\\d";
console.log(regStr); // "\d\.\d"
let regexp2 = new RegExp(regStr);
console.log("Chapter 5.1".match(regexp2)); // ['5.1']
```

### Dynamic Regexp with Escaping

```javascript
function escapeRegexp(str) {
  return str.replace(/[.*+?^${}()|[\]\\]/g, "\\$&");
}

let userInput = "price (5.99)";
let regexp = new RegExp(escapeRegexp(userInput));
console.log(regexp); // /price \(5\.99\)/
console.log(regexp.test("price (5.99)")); // true
```

## Common Escaping Examples

| Character            | Escaped   | Example                     |
| -------------------- | --------- | --------------------------- | ---------------- | --- |
| Dot `.`              | `\.`      | `/\d\.\d/` matches `5.1`    |
| Slash `/`            | `\/`      | `/\//` matches `/`          |
| Backslash `\`        | `\\`      | `/\\/` matches `\`          |
| Parentheses `()`     | `\(` `\)` | `/g\(\)/` matches `g()`     |
| Square brackets `[]` | `\[` `\]` | `/\[abc\]/` matches `[abc]` |
| Braces `{}`          | `\{` `\}` | `/\{2\}/` matches `{2}`     |
| Asterisk `*`         | `\*`      | `/a\*b/` matches `a*b`      |
| Plus `+`             | `\+`      | `/a\+b/` matches `a+b`      |
| Question `?`         | `\?`      | `/a\?b/` matches `a?b`      |
| Caret `^`            | `\^`      | `/\^/` matches `^`          |
| Dollar `$`           | `\$`      | `/\$/` matches `$`          |
| Pipe `               | `         | `\|`                        | `/\|/` matches ` | `   |

## Practical Examples

### Escape User Input for Regexp

```javascript
function safeRegexp(text) {
  return text.replace(/[.*+?^${}()|[\]\\]/g, "\\$&");
}

let search = "hello? (world)";
let regexp = new RegExp(safeRegexp(search));
console.log(regexp); // /hello\? \(world\)/
console.log(regexp.test("hello? (world)")); // true
```

### Find Literal Dots in Filename

```javascript
let filename = "script.js";
let hasDot = /\./.test(filename);
console.log(hasDot); // true

// Find files with .js extension
let isJS = /\.js$/.test(filename);
console.log(isJS); // true
```

### Match Literal Brackets

```javascript
let code = "array[0]";
let match = code.match(/\[0\]/);
console.log(match); // ['[0]']
```

### Match Literal Asterisk

```javascript
let text = "Select * FROM users";
let match = text.match(/\*/);
console.log(match); // ['*']
```

## Escaping in Different Contexts

| Context               | Escaping Example                   |
| --------------------- | ---------------------------------- |
| `/.../` syntax        | `/\d\.\d/`                         |
| `new RegExp()` string | `new RegExp("\\d\\.\\d")`          |
| String literal        | `"\\d\\.\\d"` (double for display) |

## Summary Table

| Symbol | Escaped Form | `/.../` Example | `new RegExp()` Example |
| ------ | ------------ | --------------- | ---------------------- |
| `.`    | `\.`         | `/\./`          | `new RegExp("\\.")`    |
| `/`    | `\/`         | `/\//`          | `new RegExp("/")`      |
| `\`    | `\\`         | `/\\/`          | `new RegExp("\\\\")`   |
| `()`   | `\(\)`       | `/\(\)/`        | `new RegExp("\\(\\)")` |
| `[]`   | `\[\]`       | `/\[\]/`        | `new RegExp("\\[\\]")` |
| `{}`   | `\{\}`       | `/\{\}/`        | `new RegExp("\\{\\}")` |

## Key Points

1. **Special characters**: `[ ] { } ( ) \ ^ $ . | ? * + /`
2. **Escape with `\`** to treat as literal
3. **In `/.../`**: escape `/` and special chars
4. **In `new RegExp()`**: double backslashes (string consumes one)
5. **Dynamic regexp**: escape user input to avoid unintended special meaning
6. **Common mistake**: forgetting to escape when using `new RegExp()`
