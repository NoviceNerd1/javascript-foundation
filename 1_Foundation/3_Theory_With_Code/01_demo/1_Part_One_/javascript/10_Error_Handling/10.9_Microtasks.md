# Microtasks

## Overview

Promise handlers (`.then`/`.catch`/`.finally`) are always asynchronous. They execute after the current code finishes, even if the promise is already resolved.

## Basic Example

```javascript
let promise = Promise.resolve();

promise.then(() => alert("promise done!"));
alert("code finished");

// Output:
// code finished
// promise done!
```

Even though the promise is already resolved, the `.then` handler runs **after** the current code completes.

## Microtasks Queue (PromiseJobs)

Promise handlers are placed in an internal **microtask queue**:

- First-in-first-out order
- Tasks execute only when nothing else is running
- After current code finishes, the engine takes tasks from the queue

```
Current code → Microtask Queue → Next
```

### To Ensure Order

```javascript
Promise.resolve()
  .then(() => alert("promise done!"))
  .then(() => alert("code finished"));

// Output:
// promise done!
// code finished
```

## Unhandled Rejection Detection

The `unhandledrejection` event triggers when the microtask queue is empty and a rejected promise has no `.catch`.

```javascript
// Error is handled → no unhandledrejection
let promise = Promise.reject(new Error("Failed!"));
promise.catch((err) => alert("caught"));

// Error NOT handled → unhandledrejection triggers
let promise = Promise.reject(new Error("Failed!"));
window.addEventListener("unhandledrejection", (event) => alert(event.reason));
```

### Delayed .catch (Doesn't Prevent unhandledrejection)

```javascript
let promise = Promise.reject(new Error("Failed!"));
setTimeout(() => promise.catch((err) => alert("caught")), 1000);

// unhandledrejection triggers immediately (microtask queue empty)
// Then after 1 second, "caught" appears
window.addEventListener("unhandledrejection", (event) => alert(event.reason));
```

The `.catch` is added too late – the microtask queue has already emptied, so `unhandledrejection` fires.

## Summary

| Concept                | Description                                                          |
| ---------------------- | -------------------------------------------------------------------- |
| **Microtask queue**    | Internal queue for promise handlers                                  |
| **Execution order**    | Current code → microtasks → next                                     |
| **unhandledrejection** | Triggers when microtask queue empties with unhandled rejections      |
| **Delayed handlers**   | Added after microtask queue empties don't prevent unhandledrejection |

**Key Points:**

- Promise handlers always run asynchronously (after current code)
- Microtask queue ensures first-in-first-out execution
- Unhandled rejections are detected when microtask queue empties
- To guarantee order, chain `.then` calls
