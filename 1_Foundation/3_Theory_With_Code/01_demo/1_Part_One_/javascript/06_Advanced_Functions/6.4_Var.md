# The Old "var"

## Overview

`var` is the old way to declare variables. Modern code uses `let` and `const`. Understanding `var` helps when working with legacy code.

## Key Differences from `let`

### 1. No Block Scope

`var` is **function-scoped** or **global-scoped**, not block-scoped.

```javascript
if (true) {
  var test = true; // var, not let
}
alert(test); // true (visible outside block)

// With let - would cause error
if (true) {
  let test = true;
}
alert(test); // ReferenceError
```

**Loop example:**

```javascript
for (var i = 0; i < 10; i++) {
  var one = 1;
}
alert(i); // 10 (visible after loop)
alert(one); // 1 (visible after loop)
```

### Function Scope

Inside a function, `var` is visible throughout the function:

```javascript
function sayHi() {
  if (true) {
    var phrase = "Hello";
  }
  alert(phrase); // Works (function-scoped)
}
sayHi();
alert(phrase); // ReferenceError (not global)
```

### 2. Redeclaration Allowed

`var` allows redeclaring the same variable without error:

```javascript
var user = "Pete";
var user = "John"; // No error, just ignored
alert(user); // John

// With let - would cause error
let user = "Pete";
let user = "John"; // SyntaxError
```

### 3. Hoisting – Declared Below Usage

`var` declarations are processed at function start (hoisted). Assignments stay in place.

```javascript
function sayHi() {
  alert(phrase); // undefined (not error)
  var phrase = "Hello";
}
sayHi();
```

**How it works internally:**

```javascript
function sayHi() {
  var phrase; // Declaration hoisted
  alert(phrase); // undefined
  phrase = "Hello"; // Assignment stays
}
```

## IIFE (Immediately-Invoked Function Expression)

Before `let` existed, programmers used IIFE to create block-like scope:

```javascript
(function () {
  var message = "Hello";
  alert(message); // Hello
})();

// message is not accessible here
```

**Why parentheses?** To make JavaScript treat it as a Function Expression, not a Declaration.

```javascript
// Function Declaration - can't call immediately
function go() { }();  // SyntaxError

// Function Expression - can be called immediately
(function() { })();   // Works
```

**Other IIFE syntax variants:**

```javascript
(function () {})();
(function () {})();
!(function () {})();
+(function () {})();
```

## Comparison: `var` vs `let`

| Feature               | `var`                 | `let`                    |
| --------------------- | --------------------- | ------------------------ |
| Block scope           | ❌                    | ✅                       |
| Redeclaration allowed | ✅                    | ❌                       |
| Hoisting              | Yes (value undefined) | Yes (temporal dead zone) |
| Global property       | Creates               | Doesn't create           |

## Summary

- **No block scope** – visible throughout function or globally
- **Redeclaration** – allowed without error
- **Hoisting** – declaration moved to function start, assignment stays
- **IIFE** – old pattern to emulate block scope (no longer needed)
- **Modern code** – use `let` and `const` instead

> ⚠️ **Note:** This chapter is for understanding legacy code. Modern JavaScript uses `let` and `const`.
