# Fetch: Download Progress Summary

`fetch()` can track download progress using `response.body` (a `ReadableStream`), but cannot track upload progress (use `XMLHttpRequest` for that).

## Basic Progress Tracking

```javascript
const response = await fetch(url);
const reader = response.body.getReader();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  console.log(`Received ${value.length} bytes`);
}
```

- `reader.read()` returns `{done, value}` where `value` is `Uint8Array`
- Loop continues until `done === true`

## Complete Download Progress Example

```javascript
// Step 1: Start fetch and get reader
let response = await fetch(
  "https://api.github.com/repos/javascript-tutorial/en.javascript.info/commits?per_page=100",
);
const reader = response.body.getReader();

// Step 2: Get total length (if available)
const contentLength = +response.headers.get("Content-Length");

// Step 3: Read data chunks
let receivedLength = 0;
let chunks = [];

while (true) {
  const { done, value } = await reader.read();

  if (done) break;

  chunks.push(value);
  receivedLength += value.length;

  console.log(
    `Received ${receivedLength} of ${contentLength} (${Math.round((receivedLength / contentLength) * 100)}%)`,
  );
}

// Step 4: Concatenate chunks
let chunksAll = new Uint8Array(receivedLength);
let position = 0;
for (let chunk of chunks) {
  chunksAll.set(chunk, position);
  position += chunk.length;
}

// Step 5: Decode to string
let result = new TextDecoder("utf-8").decode(chunksAll);
let commits = JSON.parse(result);
```

## Alternative: Using Blob (for Binary Data)

```javascript
let response = await fetch(url);
const reader = response.body.getReader();

let receivedLength = 0;
let chunks = [];

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  chunks.push(value);
  receivedLength += value.length;
  console.log(`Downloaded ${receivedLength} bytes`);
}

// Create blob from chunks
let blob = new Blob(chunks);

// Use blob (e.g., for image)
let imgUrl = URL.createObjectURL(blob);
document.getElementById("img").src = imgUrl;
```

## Progress Bar Implementation

```html
<progress id="progressBar" value="0" max="100"></progress>
<div id="status"></div>

<script>
  async function downloadWithProgress(url) {
    const response = await fetch(url);
    const contentLength = +response.headers.get('Content-Length');
    const reader = response.body.getReader();

    let receivedLength = 0;
    let chunks = [];
    const progressBar = document.getElementById('progressBar');
    const status = document.getElementById('status');

    while(true) {
      const {done, value} = await reader.read();
      if (done) break;

      chunks.push(value);
      receivedLength += value.length;

      const percent = (receivedLength / contentLength * 100).toFixed(1);
      progressBar.value = percent;
      status.textContent = `Downloaded ${percent}% (${receivedLength} / ${contentLength} bytes)`;
    }

    // Combine chunks
    let chunksAll = new Uint8Array(receivedLength);
    let position = 0;
    for(let chunk of chunks) {
      chunksAll.set(chunk, position);
      position += chunk.length;
    }

    return chunksAll;
  }

  // Usage
  const data = await downloadWithProgress('https://example.com/large-file.zip');
</script>
```

## Handling Unknown Content Length

When `Content-Length` header is missing:

```javascript
async function downloadStream(url, maxBytes = 10 * 1024 * 1024) {
  // 10MB limit
  const response = await fetch(url);
  const reader = response.body.getReader();

  let receivedLength = 0;
  let chunks = [];

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    chunks.push(value);
    receivedLength += value.length;

    // Safety limit to prevent memory issues
    if (receivedLength > maxBytes) {
      console.warn(`Exceeded ${maxBytes} bytes limit`);
      break;
    }

    console.log(`Downloaded ${receivedLength} bytes (size unknown)`);
  }

  return new Blob(chunks);
}
```

## Important Notes

| Aspect                | Details                                                   |
| --------------------- | --------------------------------------------------------- |
| **Download progress** | ✅ Supported via `response.body.getReader()`              |
| **Upload progress**   | ❌ Not supported in `fetch()` (use `XMLHttpRequest`)      |
| **One-time read**     | Cannot use both reader and `response.json()`/`text()`     |
| **Content-Length**    | May be absent (cross-origin or server not setting it)     |
| **Memory**            | Store chunks until complete; monitor size for large files |

## Limitations

1. **Cannot use both reader and response methods** - choose one:

   ```javascript
   // Either:
   let reader = response.body.getReader();

   // OR:
   let data = await response.json(); // Not both!
   ```

2. **Upload progress** - Not available in `fetch()`:

   ```javascript
   // Use XMLHttpRequest for upload progress
   let xhr = new XMLHttpRequest();
   xhr.upload.onprogress = (event) => {
     console.log(`Uploaded ${event.loaded} of ${event.total}`);
   };
   ```

3. **Memory management** - Large downloads require chunk handling or streaming to disk

## Summary

- Use `response.body.getReader()` to get a `ReadableStream` reader
- Loop `await reader.read()` until `done === true`
- Track `receivedLength` for progress
- Concatenate chunks using `Uint8Array.set()` or create a `Blob`
- Decode with `TextDecoder` for text, or use `Blob` for binary data
